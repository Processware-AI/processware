# 04_AsIs

## 들어가는 것
- 고객사가 **이미 사용 중인** 정책·절차·지침·양식
- 내부 매뉴얼, 운영 규정, 조직도
- 과거 심사·감사 리포트 (이슈 이력)
- 기존 RACI·업무 분장표

## 예시
```
04_AsIs/
├── 현_품질매뉴얼_v3.docx
├── 현_문서관리규정.pdf
├── 2024_내부심사_리포트.pdf
├── 조직도_2026.png
└── 기존_양식_모음/
    ├── 개정요청서_양식.xlsx
    └── 변경관리_양식.docx
```

## 들어가지 않는 것
- 표준 원본 → `01_표준원문/`
- 외부 법규 → `02_법규/`
- 다른 고객사 자료 → **투하 금지** (유출)

## 저작권·기밀 (CRITICAL)
- 고객사 **내부 자산** — 외부 공개 금지
- `.gitignore` 로 완전 제외 (공개 repo 에 절대 올라가지 않음)
- **타 고객 산출물에 섞이지 않도록 엄격 분리**
- 마스킹 필요: 개인정보·계약가·고객 목록 등

## 활용 가치 (매우 큼)
- 고객사 **용어·문체·체계 존중** → 산출물이 현업에서 실제 쓰이려면 필수
- 기존 승인 체계·RACI 유지
- **신규가 아닌 개선 접근** (기존 확장 우선, 구성원칙 §8)

## 에이전트 동작
- `process-designer`: POL/PRO 설계 시 **신규 생성보다 기존 확장 우선**
- `wi-tmp-writer`: **기존 양식 계승** — TMP 신규 생성보다 기존 정제
- `qa-reviewer`: 기존 자산 반영 여부 점검

## 변환 권장
- **원본 그대로 복사** (내부 자산)
- MD 변환 가능하면 Grep·링크 유리
- PDF/Word/Excel 모두 허용 (작은 파일 위주)

## 에이전트 참조 우선순위
별도 축. 표준·법규 준수 위에서 **고객사 용어·구조 존중** 용도.
