# 문서번호 체계

## 기본 파일명 형식
```
[유형코드]-[식별번호]_[문서명]_v[버전].md
```

## 계층 번호 설계 원칙

POL 번호를 기준으로 하위 문서가 번호를 계승하여, 코드만으로 문서 계보를 추적할 수 있다. 모든 단위는 **2자리 zero-padding** 으로 통일한다.

```
POL-{영역}-{##}                                          (POL 2자리)
 └─ PRO-{영역}-{POL2}-{##}                               (POL2 + PRO 세부2)
      └─ WI-{영역}-{POL2}-{PRO2}-{##}                    (POL2-PRO2-WI세부2)
           ├─ TMP-{영역}-{POL2}-{PRO2}-{WI2}-{##}        (WI 6자리 + TMP 일련2)
           ├─ EX-{영역}-{POL2}-{PRO2}-{WI2}-{##}         (TMP 와 동일 구조)
           └─ REC-{영역}-{POL2}-{PRO2}-{WI2}-{TMP2}-{YYYY}-{###}
                                                         (TMP/EX 식별 + 연도 + 일련3)
```

## 유형별 식별번호 규칙

| 유형 | 식별번호 규칙 | 예시 |
|---|---|---|
| POL | `POL-{영역}-{##}` (2자리) | `POL-QMS-01_품질방침_v1.0.md` |
| PRO | `PRO-{영역}-{POL2}-{##}` (POL2-세부2) | `PRO-MDCS-04-01_접근통제_및_인증_관리_절차_v1.0.md` |
| WI  | `WI-{영역}-{POL2}-{PRO2}-{##}` (POL2-PRO2-WI2) | `WI-QMS-01-01-02_문서_검토_및_승인_v1.0.md` |
| TMP | `TMP-{영역}-{POL2}-{PRO2}-{WI2}-{##}` (WI6+TMP2) | `TMP-QMS-01-01-02-01_검토_확인서_v1.0.md` |
| EX  | `EX-{영역}-{POL2}-{PRO2}-{WI2}-{##}` | `EX-QMS-01-01-02-01_검토_확인서_작성예시_v1.0.md` |
| REC | `REC-{영역}-{POL2}-{PRO2}-{WI2}-{TMP2}-{YYYY}-{###}` | `REC-QMS-01-01-02-01-2026-001_검토_확인서.md` |
| MAT | `MAT-{###}` (3자리 고정 — 별도 축, 변경 없음) | `MAT-002_규제요구사항_대조표_v1.0.md`, `MAT-011_{표준코드}_추적성_v1.0.md` |
| REF | `REF-{###}` (3자리 고정 — 별도 축, 변경 없음) | `REF-001_개인정보보호법_요약_v1.0.md` |

### 자릿수 배정 원칙

- **POL 2자리** (`01~99`): 단일 영역 내 POL 99개까지. 일반 조직 10~30개 수준이라 충분.
- **PRO 세부 2자리** (`01~99`): 단일 POL 하위 PRO 99개까지.
- **WI 세부 2자리** (`01~99`): 단일 PRO 하위 WI 99개까지.
- **TMP/EX 일련 2자리** (`01~99`): 단일 WI 하위 TMP/EX 99개까지. **단일 양식이라도 `-01` 부여 (필수)**.
- **REC 일련 3자리** (`001~999`): 연도별 단일 TMP 양식 기반 기록 999건까지.

> **계층 변경의 연쇄 효과**: POL 번호 변경 시 하위 PRO/WI/TMP/EX/REC 모두 연쇄 변경.
> **REC는 버전 불필요**: 기록은 수정 통제 대상 아님. 재발행 시 신규 REC.
> **MAT/REF 는 별도 축**: 계층 외 전사 공통 매트릭스·참고자료. 3자리 자릿수 그대로 유지.

## MAT 번호 할당 원칙

MAT 는 3자리 순차 번호로, 역할에 따라 구역을 나누어 관리한다.

| 구역 | 용도 | 할당 규칙 |
|---|---|---|
| **MAT-001 ~ MAT-010** | 전사 공통 관리 매트릭스 (최대 10종) | 재배정 금지. 현재 9종 운영 (001~009), MAT-010 은 flow-mapper 실행 시 자동 파생 |
| **MAT-011 ~ MAT-099** | 표준별 추적성 매트릭스 | 표준 편입 순서대로 순차 부여 |
| **MAT-100 이후** | 예약 (통합 뷰·교차 매트릭스 등) | 필요 시 규칙 합의 후 사용 |

| 번호 | 문서 | 도입 차원 | 상태 |
|---|---|---|---|
| MAT-001 | 문서관리대장 | 차원 1 / 4 (§개정 이력 자동 누적) | 운영 |

| MAT-006 | 문서 계층 추적 매트릭스 (POL→PRO→WI→TMP→EX) | 차원 1 | 운영 |

### 표준별 추적성 (순차, MAT-011 부터)
파일명: `MAT-{###}_{표준코드}_추적성_v{버전}.md`

| 번호 | 표준 | 편입일 | 비고 |
|---|---|---|---|
| (현재 등재된 표준 없음 — 첫 표준 편입 시 MAT-011 부여) | | | |

`traceability-mapper` 는 표준 편입 시 **MAT-011 부터 순차적으로 다음 미사용 번호**를 부여한다. 번호 재사용 금지 — 표준이 삭제되어도 그 번호는 결번 처리.

## 버전 규칙
- **v0.x** : 초안(draft) — 미승인
- **v1.0** : 최초 승인판
- **v1.n** : 경미 개정 (오탈자·링크·소절 수정)
- **v{n+1}.0** : 본문 구조/책임/범위 변경
- **REC 는 버전 불필요** (기록은 수정 통제 대상, 재발행 시 신규 REC로)
