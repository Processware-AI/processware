# 문서번호 체계

## 기본 파일명 형식
```
[유형코드]-[식별번호]_[문서명]_v[버전].md
```

## 계층 번호 설계 원칙

POL 번호를 기준으로 하위 문서가 번호를 계승하여, 코드만으로 문서 계보를 추적할 수 있다.

```
POL-{영역}-{###}
 └─ PRO-{영역}-{P}{##}                         (P = POL 일련번호, ## = POL 내 순번)
      └─ WI-{영역}-{POL###}-{PRO##}-{##}        (POL번호 + PRO 십일자리 + WI 순번)
           ├─ TMP-{영역}-{POL###}-{PRO##}-{WI##}
           ├─ EX-{영역}-{POL###}-{PRO##}-{WI##}
           └─ REC-{영역}-{POL###}-{PRO##}-{WI##}-{YYYY}-{##}
```

## 유형별 식별번호 규칙

| 유형 | 식별번호 규칙 | 예시 |
|---|---|---|
| POL | `POL-{영역}-{###}` | `POL-QMS-001_품질방침_v1.0.md` |
| PRO | `PRO-{영역}-{P}{##}` | `PRO-MDCS-401_접근통제_및_인증_관리_절차_v1.0.md` |
| WI  | `WI-{영역}-{POL###}-{PRO##}-{##}` | `WI-QMS-001-01-02_문서_검토_및_승인_v1.0.md` |
| TMP | `TMP-{영역}-{POL###}-{PRO##}-{WI##}` | `TMP-QMS-001-01-02_검토_확인서_v1.0.md` |
| EX  | `EX-{영역}-{POL###}-{PRO##}-{WI##}` | `EX-QMS-001-01-02_검토_확인서_작성예시_v1.0.md` |
| REC | `REC-{영역}-{POL###}-{PRO##}-{WI##}-{YYYY}-{##}` | `REC-QMS-001-01-02-2026-001_검토_확인서.md` |
| MAT | `MAT-{###}` (3자리 고정) | `MAT-002_규제요구사항_대조표_v1.0.md`, `MAT-011_{표준코드}_추적성_v1.0.md` |
| REF | `REF-{###}` | `REF-001_개인정보보호법_요약_v1.0.md` |

### PRO 번호 배정 원칙
PRO 3자리 번호는 `{P}{##}` 구조로 구성한다.
- **P (백의 자리)**: 부모 POL의 일련번호와 일치시킨다.
  - POL-001 하위 PRO → 1xx (예: 101, 102, ...)
  - POL-002 하위 PRO → 2xx (예: 201, 202, ...)
  - POL-003 하위 PRO → 3xx, POL-004 → 4xx, ... (이하 동일)
- **{##} (십·일의 자리)**: 해당 POL 하위에서 생성 순서대로 01부터 순차 부여.
- WI는 `WI-{영역}-{POL###}-{PRO##}-{##}` 규칙에 따라 POL·PRO 번호를 계층적으로 계승한다.
  따라서 PRO 번호 변경 시 하위 WI 번호도 연쇄 변경된다.

> **WI당 TMP·EX가 복수인 경우**: `TMP-{영역}-{POL###}-{PRO##}-{WI##}-{##}` 형식으로 끝에 2자리 순번 추가.
> **REC는 버전 불필요**: 기록은 수정 통제 대상이 아니므로 YYYY-## 로 연도별 일련번호만 부여.

## MAT 번호 할당 원칙

MAT 는 3자리 순차 번호로, 역할에 따라 구역을 나누어 관리한다.

| 구역 | 용도 | 할당 규칙 |
|---|---|---|
| **MAT-001 ~ MAT-010** | 전사 공통 관리 매트릭스 (최대 10종) | 재배정 금지. 현재 6종 사용 (001~006), 007~010 은 예약 |
| **MAT-011 ~ MAT-099** | 표준별 추적성 매트릭스 | 표준 편입 순서대로 순차 부여 |
| **MAT-100 이후** | 예약 (통합 뷰·교차 매트릭스 등) | 필요 시 규칙 합의 후 사용 |

## 버전 규칙
- **v0.x** : 초안(draft) — 미승인
- **v1.0** : 최초 승인판
- **v1.n** : 경미 개정 (오탈자·링크·소절 수정)
- **v{n+1}.0** : 본문 구조/책임/범위 변경
- **REC 는 버전 불필요** (기록은 수정 통제 대상, 재발행 시 신규 REC로)
