# 전사 용어집

> 모든 산출물·에이전트 프롬프트에서 사용하는 용어의 단일 출처. 항목 추가 시 알파벳/가나다 순 유지.

## A–Z

| 용어 | 영문 | 정의 | 관련 표준 |
|---|---|---|---|
| CAPA | Corrective Action / Preventive Action | 시정·예방조치. 부적합(NCR) 원인 제거 + 재발 방지 대책 | ISO 9001 §10 |
| GAP 분석 | GAP Analysis | 외부 표준 요건 대비 내부 산출물(POL/PRO/WI)의 커버리지 판정. 결과: COVERED / PARTIAL / GAP / N/A | — |
| HITL | Human-In-The-Loop | AI 자동화 파이프라인에서 사람이 개입하는 확정·승인 단계. 에이전트 제안 → 사람 확정 구조 | — |
| HLS | High-Level Structure | ISO 경영시스템 표준의 공통 목차(조항 4~10). HLS 기반 표준은 통합 운영(IMS)이 가능 | ISO Annex SL |
| IMS | Integrated Management System | 품질·환경·보안 등 여러 경영시스템을 HLS 기반으로 통합 운영 | ISO Annex SL |
| KPI | Key Performance Indicator | 핵심 성과지표. 이 플랫폼에서는 PRO/WI §KPI 절에서 자동 추출·추적 | — |
| Library Module | — | Redmine에서 vault 관리하는 표준 프로세스 모듈. `lib-{scope_code}-{pro_slug}` 형식. 직접 편집 금지 | — |
| MOC | Map of Content | Obsidian 허브 노트(목차·랜딩 페이지). `00_MOC/` 폴더 | — |
| NCR | Non-Conformance Report | 부적합 보고서. 심사(Check)에서 발행. REC-NCR 형식. MAT-009 관리대장에 누적 | ISO 9001 §10 |
| PDCA | Plan-Do-Check-Act | 지속 개선 사이클. 이 플랫폼은 4차원으로 확장(차원 0 Ingest 포함) | ISO 9001 |
| PCB | Process Control Board | 프로세스 표준화 최고 의사결정체. 개정안·GAP 대응·아키텍처 변경 승인 | — |
| PDD | Process Definition Document | 프로세스 정의서. 이 체계에서는 **PRO** 에 해당 | — |
| RACI | Responsible/Accountable/Consulted/Informed | 책임 분담 매트릭스. 모든 PRO 필수 포함 | — |
| RCA | Root Cause Analysis | 근본 원인 분석. 5-Why / Fishbone 기법. 차원 4 Act 시작 단계 | — |
| RBAC | Role-Based Access Control | 역할 기반 접근 제어. 6 역할: auditor / executor / process_owner / qmr / admin / viewer | — |
| REC | Record | 기록본. 실제 업무 수행 후 생성되는 증빙 문서. `REC-{영역}-...` 형식 | — |
| SBOM | Software Bill of Materials | 소프트웨어 구성 명세 | NIST |
| SEPG | Software Engineering Process Group | 프로세스 엔지니어링·품질보증 조직 | CMMI |
| SLA | Service Level Agreement | 서비스 수준 협약. 이 플랫폼에서는 NCR 해결 기한(critical 20영업일 / major 60일 / minor 90일)에 적용 | — |
| SSoT | Single Source of Truth | 단일 정보 원본. vault 가 모든 표준 프로세스 자산의 SSoT | — |
| Workspace | — | 현업 업무 단위를 묶은 가상 공간. Redmine에서 `ws-{slug}` 상위 프로젝트 + PRO 단위 서브프로젝트로 구현. Library 복사본 → 팀 자율 편집 | — |

## 가–힣

| 용어 | 영문 | 정의 | 관련 표준 |
|---|---|---|---|
| 감사 증적 | Audit Evidence | 심사 판정의 근거가 되는 기록. REC + trace.jsonl + source_citation | ISO 9001 §9 |
| 개정 이력 | Revision History | 문서 버전 변경 내역. MAT-001 §개정이력 절에 차원 4 act-coordinator가 자동 누적 | — |
| 골든샘플 | Golden Sample | POL/PRO/WI 품질 하한선 참조 예시. `vault/99_템플릿/_골든샘플/` | — |
| 부적합 | Non-Conformance | 요구사항 미충족. 차원 3 심사에서 finding으로 식별 → NCR 발행 | ISO 9001 §10 |
| 업무지침서 | Work Instruction | WI. 담당자 관점의 단계별 실무 수행 지침. 이중 포맷: WI-*.md(사람용) + WI-*.steps.yaml(Agent 실행용) | — |
| 영역코드 | Scope Code | 표준 영역을 식별하는 3~5자 대문자 약어. 예: QMS, ISMS, CMMI. 파일명·프로젝트 식별자에 사용 | — |
| 적용범위 | Scope | 프로세스·시스템이 적용되는 조직·제품·서비스·지역의 경계 | ISO 9001 §4.3 |
| 절차서 | Procedure | PRO. 업무 흐름·관리 절차를 기술. RACI·KPI·Mermaid 다이어그램 필수 포함 | — |
| 정책서 | Policy | POL. 조직의 방침·원칙·책임을 기술. 실무 절차 포함 금지 | — |
| 추적성 | Traceability | 표준 요건 ↔ POL ↔ PRO ↔ WI ↔ TMP ↔ REC 간 연결 관계. MAT-001~010 + MAT-011~ 에서 관리 | ISO 9001 §7.5 |
| 테일러링 | Tailoring | 조직·프로젝트 특성에 맞게 표준 프로세스를 축소·변형하는 것. 테일러링 근거는 기록 필수 | CMMI, ASPICE |
| 템플릿 | Template | TMP. 기록 작성용 빈 양식. 샘플 데이터 포함 금지 (→ EX 로 분리) | — |
| 폐쇄 루프 | Closed Loop | Plan→Do→Check→Act→Plan' 사이클이 끊김 없이 자동으로 순환되는 구조 | — |
| 표준 | Standard | ISO/IEC/KS/법규 등 외부 요구사항 문서. `/process-ingest` 로 편입 가능 | — |
| 형상 관리 | Configuration Management | 산출물의 버전·변경·이력을 체계적으로 관리하는 활동. vault + git 으로 구현 | ASPICE SUP.8 |
