# 문서체계 — 8종 문서 유형

> 상위 기준: [[표준프로세스_구성원칙]]

## 8종 유형 개요

| 코드 | 유형 | 목적 | 작성 시점 | 판단 기준 |
|---|---|---|---|---|
| **POL** | 정책서 | 원칙·방침·책임 정의 | 체계 수립 시 | "우리 조직은 어떤 원칙으로 관리하는가" |
| **PRO** | 절차서 | 업무 흐름·관리 절차 | 정책 하위 | "업무가 어떤 순서로 관리되는가" |
| **WI**  | 업무지침서 | 실무 수행 방법 상세 | 절차 하위 | "담당자가 실제로 어떻게 수행하는가" |
| **TMP** | 템플릿 | 기록/산출물 양식(빈 양식) | WI 연계 | "복사해서 기록 작성에 사용 가능한가" |
| **EX**  | 작성예시 | 템플릿 작성 샘플 | TMP 연계 | "예시 값이 채워진 교육용 샘플인가" |
| **REC** | 기록본 | 실제 수행 증빙 | 업무 수행 후 | "운영 중 생성된 실제 기록인가" |
| **MAT** | 매핑/관리대장 | 추적성·목록·대조표 | 체계 운영 시 | "문서 간 관계/상태를 관리하는가" |
| **REF** | 참고자료 | 외부 규정·가이드 요약 | 상시 | "없어도 체계는 돌지만 있으면 이해가 쉬운 자료" |

## 상하관계
```
POL
└─ PRO
   └─ WI ─── (이중 포맷)
      ├─ WI-*.md          ← 사람이 읽는 서술형 지침서 (정본)
      └─ WI-*.steps.yaml  ← 차원 2 Agent 가 실행하는 step 명세 (자매 파일)
      │
      ├─ TMP  (빈 양식)
      ├─ EX   (작성예시)
      └─ REC  (실제 기록)

MAT = 전체 문서 추적/통합 관리   (별도 축)
REF = 참조용 자료                (별도 축)
```

> **WI 이중 포맷 (Phase 4 도입)**: WI 는 MD(사람용) 와 steps.yaml(Agent 실행용) 을 짝으로 보관한다. wi-tmp-writer 가 차원 1 빌드 시 둘 다 생성·동기화하며, qa-reviewer §11-A 가 메타·본문 일치를 검증한다. 자세한 스키마는 [[T16_WI_steps_yaml.yaml]] 참조.

## 핵심 운영 원칙
1. **정책서는 짧고 명확**하게 — 실무 세부 절차 금지
2. **절차서는 흐름 중심** — 수행법보다 관리흐름·책임
3. **업무지침은 담당자 관점** — 입출력·단계 명시
4. **템플릿은 빈 양식만** — 샘플 데이터 금지
5. **예시는 별도 문서(EX)** 로 분리
6. **템플릿과 기록은 같은 폴더에 두지 않기** — 원본/사용본 혼재 방지

## 골든 샘플 (품질 하한선)
POL/PRO/WI 유형별로 "이 정도 수준으로 써야 한다" 의 참조 예시가 `vault/99_템플릿/_골든샘플/` 에 있음. 모든 산출물은 해당 유형의 골든샘플의 **필수 섹션 구조** 를 충족해야 함. 상세: [[_골든샘플/README]]
