# 영향 평가 보고서 작성예시

> 본 문서는 작성 요령 학습용 예시입니다. 가상의 ABC Motors Co., Ltd. ADAS DCU V2 프로젝트(ABC-ADAS-2026-001) 데이터를 사용합니다.

## 작성정보
| 항목 | 내용 |
|---|---|
| 영향 평가 ID | IA-ABC-ADAS-2026-001-CR-007 |
| 작성일 | 2026-05-06 |
| 작성자 (Change Control Engineer) | 임변경 (Change Control Engineer) |
| 승인자 (CCB Chair) | 윤시비 (CCB Chair, Program Director) |

## 1. 변경 요청 참조
| 항목 | 내용 |
|---|---|
| CR-ID | CR-ABC-ADAS-2026-001-CR-007 |
| CR 제목 | CAN-FD 메시지 ID 변경 (0x101 → 0x201) — OEM 공통 게이트웨이 정합 |
| CR 등록일 | 2026-05-04 |
| 요청자 | 김OEM (System Engineer, ABC Motors 차량통신팀) |
| 변경 사유 | OEM 공통 게이트웨이 메시지 ID 표준 갱신에 따른 정합 — 기존 0x101 이 다른 ECU 와 충돌 위험 |

## 2. 변경 유형 분류
| 항목 | 내용 |
|---|---|
| 분류 | Minor |
| 분류 근거 | 메시지 페이로드 구조·주기 불변, ID 만 변경. ASIL 영향 없음. 영향 CI 2종, 작업량 < 3 MD. |

## 3. 기술 영향
| 항목 | 내용 |
|---|---|
| 영향 받는 CI 목록 | CI-ABC-SRC-001 (펌웨어: canfd_msg.c, canfd_msg.h), CI-ABC-DOC-003 (SW 인터페이스 명세서 §4.2 CAN-FD 메시지표) |
| 변경되는 인터페이스 | CAN-FD 외부 인터페이스 (게이트웨이 ↔ DCU) — 메시지 ID 0x101 → 0x201 |
| 추적성 변경 (RTM 갱신 범위) | SWE-REQ-027 (CAN-FD 통신) ↔ canfd_msg.c ↔ TC-COMM-012 추적 갱신 (ID 변경 반영) |
| 영향 받는 산출물 | 펌웨어 v1.0.0 → v1.0.1, SW 인터페이스 명세서 v1.0 → v1.1, 통합 테스트 케이스 TC-COMM-012 |

## 작성 요령
- **영향 받는 CI 는 RTM 역추적**: 직관 추정 금지. 추적성 매트릭스로 모든 종속 CI 식별.
- **ASIL·TARA 평가 의견은 서명 포함**: Safety/Cybersecurity Engineer 의 명시 의견 + 서명 없으면 안전 게이트 통과 불가.
- **권고안에는 조건 명시**: 조건부 승인 시 어떤 조건이 충족되어야 종결되는지 측정 가능하게 기재.
- **종속 CR 명기**: 동일 CI 영향 CR 이 동시 진행 시 의존 관계 명시.

## 잘못된 사례
- "영향 적음" 같은 모호한 분류 — Minor/Major/Emergency 정량 기준 적용 누락.
- ASIL 영향 평가 없이 Major 변경 처리 — 안전 게이트 미통과, 추후 사고 시 책임.
- RTM 역추적 없이 "관련 파일 추정" — 영향 누락 위험.
- CCB 권고안 "검토 요망" 같은 책임 회피성 표현 — 명확한 결정 권고 필수.
