## 작성 요령
- **PCA·FCA 분리 수행**: PCA 는 CM Engineer, FCA 는 QA 가 독립적으로 수행하여 객관성 확보. 동일인 수행 금지.
- **체크섬 비교는 자동화**: 수기 비교 금지. `sha256sum` + diff 스크립트로 전수 비교.
- **추적성 단절도 FCA Fail**: 요구사항 → 시험 추적성이 끊긴 항목은 충족 증거 부족으로 Fail 처리.
- **Conditional Pass 시 잔존 위험 명문화**: 고객 동의 서명까지 받아야 인도 가능.

## 잘못된 사례
- "전부 OK" 만 적고 CI별·요구사항별 결과 미상세 → 어떤 항목을 검사했는지 추적 불가, 감사 무효.
- PCA 와 FCA 를 같은 사람이 수행 → 독립성 위반, ASPICE 평가 시 GP 2.1 미흡 판정 위험.
- Critical 불일치 발견했는데 인도 "Pass" 판정 → 절차 위반, 추후 사고 시 책임 추궁.
- 잔존 위험 없이 Conditional Pass 표기 → 무엇이 조건인지 불명확.
