# 변경요청서 작성예시 (EX-CMMI-04-01-02-01)

> 원본 양식: [[TMP-CMMI-04-01-02-01_변경요청서_CR]]

> ⚠️ 본 문서는 **교육·가이드용 샘플**.

## 샘플 컨텍스트
"**NEXT 플랫폼 v3.0**" 의 인증 모듈 변경 요청 예시.

## 1. CR 식별 (샘플)
| 항목 | 예시값 |
|---|---|
| CR ID | CR-2026-0042 |
| 접수 일시 | 2026-05-02 10:15 |
| 요청자 | 윤OO (QA Lead) |
| 요청 부서 | QA 팀 |
| 우선순위 | High |
| 대상 베이스라인 ID | BL-NEXT3-REQ-001 |

## 2. 변경 요청 내용 (샘플)
| 항목 | 내용 |
|---|---|
| 변경 요약 | OAuth2 토큰 만료 시간을 2h → 1h 로 단축 |
| 변경 사유 | 보안 감사 결과 권고. 세션 탈취 위험 완화 |
| 기대 효과 | 보안 등급 상향, 침해 영향 범위 축소 |

## 3. 영향 받는 CI (샘플)
| CI ID | 현재 버전 | 변경 유형 |
|---|---|---|
| CI-CODE-AUTH-001 | 1.2.0 | 개정 |
| CI-API-001 (OpenAPI) | 1.0 | 개정 (문서) |
| CI-REQ-002 (운영 시나리오) | 1.0 | 개정 (참조 절) |

## 4. 영향평가 (샘플)
| 항목 | 평가 결과 |
|---|---|
| 기술 영향 | 토큰 갱신 빈도 ↑ → 클라이언트 SDK 핸들링 변경 필요 |
| 비용 영향 | 약 2,000,000원 (SDK 패치+테스트) |
| 일정 영향 | 5영업일 |
| 리스크 영향 | Medium (클라이언트 회귀 가능성) |
| 평가자·일자 | 정OO (Architect) 2026-05-03 |

## 작성 시 유의사항
- 우선순위는 비즈니스 영향 + 보안·법규 영향 함께 고려.
- 영향평가는 평가자 단일이 아닌 다관점(개발·QA·아키텍처) 권장.
- 조건부 승인 시 조건 해소 증빙을 후속 처리란에 명시.

## 잘못된 작성 사례
> ❌ 영향 CI 미식별·"전체 시스템" 식 모호 표기
> ✅ 구체 CI ID 와 버전 명시

> ❌ 비용·일정 영향을 "거의 없음" 으로 표기
> ✅ 산정 근거(인일·금액) 명기
