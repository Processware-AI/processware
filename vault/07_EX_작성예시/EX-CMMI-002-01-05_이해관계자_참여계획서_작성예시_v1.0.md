## 2. 이해관계자 등록부
| ID | 이해관계자 | 역할 | 영향력 | 관심도 |
|---|---|---|---|---|
| SH-01 | 마케팅본부장 | 비즈니스 후원자 | High | High |
| SH-02 | 콜센터 운영팀장 | 최종 사용자 | Med | High |
| SH-03 | 보안팀 | 컴플라이언스 | High | Med |
| SH-04 | 외부 결제사 | API 인터페이스 | Med | Med |

## 3. 참여 전략
| ID | 빈도 | 채널 | 산출 |
|---|---|---|---|
| SH-01 | 격주 | Steering | 진행 보고 |
| SH-02 | 스프린트 종료 | 데모 | 피드백 |
| SH-03 | 마일스톤 | 보안 게이트 검토 | 통과 증적 |
| SH-04 | 분기 | 인터페이스 회의 | API 합의 |

## 4. 의사소통 일정
| 회의/보고 | 대상 | 주기 | 책임 |
|---|---|---|---|
| Steering | SH-01, CTO | 격주 | PM |
| 데모 | SH-02 | 스프린트 (2주) | PM |
| 보안 게이트 | SH-03 | MS1·MS2·MS3 | PM + 보안팀 |

## 작성 요령
- 영향력×관심도 매트릭스로 우선순위 4분면 활용.
- 채널은 비공식·공식 구분.

## 잘못된 사례
- 모든 SH 영향력=High — 우선순위 상실.
