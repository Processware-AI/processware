# 시스템 통합 시험 계획서 및 결과 보고서 작성예시 (EX-ASPICE-01-01-04-01)

> 원본 양식: [[TMP-ASPICE-01-01-04-01_통합시험계획서및결과보고서]]

## 작성 정보 (예시)
| 항목 | 내용 |
|---|---|
| 프로젝트 코드 | VW-ADAS-2026-001 |
| Stage | Stage 2 (Perception + Decision SW 통합) |
| 작성자 | 김시스 |
| 검토자 | 이큐에이 (QA) |
| 승인자 | 최리드 |

## 2. 통합 전략·단계 (예시)
| Stage | 범위 | 환경 | 종료 기준 |
|---|---|---|---|
| 1 | Camera HW + Perception SW | Bench | 입력 영상→객체리스트 출력, FPS ≥ 30 |
| 2 | Perception + Decision SW | HIL | LKA/AEB 결정 신호 송출, 지연 ≤ 100ms |
| 3 | Decision + Vehicle CAN GW | HIL + 차량 | CAN 신호 실차 송출 검증 |

## 3. 시험 케이스 (예시 발췌)
| TC-ID | Stage | ICD | 입력 | 기대 출력 | 합격 기준 |
|---|---|---|---|---|---|
| TC-INT-001 | 2 | ICD-002 | 정지 차량 시뮬 (50m 전방) | AEB Pre-Brake 신호 송출 | 95ms 이내 |
| TC-INT-002 | 2 | ICD-002 | 보행자 시뮬 (10m 측면) | LKA 차선유지 + 경고음 | 100ms 이내 |
| TC-INT-003 | 2 | ICD-002 | 빈 도로 (false positive 시험) | 어떤 신호도 송출 X | 0건/1000회 |

## 4. 실행 결과 로그 (예시)
| TC-ID | 실행일 | 환경 | Pass/Fail | 측정값 | 결함 ID |
|---|---|---|---|---|---|
| TC-INT-001 | 2026-06-15 | HIL #2 | Pass | 88 ms | - |
| TC-INT-002 | 2026-06-15 | HIL #2 | Pass | 95 ms | - |
| TC-INT-003 | 2026-06-15 | HIL #2 | Fail | FP 3건/1000회 | DEF-2026-0142 |

## 5. 결함 요약 (예시)
| 결함 ID | 심각도 | 인터페이스 | 상태 |
|---|---|---|---|
| DEF-2026-0142 | Major | Perception output | Open (담당: ML 팀) |

## 6. 잔여 리스크 / 인계 결정 (예시)
- 잔여 리스크: False Positive 0.3% (목표 0%)
- SYS.5 인계: No
- 사유: DEF-2026-0142 폐쇄 후 재인계 (목표 6/22)

## 작성 시 유의사항
- 합격 기준은 정량 (시간/비율/허용 오차)
- 모든 Fail 은 결함 ID 와 연결
- 인계 결정은 잔여 리스크 + 결함 폐쇄 상황 기반

## 잘못된 작성 사례
> "잘 동작함" — 측정값 없음 → 시간·비율 명시 필수
> Fail 인데 결함 ID 미등록 → SUP.9 등록 강제
