# 측정 분석 보고서 작성예시 (EX-CMMI-04-02-02-02)

## 1. 보고서 정보 (샘플)
| 항목 | 내용 |
|---|---|
| 보고서 ID | MAR-NEXT3-2026Q1 |
| 보고 기간 | 2026 Q1 |
| 작성자·일자 | 정OO (MA Lead) / 2026-04-08 |

| 승인자 | 정OO (MA Lead — 본인) |
| 데이터셋 ID | DS-NEXT3-2026Q1 |

## 2. 분석 방법 (샘플)
| 방법 | 적용 측정 | 도구 |
|---|---|---|
| 기술통계 | 결함 밀도, LOC | Python pandas |
| 추세분석(이동평균) | 결함 등록 수 | Python statsmodels |
| 비교(목표-실적) | 일정 준수율 | Excel |

## 3. 분석 결과 (샘플)
| 측정명 | 결과 요약 | 목표 | 부합 |
|---|---|---|---|
| 결함 밀도 (Q1) | 0.84/KLOC (3개월 평균) | ≤1.0 | ✅ |
| 일정 준수율 (Q1) | 92% (5개 릴리즈 중 4개 정시) | ≥95% | ✗ |
| 결함 추세 | 1월 0.92 → 3월 0.78 (감소 추세) | 감소 | ✅ |

## 4. 컨텍스트·해석 (샘플)
| 항목 | 내용 |
|---|---|
| 프로젝트·조직 컨텍스트 | NEXT v3.0 MVP 출시 1분기. 신규 인력 3명 합류 |
| 핵심 발견 | (1) 결함 밀도 목표 달성, (2) 1개 릴리즈 일정 미준수 — 외부 인증 지연이 원인, (3) 결함 감소는 자동 테스트 커버리지 향상과 상관 |
| 한계·가정 | LOC 측정 시 생성 코드 일부 포함됨(자동 생성 30%). 결함 분류 일관성 검증 미실시 |

## 5. 의사결정 권고 (샘플)
| # | 권고 | 대상 |
|---|---|---|
| 1 | 외부 인증 의존 일정 버퍼 +10% 반영 | PM |
| 2 | 자동 테스트 커버리지 80% → 90% 상향 목표 | QA Lead |
| 3 | 자동 생성 코드를 LOC 산정에서 분리 | MA Lead |

## 6. 결재 (샘플)
| 검토 | 승인 | 일자 |
|---|---|---|
| 이OO (QA Director) | 정OO (MA Lead) | 2026-04-10 |

## 작성 시 유의사항
- 결과·해석·권고는 명확히 분리.
- 한계·가정은 의사결정자에 투명 공개.
- 권고는 책임자(대상)와 함께 명시.

## 잘못된 작성 사례
> ❌ 결과를 가설에 맞추어 해석
> ✅ 일관 시 결과 그대로 보고 + 별도 해석 섹션
