# SW 유닛 시험 계획서 및 결과 보고서 작성예시 (EX-SPICE-01-02-04-01)

> 원본 양식: [[TMP-SPICE-01-02-04-01_유닛시험계획서및결과보고서]]

## 작성 정보 (예시)
| 항목 | 내용 |
|---|---|
| 모듈 | aeb_decision |
| ASIL | D |
| 작성자 | 한소프 |

## 1. 시험 환경 (예시)
- 프레임워크: VectorCAST 2025
- Stub: ObjectList Mock (테스트용 데이터 생성기)
- CI: GitLab CI + Jenkins, 빌드당 자동 실행

## 2. 시험 케이스 (예시 발췌)
| TC | 함수 | 시나리오 | 입력 | 기대 |
|---|---|---|---|---|
| TC-AEB-001 | aeb_decide | 정상: 정지 차량 50m | obj=stationary, dist=50m, speed=80km/h | BrakeReq.valid=1, decel=0.6g |
| TC-AEB-002 | aeb_decide | 경계: 100m 한계 | dist=100m | BrakeReq.valid=0 |
| TC-AEB-003 | aeb_decide | 예외: NULL 입력 | obj=NULL | BrakeReq.invalid 반환 |
| TC-AEB-004 | aeb_calc_decel | 경계: ttc=0 | ttc_s=0 | decel=0.6g |

## 3. 실행 결과 (예시)
| TC | Pass/Fail | 측정 | 결함 ID |
|---|---|---|---|
| TC-AEB-001 | Pass | decel=0.6g | - |
| TC-AEB-002 | Pass | valid=0 | - |
| TC-AEB-003 | Pass | invalid 반환 | - |
| TC-AEB-004 | Pass | decel=0.6g | - |

## 4. 커버리지 (예시 ASIL D)
| 메트릭 | 목표 | 실측 | P/F |
|---|---|---|---|
| Statement | 100% | 100% | Pass |
| Branch | 100% | 100% | Pass |
| MC/DC | 100% | 100% | Pass |

## 5. Dead Code (예시)
| 위치 | 유형 | 사유 | 승인 |
|---|---|---|---|
| aeb_decision.c:142 (else if HW timer fail) | Defensive | HW 타이머 실패 시 안전 분기, 정상 시 도달 불가 | 이엠엘 (Safety) |

## 작성 시 유의사항
- 정상·경계·예외 시나리오 모두 포함
- ASIL D 는 MC/DC 100% 필수
- Dead Code 는 Safety Engineer 승인 필수

## 잘못된 작성 사례
> 경계 시나리오 누락 → 회귀 결함 위험
> Dead Code Justification 없음 → 감사 부적합
