## 작성 시 유의사항
- 모든 측정값은 단위·반복 횟수 명시 (예: "19.8m (3회 평균)")
- Pass/Fail 판정은 Spec 의 정량 기준에 1:1 매핑 — 주관 코멘트와 분리
- Skip 시나리오는 사유 + 재시험 일정 필수
- 편차 발견 시 NCR ID 즉시 부여 (SUP.9 시스템 자동 채번)
- 시험 데이터는 immutable storage(WORM/S3 Object Lock) 에 보관 — 변조 방지

## 잘못된 작성 사례
> "AEB 잘 작동함" — 정량 측정값 없음. → "정지 거리 19.8m (기준 22m 이하)" 로 정량화.
> Pass/Fail 컬럼만 채우고 증적 파일 경로 누락 — 감사 시 재현 불가. → 시나리오별 데이터 파일 경로 필수.
> Fail 항목을 Open Issue 로만 처리 — NCR 미발행. → 모든 Fail 은 SUP.9 NCR 발행 강제.
