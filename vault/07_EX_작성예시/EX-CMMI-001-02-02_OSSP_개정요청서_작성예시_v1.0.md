# OSSP 개정요청서 작성예시 (EX-CMMI-001-02-02)

> 원본 양식: [[TMP-CMMI-001-02-02_OSSP_개정요청서_v1.0]]

> 이 파일은 **교육·가이드용 샘플** 입니다. 실제 기록으로 사용하지 마세요. 가상 프로젝트 "디지털콜 v3" 기준.

## 1. 헤더
| 항목 | 내용 |
|---|---|
| 요청번호 | OSSP-CR-2026-014 |
| 요청일 | 2026-04-15 |
| 요청자 | 김SEPG (SEPG 위원) |
| 대상 자산 | PRO-CMMI-301 §5.4 + WI-CMMI-003-01-05 |
| 현재 버전 | v1.0 |

## 2. 변경 내역 요청
| 구분 | 현재 | 변경(안) | 사유 |
|---|---|---|---|
| 추적성 점검 주기 | 월 1회 | 격주 1회 | 디지털콜 v3 요구변경률 12% (목표 8% 초과) |
| 추적성 도구 | Excel | JIRA Premium 추적기 + Excel 백업 | 자동 갱신·이력 관리 강화 |

## 3. 영향 분석
| 영향 영역 | 영향 정도 | 후속 조치 |
|---|---|---|
| 인접 PRO (301/304) | Med | PRO 본문 §5.4 동기 갱신 필요 |
| 진행 프로젝트 (3건) | Low | 차기 점검부터 격주 적용 (소급 없음) |
| 교육/공지 | High | 전사 공지 + RDM 담당자 1H 워크숍 |

## 4. 검증 / 시범 적용
| 시범 대상 | 기간 | 결과 요약 |
|---|---|---|
| 디지털콜 v3 | 2026-02-01 ~ 03-31 (8주) | 요구변경률 12% → 7% 감소, 도구 만족도 4.2/5 |

## 5. 결재
| 작성 | 검토(SEPG) | 승인(PCB) |
|---|---|---|
| 김SEPG (2026-04-15) | 박SEPG Lead (2026-04-20) | PCB 의장 (2026-04-29) |

## 작성 요령
- "변경(안)" 은 차이 부분만 표기 (전체 재작성 금지).
- 시범 적용 데이터는 표본 10건 이상 권장.
- 영향 영역에 진행 프로젝트가 다수면 별표 첨부.

## 잘못된 작성 사례
- "필요해서 변경" — 정량 사유 부재.
- 시범 적용 칸 비워두고 "급한 변경" 으로만 표기 — PCB 반려 사유.
- 영향 분석에 모든 영역을 Low 로 표기 — 검토 회피로 의심.
