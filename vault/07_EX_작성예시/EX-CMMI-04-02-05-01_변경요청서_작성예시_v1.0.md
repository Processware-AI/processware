# 변경요청서 작성예시 (EX-CMMI-04-02-05-01)

## 1. 헤더
| 항목 | 내용 |
|---|---|
| CR ID | CR-DC-2026-014 |
| 요청자 | 김PO |
| 요청일 | 2026-08-10 |
| 영향 등급 | Major |

## 2. 변경 사항
- 현재: REQ-DC-0001 — 3개 채널 통합(전화·메신저·이메일)
- 변경 후: REQ-DC-0001 — 4개 채널 통합(추가: 카카오톡 비즈)
- 사유: 마케팅팀 7월 회의 결정, 고객 채널 점유율 35%

## 3. 영향 평가
| 영역 | 영향 | 비고 |
|---|---|---|
| 일정 | +2주 | 11월 인도 → 11월 말로 지연 |
| 비용 | +50,000천원 | API 라이선스 + 개발 |
| 품질 | 회귀 케이스 +12 | 기존 채널 영향 없음 |
| 보안 | 카카오톡 OAuth 인증 추가 | 보안팀 검토 필수 |
| 법규 | 영향 없음 | 정보통신망법 동일 적용 |

## 4. 대안
| 대안 | 장점 | 단점 |
|---|---|---|
| 차기 릴리스 | 일정 영향 없음 | 시장 점유 기회 손실 |
| 즉시 추가 | 점유율 확보 | 일정·비용 증가 |
| 외부 위탁 | 빠른 도입 | 통합 품질 리스크 |

## 5. CCB 결정
- 결정: 승인 (즉시 추가)
- 사유: 점유율 35%·DAR 결과 즉시 추가 우세, Sponsor 추가 예산 승인

## 6. 결재
| 요청자 | CCB 위원장 | Sponsor |
|---|---|---|
| 김PO (2026-08-10) | 정CCB위원장 (2026-08-15) | 이CTO (2026-08-18) |

## 작성 요령
- 영향 평가는 5개 영역 모두 작성. 영향 없음도 명시.
- 대안은 3개 이상 비교.
- Major 변경은 Sponsor 결재 필수.

## 잘못된 작성 사례
- 영향 평가 없이 승인 — CCB 의 본질 위반
- 대안 단일 — DAR 의미 상실
- 사유 모호 — 추후 책임 추적 불가
