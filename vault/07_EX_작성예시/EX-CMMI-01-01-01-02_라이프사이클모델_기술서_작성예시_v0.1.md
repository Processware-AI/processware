# 라이프사이클 모델 기술서 작성예시 (EX-CMMI-01-01-01-02)

> 원본 양식: [[TMP-CMMI-01-01-01-02_라이프사이클모델_기술서]]

## 샘플 컨텍스트
"**알파소프트(주)**"가 채택한 라이프사이클 모델 3종 예시.

## 1. 문서 정보 (샘플)
| 항목 | 예시값 |
|---|---|
| 문서번호 | LCM-2026-001 |
| 버전 | 1.0 |
| 작성자 | 김OO (EPG Lead) |
| 작성일 | 2026-04-15 |

## 2. 채택 모델 (샘플)
| 모델명 | 적용 대상 프로젝트 유형 | 선정 근거 |
|---|---|---|
| Waterfall | 정형 SI(공공/금융) | 요구사항 변동 ≤ 20%, 산출물 결재 다수 |
| Iterative (RUP-lite) | 솔루션 제품 v1 → v2 | 신기능 분기별 릴리즈, 리스크 분산 |
| Agile/Scrum (2-week sprint) | 자사 SaaS 운영 | 사용자 피드백 빈도 ≥ 월 1회 |

## 3. 모델별 단계·게이트 (Waterfall 샘플)
| 모델 | 단계명 | 단계 목적 | 게이트 조건 | 주요 산출물 |
|---|---|---|---|---|
| Waterfall | 분석 | 요구사항 확정 | 고객 사인오프 + REQM 베이스라인 | 요구사항 명세서 |
| Waterfall | 설계 | 아키텍처·상세 설계 | 설계 리뷰 결함 0건 | TDP |
| Waterfall | 구현 | 코드 작성 | 단위 테스트 ≥ 80% | 소스코드 |
| Waterfall | 시험 | 통합·시스템 테스트 | 결함밀도 ≤ 0.5/KLOC | 시험 결과서 |
| Waterfall | 인도 | 운영 환경 이관 | UAT 통과 | 인수확인서 |

## 4. OSSP 매핑 (샘플)
| 라이프사이클 단계 | 매핑 OSSP 프로세스 |
|---|---|
| 분석 | RD, REQM |
| 설계 | TS |
| 구현 | TS, CM |
| 시험 | VER, VAL |
| 인도 | PI |

## 5. 모델 선택 가이드 (샘플)
| 프로젝트 특성 | 권장 모델 |
|---|---|
| 요구사항 변경 빈도 ≤ 분기 1회 + 외부 인도 | Waterfall |
| 신기능 분기 릴리즈 + 내부 솔루션 | Iterative |
| 일/주 단위 배포 + 사용자 직접 피드백 | Agile/Scrum |

## 작성 시 유의사항
- 모든 채택 모델에 대해 단계·게이트·산출물 3종 누락 없이 작성.
- 게이트 조건은 측정 가능해야 함 (예: 결함 0건 ○ vs "충분히 검토되어야 함" ✗).

## 잘못된 작성 사례
> ❌ "상황에 맞게 선택" — 선택 기준 부재
> ✅ "요구사항 변동 ≤ 20%, 외부 결재 ≥ 3회 → Waterfall" 처럼 정량 기준
