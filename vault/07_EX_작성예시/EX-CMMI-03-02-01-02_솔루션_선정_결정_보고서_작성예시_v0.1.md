## 2. 선정 결과 (샘플)
| 항목 | 내용 |
|---|---|
| 선정 대안 ID | ALT-D (Hybrid) |
| 대안 유형 | Reuse + Make |
| 가중 점수 | 7.85 |
| 2위 대안 격차 | +0.20 (vs ALT-A 7.65) |
| 조건부 선정 여부 | 조건부 — 사내 프레임워크 v3 의 5초 새로고침 PoC 결과 통과 시 본 선정 |

## 3. 선정 근거
- ALT-D 는 K-01(기능)·K-08(확장성)에서 최고점, K-05(일정)도 ALT-A 대비 +1점.
- ALT-A 와 격차는 작으나(0.20) **모든 핵심 기준에서 동등 이상**, K-05 일정 위험이 더 낮음.
- 민감도 분석에서 일정 가중치 20%로 변동 시에도 순위 유지(0.05 차이) → 안정적 선정.
- 사내 프레임워크 재사용은 OPEX(K-04)에서 결정적 우위(9점) — 라이선스 비용 절감.

## 작성 시 유의사항
- 2위 격차 0.5 미만이면 정성 분석 명시 의무.
- 조건부 선정은 PoC·검증 마감일 명시.
- DAR 적용 시 DAR 결재 ID 인용 — 둘은 분리 결재.

## 잘못된 작성 사례
> ❌ "가장 좋은 안" 한 줄로 근거 마무리
> ✅ 평가 인용 + 민감도 인용 + 정성 분석 3요소

> ❌ 조건부 선정인데 마감일 미정
> ✅ PoC 일자·책임자 명시 (마감일 모니터링 기준)
