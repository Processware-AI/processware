## 1. 재사용 전략 목표 (예시)

| 항목 | 내용 |
|---|---|
| 재사용 비전 | "검증된 자산의 재사용으로 신규 프로젝트 개발 기간 30% 단축, ASPICE PA 2.2 충족 가속화" |
| 재사용율 목표 (Year1) | 30% |
| 재사용율 목표 (Year2) | 40% |
| 재사용율 목표 (Year3) | 50% |
| 적용 범위 (자산 유형) | SW 라이브러리, 시험 케이스 묶음, 설계 템플릿(WI/TMP), 요구사항 카탈로그, ML 모델 |
| 적용 도메인 | Powertrain, Chassis, Body, Infotainment, ADAS (5개) |

## 2. 재사용 자산 분류 체계 (예시)

| 차원 | 분류 항목 |
|---|---|
| 도메인 | Powertrain / Chassis / Body / Infotainment / ADAS |
| 자산 유형 | SW Lib (BSW, AUTOSAR, Application) / Test Suite / Design Template / Requirement Catalog / ML Model |
| 성숙도 | Experimental (사내 PoC 단계) / Validated (1개 프로젝트 양산 검증) / Certified (3개 이상 프로젝트 양산 검증 + Safety/Security 인증) |
| 안전 등급 | QM / ASIL A / ASIL B / ASIL C / ASIL D |

## 3. 재사용 인프라 (예시)

| 자산 유형 | 저장소 | 검색 도구 | 접근 권한 그룹 |
|---|---|---|---|
| SW 라이브러리 | Artifactory `https://artifactory.abc-motors.com` | Artifactory Search API + 태그 | abc-dev (Read), abc-arch (Write), abc-cm (Tag) |
| 시험 케이스 | Polarion Test Module | Polarion Query | abc-qa (Read/Write) |
| 설계 템플릿 / 문서 | Confluence `https://confluence.abc-motors.com/reuse` | Confluence Search + Label | abc-all (Read), abc-pqo (Write) |
| ML 모델 | ABC ML Registry (MLflow 기반) | MLflow UI + Tag | abc-ml (Read), abc-ml-admin (Write) |

## 4. 거버넌스 (예시)

| 항목 | 내용 |
|---|---|
| 자산 심의위원회 구성 | SW Architect 1명 (의장) + Tech Lead 2명 + QA 1명 + Safety Engineer 1명 + Security Engineer 1명 (총 6명) |
| 인증 기준 (요약) | 테스트 커버리지 ≥ 80%, 문서 완전성 100%, ASPICE 추적성 BP 충족, 안전/보안 등급 명시 |
| 정기 리뷰 주기 | 분기 1회 (3월·6월·9월·12월 둘째 주 화요일) |
| 폐기·갱신 결정 절차 | 12개월 미사용 자산은 폐기 후보 검토. 상위 호환 신규 자산 등장 시 기존 자산 deprecated 표시 후 6개월 후 폐기. |

## 5. 교육·훈련 계획 (예시)

| 대상 | 모듈 | 시기 / 주기 | 책임 |
|---|---|---|---|
| 신규 입사자 (전 엔지니어) | "재사용 카탈로그 사용 가이드 (90분)" | 입사 1주차 온보딩 | HR + PQO |
| 기존 엔지니어 | "분기 재사용 우수 사례 발표회 (2시간)" | 분기 1회 | PQO |
| Tech Lead / SW Architect | "재사용 자산 등록 워크숍 (4시간)" | 반기 1회 | SW Architecture Team |

## 6. 측정 지표 (예시)

| 지표 | 측정 방법 | 보고 주기 | 책임자 |
|---|---|---|---|
| 재사용율 (전사) | (재사용 컴포넌트 수) / (신규 프로젝트 컴포넌트 총수) — Polarion + Artifactory 자동 집계 | 분기 | 박개선 (PQO) |
| 자산별 활용 횟수 | Artifactory download count + Polarion test case copy count | 분기 | 강자산 (Reuse Asset Owner) |
| 평균 검색→활용 시간 | 사용자 설문 + Confluence 분석 | 반기 | 박개선 (PQO) |
| 재사용 자산 결함률 | 재사용 자산 기인 결함 / 총 결함 | 반기 | 최품질 (QA) |

## 7. 리스크 · 예외 정책 (예시)

| 리스크 항목 | 대응 정책 |
|---|---|
| 라이선스 충돌 (GPL/AGPL 포함 자산) | 등록 단계에서 라이선스 화이트리스트(MIT/BSD/Apache-2.0/사내 라이선스) 자동 검사 + 위반 시 등록 거부 |
| 안전 등급 불일치 (ASIL QM 자산을 ASIL D 시스템에 재사용) | 안전 분석(SCC, Safety Element out of Context) 의무화. 미수행 시 사용 금지 |
| 도메인 격차 (Infotainment 라이브러리를 ADAS 에 재사용) | 분류 체계의 도메인 차원 검색 시 경고 표시 + 도메인별 Tech Lead 사전 승인 필수 |
| 자산 책임자 부재 (퇴사·이동) | 분기 리뷰 시 책임자 현황 점검 + 부재 자산은 임시 PQO 가 인계받고 30일 내 후임 지정 |

## 작성 시 유의사항

- 재사용율 목표는 측정 가능한 분자/분모 정의를 함께 명시한다.
- 분류 체계는 검색·필터링이 가능하도록 메타데이터 필드와 1:1 매핑한다.
- 인증 기준은 [[WI-ASPICE-01-11-04_재사용자산등록및인증]] 의 품질 게이트와 일치시킨다.

## 잘못된 작성 사례

> "재사용 활성화" — 정량 목표 없음.
> "Tech Lead 가 알아서 결정" — 거버넌스 미정의. 책임 모호.
