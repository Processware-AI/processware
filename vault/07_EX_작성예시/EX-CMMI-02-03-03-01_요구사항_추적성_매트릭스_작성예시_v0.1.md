## 작성 시 유의사항
- link 강도: Strong = 산출물에 명시 + 시험 검증 / Weak = 산출물에만 명시, 시험 미실행 또는 운영문서 미작성.
- 마일스톤 audit 전 Weak 항목 0건이 권장.

## 잘못된 작성 사례
> ❌ link 를 "X" "O" 만으로 표시 — 강도 구분 없음
> ✅ Strong/Weak 으로 audit 품질 차별화

> ❌ 변경 이력 누락 → 산출물 변경 시 RTM 동기 안 됨
> ✅ 변경 이력 timestamped 기록 + 책임자 명시
