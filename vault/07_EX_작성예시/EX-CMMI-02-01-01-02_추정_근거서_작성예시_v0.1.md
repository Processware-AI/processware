# 추정 근거서 작성예시 (EX-CMMI-02-01-01-02)

> 원본 양식: [[TMP-CMMI-02-01-01-02_추정_근거서]]

## 샘플 컨텍스트
가상 프로젝트 "**모바일 헬스케어 앱 v2.0**" 의 추정 근거서.

## 1. 문서 정보 (샘플)
| 항목 | 예시값 |
|---|---|
| 프로젝트명 | 모바일 헬스케어 앱 v2.0 |
| 문서번호 | EST-2026-HCA-001 |
| 버전 | 1.0 |
| 작성자 | 김OO (PM) |
| 작성일 | 2026-05-08 |

## 2. 라이프사이클 단계 정의 (샘플)
| 단계 | 게이트 조건 | 주요 산출물 |
|---|---|---|
| 분석 | RD 검토 통과 | 요구사항 명세서 |
| 설계 | 아키텍처 review 통과 | 설계서, DB 스키마 |
| 구현 | 단위시험 통과율 ≥ 95% | 소스, UT 결과 |
| 시험 | VAL/VER 통과 | 시험 보고서 |
| 출시 | App Store 승인 | 출시 노트 |

## 3. 추정 기법 (샘플)
| 항목 | 내용 |
|---|---|
| 사용 기법 | Wide-band Delphi + Analogy (PAL 비교) |
| 선택 사유 | 신기능(HealthKit)은 미경험 → Delphi; 나머지는 유사 프로젝트 보유 |
| 도구 | Atlassian Jira Story Point + Excel 추정표 |

## 4. 노력·비용 추정 (샘플)
| 분류 | 값 | 단위 |
|---|---|---|
| 총 effort | 12.0 | person-month |
| 단가 | 8,000,000 | 원/PM |
| 직접비 | 15,000,000 | 원 (Apple Developer, 클라우드 등) |
| 총 비용 | 111,000,000 | 원 |

## 5. 추정 가정·근거 데이터 (샘플)
| 항목 | 내용 |
|---|---|
| 핵심 가정 | 1) 풀타임 개발자 3명 평균 가용성 90%, 2) HealthKit API 안정 (Apple side change 0), 3) 외부 의존성 (피트니스 SDK) 라이선스 4월 확정 |
| 사용 PAL 측정값 | 2024-HCM-001 (10.5 PM, 유사 도메인), 2025-FIT-002 (9.8 PM) |
| 외부 벤치마크 | ISBSG benchmark — 헬스케어 모바일 평균 11.3 PM |

## 6. PAL 편차 분석 (샘플)

## 작성 시 유의사항
- PAL 측정값이 2건 이상일 것 권장 (1건만으로는 단일 케이스 편향 위험).
- 외부 벤치마크 사용 시 출처·접근일자·라이선스 명시 (license 미확인 데이터 사용 금지).
- 핵심 가정은 RSKM SP2.1 입력으로 등록.

## 잘못된 작성 사례
> ❌ 추정 기법 "PM 직관" 단독 — 정량 근거 0
> ✅ 정량 기법 + 정성 보완 조합

> ❌ PAL 편차 분석 누락 — 후행 일정·예산 변경 시 책임 추적 불가
> ✅ 편차율과 처리 분기 명시 필수
