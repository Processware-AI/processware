# 팀 헌장 및 공유 비전 작성예시

> 원본: [[TMP-CMMI-02-05-03-01_팀_헌장_공유_비전]]

## 1. 팀 정보 (샘플)
| 항목 | 예시값 |
|---|---|
| 프로젝트명 | 모바일 헬스케어 앱 v2.0 |
| 팀 명 | HCA-v2 Squad |
| 워크숍 일자 | 2026-05-30 |

## 2. 팀 비전 (샘플)
|  |  |
|---|---|
| 비전 | "10만 사용자에게 정확하고 안전한 헬스 데이터 인사이트를 제공하여, 일상의 건강 관리를 쉽게 만든다." |

## 3. 역할·책임 (샘플)
| 역할 | 담당자 | 책임 |
|---|---|---|
| PM | 김OO | 전체 계획·PMC·이해관계자 협업 |
| Tech Lead | 정OO | 아키텍처·코드 품질·기술 결정 |
| iOS Lead Dev | 박OO | iOS 구현·앱 스토어 |
| Android Dev | 한OO | Android 구현 |
| QA Lead | 조OO | 시험·품질 |
| Designer | 박OO | UI/UX (A스튜디오 협업) |

## 4. 운영 규칙 (샘플)
| 영역 | 규칙 |
|---|---|
| 정기 회의 | Daily standup 09:30 (15분), Weekly retro 금요 16:00 |
| 의사소통 | 비공식: Slack #hca-v2; 의사결정: Confluence 결정 로그 |
| 의사결정 | 합의 우선, 미합의 시 PM 결정. 비용 영향 ≥ 5% 는 DAR 적용 |
| 갈등 해결 | 1:1 → 팀회의 → PMO 중재 (3단계) |
| 회고·개선 | 격주 회고 — Try/Continue/Stop 형식 |

## 작성 시 유의사항
- 비전은 1~3 문장 — 너무 길면 기억 안 됨, 너무 짧으면 의미 부족.
- 운영 규칙은 측정 가능 (시간·빈도 명시).

## 잘못된 작성 사례
> ❌ "최선을 다한다" 식의 추상 비전
> ✅ 측정 가능한 결과 + 가치 명시
