## 3. 요구사항 평가표 (샘플 — 8건 중 일부)
| 요구사항 ID | 설명 | 완전성 | 검증성 | 일관성 | 실현가능성 | 추적성 | 총평 |
|---|---|---|---|---|---|---|---|
| REQ-001 | HealthKit 데이터 동기화 | Pass | Pass | Pass | Pass | Pass | **Pass** |

| REQ-003 | 사용자 ≥ 10만명 지원 | Pass | Pass | Pass | Pass | Pass | **Pass** |
| REQ-004 | 응답 시간 ≤ 200ms | Pass | Pass | Pass | Pass | Pass | **Pass** |
| REQ-005 | "충분히 빠른" 동기화 | **Fail** | **Fail** | Pass | Pass | **Fail** | **Modify** |
| REQ-006 | GDPR/개보법 준수 | Pass | Pass | Pass | Pass | Pass | **Pass** |
| REQ-007 | 푸시 알림 (어쩌면 추후) | **Fail** (어쩌면) | Pass | Pass | Pass | Pass | **Modify** |

## 4. 모호어/이상 항목 (샘플)

## 작성 시 유의사항
- 5 기준 모두 Pass 인 경우만 Pass — 1건이라도 Fail 이면 Modify 또는 Reject.
- 모호어 사전 (조직 표준) 활용 — "친화적/충분한/적절한/빠른" 등은 자동 Fail.

## 잘못된 작성 사례
> ❌ 모든 요구사항 일괄 Pass 처리 (사실상 audit 안 함)
> ✅ 기준별 평가 — Modify/Reject 발견이 정상

> ❌ "어쩌면" 요구사항 (REQ-007) 을 Pass 로 처리
> ✅ 미확정 요구사항은 본 베이스라인 제외, backlog 로 분리
