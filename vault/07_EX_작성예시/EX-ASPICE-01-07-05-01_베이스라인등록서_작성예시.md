## 작성 요령
- **체크섬은 동결 직후 산출**: 동결 전과 후 두 번 측정하여 일치 검증. 체크섬은 베이스라인 무결성의 단일 진실 원천.
- **태그명에 마일스톤 명시**: BL-SWE2-v1.0 처럼 어떤 마일스톤인지 즉시 식별 가능해야 함.
- **이전 BL 이후 변경 이력 필수**: BL 비교 시 어떤 CR 이 적용되었는지 추적 가능해야 함. 첫 BL 은 "초기 베이스라인" 으로 명기.
- **CM 도구 자동 immutable**: 가능하면 Artifactory immutable tag, Git protected tag 정책으로 무단 수정 차단.

## 잘못된 사례
- 체크섬 누락 → 무결성 검증 불가, 무단 수정 탐지 불가.
- "Git tag 만 달면 끝" → Confluence 문서·Artifactory 패키지 등 다른 저장소 동결 누락.
- 변경 이력에 CR ID 없이 "여러 변경 적용" 만 기재 → 추적성 단절.
- 무결성 검사 결과 "Pass" 만 표기, 검사 방법·도구 미기재 → 재현 불가.
