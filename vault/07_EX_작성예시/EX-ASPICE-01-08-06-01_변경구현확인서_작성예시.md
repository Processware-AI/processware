## 작성 요령
- **변경 파일은 라인 단위로**: 단순 파일명만 적지 말고 변경 라인 번호까지 명시 (감사 시 정확한 추적).
- **머지 커밋 ID 필수**: Git 커밋 해시는 영구 추적 키. 브랜치명만으로는 사후 검색 어려움.
- **재검증 시험은 전수 기재**: 조건부 승인의 조건 시험은 모두 표에 포함해야 충족 증거가 됨.
- **외부 통보 대상 명기**: 외부 인터페이스 변경 시 OEM·협력사 통보 누락 방지.

## 잘못된 사례
- 변경 파일을 "통신 모듈 일부" 같이 모호하게 기재 — 추적 불가.
- 재검증 결과 "전부 Pass" 만 적고 시험 케이스 ID 누락 — 어떤 시험을 했는지 검증 불가.
- 추적성 갱신 검증자 없음 — RTM 단절 여부 확신 불가.
- Conditional 조건 충족 증거 없이 종결 처리 — 절차 위반, ASPICE 평가 시 미흡.
