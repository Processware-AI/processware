## 작성 시 유의사항
- CR ID 는 반드시 CR Tool 자동 생성 — 수동 부여 금지 (충돌 위험).
- As-Is / To-Be 는 코드 라인·문서 §·필드명 단위로 명확히 — "전반적으로 개선" 금지.
- 영향 받을 베이스라인은 1 건이 아닌 경우가 일반적 — 코드뿐 아니라 테스트·설계 문서·인터페이스도 포함.
- 우선순위 Critical 부여 시 **반드시 안전성/규제/양산 중 1개 이상 Y** 표기 필요.
- OEM 발 CR 은 OEM 공문 ID 첨부 필수.

## 잘못된 작성 사례
> CR ID "CR-001" — 연도 누락, 충돌 위험. → `CR-2026-0089`
> As-Is "코드가 잘못됨", To-Be "코드 수정" — 변경 범위 불명확. → 라인·필드 단위 기술.
> 우선순위 Critical인데 안전성/규제/양산 모두 N — 판정 기준 위반. → 판정 근거 재검토.
