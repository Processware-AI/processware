## 2. 확인 대상 (샘플)
| 대상ID | 단계 | 방법 | 우선순위 | 사용자 그룹 |
|---|---|---|---|---|
| VT-01 | 설치 | 시연 | Must | DevOps 2 |
| VT-02 | 운영 | 사용자 시험 + 운영 시범 | Must | 라인장 6, QA 2 |

| VT-05 | 교육 (자체 학습) | 프로토타입 + 사용자 시험 | Should | 신규 사용자 5 |

## 3. 확인 환경 (샘플)
| 항목 | 내용 |
|---|---|
| 환경 유형 | **혼합** — 사내 시뮬레이션 라인 1 + 고객 측 파일럿 라인 1 |
| HW | 라인 PC 6대, 모바일 단말 4대 |
| SW | RC2 Build + 패치 적용 |
| 사용자 단말 | Win11 Chrome, iOS Safari, Android Chrome |
| 통신 | 고객 VPN |
| 데이터 | 운영 1주차 실 데이터 (익명화) + 합성 데이터 |
| 접근성 | WCAG 2.2 AA 자동 점검 통과 |

## 4. 절차·기준 (샘플 일부)
| VP-ID | 사용자 행동 | 기대 결과 | 합격 기준 | 척도 |
|---|---|---|---|---|
| VP-01 | 라인장이 작업지시 발행 | 라인 표시 + 라인장 동의 | 5초 내 표시 + 의도 충족 ≥ 4점 | 5점 |
| VP-02 | QA 가 품질 알림 시정 | 시정 완료 메시지 | 자동 시정 등록 + 4점 | 5점 |
| VP-03 | 유지보수 백업 복원 | 30분 내 복원 | 30분 이하 + 데이터 무결성 | 분/Pass-Fail |
| VP-04 | 신규 사용자 1시간 자체 학습 | 핵심 기능 수행 | 핵심 5개 기능 모두 수행 | Pass/Fail |
| VP-05 | NPS 설문 | NPS ≥ 40 | 평균 ≥ 40 | NPS |

## 작성 시 유의사항
- 사용자 ≥ 3명 — 표본 너무 작으면 §7.2 분기.
- 합격 기준은 정량 — 정성 표현 시 5점·NPS 등으로 변환.
- 라이프사이클 단계 폐기까지 포함 (자주 누락).

## 잘못된 작성 사례
> ❌ "고객이 만족하면 통과"
> ✅ "평균 4점 이상 + 핵심 기능 100% 수행"

> ❌ 운영 환경 가정 없이 사내 환경만 사용
> ✅ 사내 시뮬레이션 + 고객 파일럿 혼합 명시
