# 전사 표준 프로세스(Enterprise Standard Process) 체계 구축 방안

## 1. 체계 설계 기본 원칙

| 원칙 | 설명 |
|---|---|
| **표준 정합성** | ISO/IEC·KS·법규 요구사항을 단일 체계로 통합(IMS, Integrated Management System) |
| **계층적 구조화** | POL → PRO → WI → {TMP, EX, REC} + MAT/REF 의 8종 문서 유형 체계 ([[01_문서체계]]) |
| **프로세스 접근** | PDCA 기반 상호작용·책임자(RACI)·성과지표(KPI) 명확화 |
| **테일러링 가능** | 본사·사업부·프로젝트 단위로 축소/확장(Tailoring) 가능 |
| **디지털화** | BPM/EDMS/GRC 툴로 형상관리·추적성·감사증적 확보 |

## 2. 준거 표준 매핑 (예시)

| 영역 | 국제 표준 | 국내 표준/법규 |
|---|---|---|
| 경영시스템 통합 | ISO/IEC Directives Annex SL(HLS 공통구조) | KS Q ISO 공통 |
| 품질 | ISO 9001 | KS Q 9001 |
| 환경·안전보건 | ISO 14001, ISO 45001 | 산업안전보건법, 중대재해처벌법 |
| 정보보호·개인정보 | ISO/IEC 27001, 27701, 27017/27018 | ISMS-P, 개인정보보호법 |
| IT 서비스 | ISO/IEC 20000 | TTA/공공 ITSM 지침 |
| SW 개발·공급 | ISO/IEC/IEEE 12207, 15288, 29110 | SW사업대가기준, 공공SW사업관리 |
| 리스크·BCM | ISO 31000, ISO 22301 | 금감원 전자금융감독규정 등 |
| AI·데이터 | ISO/IEC 42001, 23894, 5259 | AI 신뢰성 가이드라인 |
| **자동차 (4종)** | **IATF 16949, Automotive SPICE, ISO 26262, ISO/SAE 21434** | KS R ISO (일부) · UN R155/R156 |
| **의료기기 (4종)** | **ISO 13485, ISO 14971, IEC 62304, IEC 81001-5-1** | 의료기기법, 식약처 가이드, 디지털의료제품법 |

→ 표준 분류(Layer·Integration Mode) 는 [[07_표준분류레지스트리]] 에서 단일 출처로 관리.

→ **공통 HLS(High-Level Structure)** 를 축으로 통합경영매뉴얼을 설계하고, 표준별 요구사항을 모듈로 부착.

## 3. 프로세스 아키텍처 (3-Tier Process Map)

```
┌─────────────────────────────────────────────────────────┐
│  [M] 경영 프로세스 (Management)                          │
│   · 전략/목표 · 리스크 · 감사 · 성과관리 · 경영검토      │
├─────────────────────────────────────────────────────────┤
│  [C] 핵심 프로세스 (Core / Value Chain)                  │
│   · 영업·제안 · R&D·설계 · 구매·생산 · 납품·서비스       │
├─────────────────────────────────────────────────────────┤
│  [S] 지원 프로세스 (Support)                             │
│   · HR · 재무 · IT/보안 · 법무/컴플라이언스 · 자산관리   │
└─────────────────────────────────────────────────────────┘
```

각 프로세스를 **Level 1(프로세스 영역) → Level 2(프로세스) → Level 3(액티비티) → Level 4(작업절차/WI)** 로 분해.

## 4. 문서 유형 체계 (8종, 상세: [[01_문서체계]])

| 코드 | 유형 | 역할 | 폴더 |
|---|---|---|---|
| **POL** | 정책서 | 원칙·방침·책임 정의 | `03_POL_정책/` |
| **PRO** | 절차서 | 업무 흐름·관리 절차 | `04_PRO_절차/` |
| **WI**  | 업무지침서 | 실무 수행 상세 | `05_WI_업무지침/` |
| **TMP** | 템플릿 | 기록용 빈 양식 | `06_TMP_템플릿/` |
| **EX**  | 작성예시 | 템플릿 작성 샘플 | `07_EX_작성예시/` |
| **REC** | 기록본 | 실제 수행 증빙 | `08_REC_기록/` |
| **MAT** | 매핑/관리대장 | 추적성·목록·대조표 | `90_MAT_통합매핑/` |
| **REF** | 참고자료 | 외부 규정·가이드 요약 | `09_REF_참고자료/` |

**상하관계**: `POL → PRO → WI → {TMP, EX, REC}` (MAT/REF 은 별도 축)

**파일명 규칙**: `[유형]-[식별번호]_[문서명]_v[버전].md` (상세: [[02_문서번호체계]])

**엄격 분리 원칙**
1. 템플릿(TMP)과 기록(REC) 같은 폴더 금지
2. 템플릿(TMP)에 샘플 데이터 기입 금지 → 작성예시는 EX 로 분리
3. 정책(POL)에 세부 절차 혼입 금지 — 흐름은 PRO, 실무는 WI

**입력자료 원칙** (상세: [[05_입력자료_규칙]])
4. 모든 산출물은 `_inputs/` 에 근거를 두고 생성, `source_citation` 필수
5. 라이선스 없는 표준 원문 직접 인용 금지 — paraphrase 만
6. 고객사 As-Is 자산은 용어·구조 존중, 타 고객으로 유출 금지

## 5. 구축 로드맵 (5단계)

**Phase 1. 진단 (Assess)**
- 표준/법규 요구사항 도출 → 현행 프로세스 As-Is 분석 → Gap 분석

**Phase 2. 설계 (Design)**
- 프로세스 아키텍처·문서체계·거버넌스·RACI·KPI 정의
- 표준-프로세스 매트릭스 수립(추적성 매트릭스)

**Phase 3. 개발 (Develop)**
- 8종 유형(POL/PRO/WI/TMP/EX/REC/MAT/REF) 별 산출물 문서화, 테일러링 가이드 작성

**Phase 4. 이행 (Deploy)**
- 파일럿 사업부 적용 → 교육/내재화 → 전사 확산
- BPM·EDMS·GRC 시스템 연계

**Phase 5. 운영·개선 (Operate & Improve)**
- 내부심사·경영검토·시정조치(CAPA), 성숙도 평가(CMMI/ISO 33000) 기반 지속 개선

---

## 6. 거버넌스 구조

- **PCB (Process Control Board)**: 프로세스 표준화 최고 의사결정체
- **Process Owner**: 프로세스별 책임자(L2 단위)
- **SEPG/QA**: 프로세스 엔지니어링·품질보증 조직
- **내부심사팀**: ISO 심사원 자격 보유, 연 1회 이상 내부심사
- **개정 관리**: 변경요청(RFC) → 영향평가 → PCB 승인 → 버전관리

## 7. 성과관리 지표 (예시 KPI)

| 관점 | 지표 |
|---|---|
| 준수성 | 표준 요구사항 커버리지, 부적합 건수, 시정조치 완료율 |
| 효율성 | 프로세스 리드타임, 자동화율, 재작업률 |
| 성숙도 | CMMI/ISO 33000 Capability Level, 내부심사 지적 감소율 |
| 가치 | 고객 클레임, 사고/리스크 발생 건수, 인증 획득/유지 |

---

## 8. 성공 요인

1. **경영층 스폰서십**과 통합경영 방침 선언
2. **단일 소스(Single Source of Truth)** 문서 플랫폼(EDMS)
3. **현업 참여형** 프로세스 설계(Top-down + Bottom-up)
4. **테일러링 원칙** 명문화 — 과도한 형식주의 방지
5. **지속 교육·내재화**와 **객관적 증적** 확보
