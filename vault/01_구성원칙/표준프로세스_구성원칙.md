# 전사 표준 프로세스(Enterprise Standard Process) 체계 구축 방안

## 1. 체계 설계 기본 원칙

| 원칙 | 설명 |
|---|---|
| **표준 정합성** | ISO/IEC·KS·법규 요구사항을 단일 체계로 통합(IMS, Integrated Management System). 어떤 구조화 문서든 편입 가능(표준 무관). |
| **계층적 구조화** | POL → PRO → WI → {TMP, EX, REC} + MAT/REF 의 8종 문서 유형 체계 ([[01_문서체계]]) |
| **프로세스 접근** | PDCA 기반 상호작용·책임자(RACI)·성과지표(KPI) 명확화 |
| **테일러링 가능** | 본사·사업부·프로젝트 단위로 축소/확장(Tailoring) 가능 |
| **AI 자동화** | Plan→Do→Check→Act 4차원 전체를 AI 에이전트로 자동화. 사람은 HITL(핵심 결정)에만 개입. |
| **단일 소스(SSoT)** | vault 가 모든 표준 프로세스 자산의 유일한 원본. 외부 시스템(ALM·PMS·QMS 등)은 vault를 단방향으로 구독. |
| **감사 추적성** | 모든 실행 단계에 `source_citation` + trace 기록. 기계가 읽을 수 있는 증적(REC)으로 감사 대응. |

## 3. 프로세스 아키텍처 (3-Tier Process Map)

```
┌─────────────────────────────────────────────────────────┐
│  [M] 경영 프로세스 (Management)                          │
│   · 전략/목표 · 리스크 · 감사 · 성과관리 · 경영검토      │
├─────────────────────────────────────────────────────────┤
│  [C] 핵심 프로세스 (Core / Value Chain)                  │
│   · 영업·제안 · R&D·설계 · 구매·생산 · 납품·서비스       │
├─────────────────────────────────────────────────────────┤
│  [S] 지원 프로세스 (Support)                             │
│   · HR · 재무 · IT/보안 · 법무/컴플라이언스 · 자산관리   │
└─────────────────────────────────────────────────────────┘
```

각 프로세스를 **Level 1(프로세스 영역) → Level 2(프로세스) → Level 3(액티비티) → Level 4(작업절차/WI)** 로 분해.

## 4. 문서 유형 체계 (8종, 상세: [[01_문서체계]])

| 코드 | 유형 | 역할 | 폴더 |
|---|---|---|---|
| **POL** | 정책서 | 원칙·방침·책임 정의 | `03_POL_정책/` |
| **PRO** | 절차서 | 업무 흐름·관리 절차 | `04_PRO_절차/` |
| **WI**  | 업무지침서 | 실무 수행 상세 | `05_WI_업무지침/` |
| **TMP** | 템플릿 | 기록용 빈 양식 | `06_TMP_템플릿/` |
| **EX**  | 작성예시 | 템플릿 작성 샘플 | `07_EX_작성예시/` |
| **REC** | 기록본 | 실제 수행 증빙 | `08_REC_기록/` |
| **MAT** | 매핑/관리대장 | 추적성·목록·대조표 | `90_MAT_통합매핑/` |
| **REF** | 참고자료 | 외부 규정·가이드 요약 | `09_REF_참고자료/` |

**상하관계**: `POL → PRO → WI → {TMP, EX, REC}` (MAT/REF 은 별도 축)

**파일명 규칙**: `[유형]-[식별번호]_[문서명]_v[버전].md` (상세: [[02_문서번호체계]])

**엄격 분리 원칙**
1. 템플릿(TMP)과 기록(REC) 같은 폴더 금지
2. 템플릿(TMP)에 샘플 데이터 기입 금지 → 작성예시는 EX 로 분리
3. 정책(POL)에 세부 절차 혼입 금지 — 흐름은 PRO, 실무는 WI

**입력자료 원칙** (상세: [[05_입력자료_규칙]])
4. 모든 산출물은 `_inputs/` 에 근거를 두고 생성, `source_citation` 필수
5. 라이선스 없는 표준 원문 직접 인용 금지 — paraphrase 만
6. 고객사 As-Is 자산은 용어·구조 존중, 타 고객으로 유출 금지

## 5. 구축 로드맵 (4차원 PDCA AI 자동화 사이클)

> 이 플랫폼은 전통적 5단계(진단→설계→개발→이행→운영)를 **AI 에이전트 기반 4차원 폐쇄 루프**로 자동화한다.

**차원 0 (Ingest) — 표준 전처리**
- 표준 원문·법규·해설서를 편입 → 요건(REQ-NNN) 추출 → 업무 시나리오(business_flow) 도출
- 사람은 시나리오 확정(HITL)만 담당

**차원 1 (Plan) — 표준 수립**
- 요건 패키지 → POL/PRO/WI/TMP/EX/MAT 자동 생성 → 추적성(MAT-011~) 완결
- 자가수정 루프 내장 (QA Fail → 담당 에이전트 재실행, 최대 3회)

**차원 2 (Do) — 프로세스 실행**
- WI 실행 → 단계별 HITL 승인 → REC 기록 자동 생성 → MAT-005 실행기록 누적
- 자연어 입력으로 WI 자동 라우팅, SLA 초과 시 자동 에스컬레이션

**차원 3 (Check) — 심사·KPI·GAP**
- REC vs PRO/WI 요건 자동 대조 → REC-AUDIT + NCR 자동 발행
- KPI 회귀 탐지 → 임계 초과 시 차원 4 큐 자동 push
- 외부 표준 GAP 분석 → 인증기관 제출용 보고서 자동 생성

**차원 4 (Act) — 제·개정**
- NCR/KPI/GAP → RCA(5-Why/Fishbone) → 개정안 → PCB 승인 → 차원 1 재트리거
- 폐쇄 루프: Act 완료 → Plan' 자동 재실행 → 개정판(v1.1) 생성

**Phase 5 (외부 연동)**
- vault SSoT → 외부 ALM·PMS·QMS 시스템에 단방향 push (Headless CMS 모델)

## 6. 거버넌스 구조

**인간 역할**
- **PCB (Process Control Board)**: 프로세스 표준화 최고 의사결정체. 개정안·GAP 대응 방향 승인
- **Process Owner**: 프로세스별 책임자(L2 단위). HITL 확정·NCR 종결·차원 4 Act 실행
- **SEPG/QA**: 프로세스 엔지니어링·품질보증 조직. 심사 기획, KPI 설정, 에이전트 산출물 최종 검토
- **내부심사팀**: ISO §9.2 독립성 원칙 준수 (auditor ≠ executor). 차원 3 심사 실행
- **개정 관리**: 변경요청(RFC) → 영향평가 → PCB 승인(HITL) → 버전관리

**AI 에이전트 역할**
- **standard-analyzer**: 표준 요건 추출·정규화 (차원 0/1)
- **process-designer / wi-tmp-writer**: POL/PRO/WI/TMP/EX 초안 자동 생성 (차원 1)
- **process-executor / rec-writer**: WI 실행 → REC 자동 기록 (차원 2)
- **audit-planner ~ act-trigger**: 심사 전 과정 자동화 + 차원 4 큐 push (차원 3)
- **rca-analyzer ~ act-coordinator**: 근본원인 분석·개정안·PCB 게이트 관리 (차원 4)
- **gap-analyzer ~ gap-exporter**: 외부 표준 GAP 분석·보고서 생성 (차원 3 Audit)

> AI 에이전트는 **초안 생성·분류·매칭·분석**을 담당. 최종 승인·확정은 사람(HITL). 에이전트 산출물은 항상 검토 가능하도록 vault에 보존.

## 8. 성공 요인

1. **경영층 스폰서십**과 통합경영 방침 선언
2. **단일 소스(SSoT)** — vault 를 유일한 원본으로, 외부 시스템은 구독자(consumer)로
3. **현업 참여형** 프로세스 설계(Top-down + Bottom-up) + AI 초안 → 사람 확정
4. **테일러링 원칙** 명문화 — 과도한 형식주의 방지. 모든 표준은 inputs/ 기반 적용
5. **객관적 증적** 자동 확보 — REC + trace.jsonl + source_citation 기계 판독 가능
6. **AI 에이전트 신뢰 구축** — HITL 게이트·PCB 승인·RBAC 독립성 검증으로 AI 오류 봉쇄
7. **폐쇄 루프 지속 개선** — NCR→Act→Plan' 사이클이 표준 자산을 자동으로 갱신
