# 90_MAT_통합매핑 — 매핑·관리대장(MAT)

## 역할
문서 간 관계·관리 상태·요구사항 커버리지를 **추적하는 통합 관리 문서**. 심사 대응과 체계 운영의 핵심.

## MAT 번호 구역 (상세: [[02_문서번호체계]] §MAT 번호 할당 원칙)
- **001~010**: 전사 공통 (최대 10종, 재배정 금지)
- **011~099**: 표준별 추적성 (순차)
- **100~**: 예약

## 관리 담당
- **차원 1**: `traceability-mapper` — 표준 편입 시마다 공통 매트릭스(MAT-001·003·004·005·006) 최신화 + 해당 표준 추적성 신규 생성
- **차원 2** (자동): `process-router` (MAT-007 갱신) / `rec-writer` (MAT-005 §실행 기록) / `traceability-mapper --rebuild-catalog` (MAT-007 일괄 재구축)
- **차원 3** (자동): `audit-reporter` (MAT-005 §심사 이력) / `ncr-drafter` (MAT-009 발행/종결 두 섹션) / `kpi-analyzer` (MAT-008 시계열·회귀 + MAT-009 §통계)
- **차원 4** (자동): `act-coordinator` (MAT-001 §개정 이력) / `act-trigger` (MAT-008 §차원 4 인계)

## 특징
- 문서 자체보다 **관리성**이 목적
- 심사자의 진입점으로 설계
- Dataview 플러그인으로 자동 집계 가능 (MAT-001)

## 판단 기준
