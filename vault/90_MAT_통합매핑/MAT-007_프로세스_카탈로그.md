# MAT-007 프로세스 카탈로그

> **용도**: 사용자가 절차 번호를 모르고도 자연어로 차원 2 실행을 시작할 수 있도록, PRO/WI 의 트리거 키워드·역할·이벤트를 인덱싱한 통합 카탈로그.

> **갱신 정책**:
> - 자동: `traceability-mapper` 가 차원 1 빌드 종료 시 본 파일 갱신 (write phase 직후, MAT-001/003/004/006 갱신과 함께).
> - 수동: `/do --rebuild-catalog` 로 수동 트리거.
> - 사람이 수정한 행은 `manual_override: true` 표기 시 자동 갱신 시 보존.

> **사용 주체**: `process-router` 에이전트가 본 파일을 1회 LLM 컨텍스트로 로드 후 자연어 → WI 매칭.

---

## 1. 스키마

각 항목은 다음 필드를 가진다:

| 필드 | 의미 |
|---|---|
| `doc_id` | PRO 또는 WI 의 식별번호 |
| `kind` | `PRO` \| `WI` |
| `title` | 문서 제목 |
| `scope_code` | 영역 코드 (표준 코드) |
| `parent_pro` | 상위 PRO (WI 만 해당) |
| `parent_pol` | 상위 POL |
| `triggers` | 자연어 트리거 키워드 배열 (사용자 입력 매칭 핵심) |
| `aliases` | 동의어·구어체·약어 |
| `responsible_role` | 주 수행자 (RACI 의 R) |
| `approver_role` | 승인자 (RACI 의 A) — HITL 발동 시 대상 |
| `event_triggers` | 어떤 상황·이벤트에서 시작하는지 |
| `hitl_required` | true/false |
| `domains` | 적용 가능한 도메인 태그 |
