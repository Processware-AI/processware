# 요구사항 할당 (WI-CMMI-003-01-09)

> 상위 절차: [[PRO-CMMI-301_요구사항_개발_및_관리_절차_v1.0]]

## 1. 업무 목적
요구사항을 시스템·하위 컴포넌트에 명확히 할당하여 책임·검증 단위를 분명히 한다.

## 2. 수행 주체
- 주 수행자: 아키텍트
- 검토자: RDM PO
- 승인자: PM

## 3. 범위
- 솔루션 요구사항 정의 후

## 4. 입력 / 산출물
- Input: 솔루션 요구명세서, 아키텍처
- Output: [[TMP-CMMI-003-01-09_요구사항_할당표_v1.0]]

## 5. 수행 절차

### 5.1 사전 준비
1. 컴포넌트 인벤토리.
2. 인터페이스 정의.

### 5.2 수행 단계
1. **할당 매핑**.
2. **충돌·과잉 점검**.
3. **인터페이스 검증**.
4. **검토**.
5. **승인**.
6. **추적성 등록**.

### 5.3 완료 조건
- [ ] 할당표 승인본
- [ ] 충돌 0건
- [ ] 추적성 갱신

## 6. 인터페이스 부서
- 개발팀, 테스트

## 7. 주의사항 / 예외 처리

### 7.1 다중 할당
- 의도된 다중 할당은 표시.

### 7.2 미할당
- 미할당 시 0건 목표, 발견 시 즉시 할당.

### 7.3 변경
- CCB.
