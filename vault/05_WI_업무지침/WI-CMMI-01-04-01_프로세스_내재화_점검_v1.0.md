# 프로세스 내재화 점검 (WI-CMMI-01-04-01)

> 상위 절차: [[PRO-CMMI-01-04_구현_인프라_운영_절차_v1.0]]

## 1. 업무 목적
표준 프로세스가 부서·프로젝트에 실제 사용·정착되었는지를 정기 점검하여 형식적 적용을 방지한다.

## 2. 수행 주체
- 주 수행자: SEPG 점검 담당
- 검토자: QA, PCB
- 승인자: CEO

## 3. 범위
- 분기 1회 + 임시(부적합 다발 시)

## 4. 입력 / 산출물
- Input: OSSP, 부서 자료, 인터뷰
- Output: [[TMP-CMMI-01-04-01-01_내재화_점검표_v1.0]] 작성본, 시정 권고

## 5. 수행 절차

### 5.1 사전 준비
1. 점검 대상 부서·항목 선정.
2. 인터뷰 일정 조율.

### 5.2 수행 단계
1. **사전 자료 검토** — 산출물·기록 표본 추출.
2. **인터뷰** — 5~10명, 표준 질문지 사용.
3. **현장 관찰** — 일상 업무 30분 참관.
4. **점수화** — 인지·이해·실행·정착 4단계.
5. **결과 보고** — PCB 보고.
6. **시정 추적** — 차분기 재점검.

### 5.3 완료 조건
- [ ] 점검표 승인본
- [ ] 시정 권고 발행
- [ ] PCB 보고

## 6. 인터페이스 부서
- 각 부서, QA, OT

## 7. 주의사항 / 예외 처리

### 7.1 인터뷰 거부
- 거부 시 부서장 협조 요청, 재거부 시 PCB 보고.

### 7.2 표본 편향
- 직급·연차 균등 표본 추출.

### 7.3 점검 후 일시적 개선
- 점검 후 1개월 후 무작위 재방문으로 지속성 확인.
