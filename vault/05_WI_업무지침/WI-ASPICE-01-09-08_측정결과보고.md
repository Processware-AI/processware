# 측정 결과 보고 (WI-ASPICE-01-09-08)

> 상위 절차: [[PRO-ASPICE-01-09_프로젝트관리프로세스]]

## 1. 업무 목적
월간 측정 데이터를 취합·트렌드 분석하여 임계 초과 지표를 식별하고, 경영진 의사결정을 지원하며, 프로세스 개선(PIM.3) 기회로 연계한다. ASPICE 4.0 MAN.6 BP4(데이터 분석), BP5(측정 결과 의사소통)에 부합한다.

## 2. 수행 주체
- **주수행자**: Project Manager
- **검토자**: QA Manager, PMO
- **승인자**: Program Director

## 3. 범위
월 1회(매월 첫째 주) 정기 보고를 원칙으로 하며, 임계 초과 지표 발생 시 임시 보고를 추가한다. 보고 대상은 Program Director 및 사업부 운영위원회.

## 4. 입력 자료 / 산출물
- **Input**: 측정 DB(주차별 적재본), GQM 목표, 직전 월 측정 결과 보고서, 베이스라인 트렌드
- **Output**: 측정 결과 보고서(TMP-ASPICE-01-09-08-01), 경영진 대시보드 갱신, PIM.3 개선 기회 등록서

## 5. 수행 절차 (단계별)

### 5.1 사전 준비
1. 측정 DB에서 보고 월의 주차별 데이터 추출 권한 확인.
2. 직전 월 보고서 및 베이스라인(트렌드 라인) 조회.
3. PIM.3 개선 기회 등록 채널(Jira project) 접근 확인.

### 5.2 수행 단계 (ASPICE BP 참조)
1. **BP4 데이터 분석** — 지표별 월 평균/최대/최소 산출, 베이스라인 대비 회귀(regression) 검출.
2. **트렌드 시각화** — 라인 차트로 4주(또는 12주) 트렌드 작성, 임계선 표시.
3. **경보 항목 식별** — 목표 미달 지표·원인 가설 작성.
4. **개선 기회 연계** — 임계 초과 지표는 PIM.3 OPP-NNN 식별자로 등록, 보고서에 ID 명시.
5. **경영진 요약 작성** — 1페이지 요약(상태·핵심 이슈·요청 사항).
6. **BP5 의사소통** — 경영진 보고 회의에서 발표, 의사결정 결과 회의록 보존.
7. **대시보드 갱신** — 사내 BI 대시보드(또는 Confluence 페이지) 동기화.

### 5.3 완료 조건 체크리스트
- [ ] 모든 지표의 월 통계가 산출되었는가
- [ ] 트렌드 차트가 첨부되어 있는가
- [ ] 임계 초과 지표가 경보 항목으로 정리되어 있는가
- [ ] 경보 지표마다 PIM.3 개선 기회 ID가 부여되어 있는가
- [ ] 경영진 보고 회의록이 보존되었는가
- [ ] [[MAT-001_문서관리대장]] 갱신이 완료되었는가

## 6. 인터페이스 부서
- 경영진/사업부 운영위원회 — 의사결정
- PMO — 대시보드 운영
- PIM.3 프로세스 개선 담당 — 개선 기회 처리
- 도메인 팀 — 경보 지표 원인 분석 협조

## 7. 주의사항 / 예외 처리
1. **데이터 부족(2주 미만)**: 부분 보고 표기, 다음 월 정상 보고 예정 명시.
2. **지표 정의 변경**: 변경된 월부터 별도 시리즈로 표기, 직접 비교 금지.
3. **경보 미해소 누적**: 동일 지표 3개월 연속 경보 시 Program Director 별도 회의 소집.
4. **민감 데이터(인력 평가 등)**: 경영진 보고에서도 익명·집계 형태로만 제공.
