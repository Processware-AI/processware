# 데이터 수집·분석 (WI-CMMI-04-02-02)

> 상위 절차: [[PRO-CMMI-04-02_측정_및_분석_절차]] · 표준: CMMI-DEV V1.3 MA SG2 (전반)

## 1. 업무 목적
[[WI-CMMI-04-02-01]] 에서 정의한 측정 명세에 따라 데이터를 수집하고, 분석 명세에 따라 분석을 수행하여 의사결정에 사용 가능한 분석 보고서를 산출한다.

## 2. 수행 주체
- **주 수행자**: Data Owner (수집), Measurement Analyst (분석)
- **검토자**: QA Director
- **승인자**: Measurement Analyst Lead

## 3. 범위
PRO-CMMI-04-02 §5 의 **SP2.1(수집) ~ SP2.2(분석)** 적용. 저장·전달은 [[WI-CMMI-04-02-03]] 참조.

## 4. 입력 자료 / 산출물
- **Input**
  - 측정 명세서·수집·분석 절차서 ([[WI-CMMI-04-02-01]] 산출)
  - 원천 시스템 데이터
- **Output**
  - [[TMP-CMMI-04-02-02-01_측정_데이터셋_레지스터]]
  - [[TMP-CMMI-04-02-02-02_분석_보고서]]

## 5. 수행 절차

### 5.1 사전 준비
1. 측정 명세서 최신 버전 확인.
2. 수집 도구·스크립트·설문 양식 준비.
3. 분석 도구(Excel/Python/BI 등) 환경 점검.

### 5.2 수행 단계
1. **데이터 수집 (SP2.1 전반)**
   - 자동 수집: 스크립트·CI/CD 훅·로그 파이프라인 실행.
   - 수동 수집: 설문·인터뷰·체크리스트 적용.
   - 수집 일시·출처·수집자 기록.
2. **무결성 점검**
   - 측정 명세 §무결성 룰 적용 (중복·범위·타입·결측).
   - 이상치(outlier) 식별·근거 기록.
3. **데이터셋 등록**
   - 측정 데이터셋 레지스터에 메타데이터 등록 (ID·기간·레코드 수·해시).
4. **분석 수행 (SP2.2 전반)**
   - 분석 명세에 따라 통계·추세·비교·예측 수행.
   - 결과 해석 시 컨텍스트(프로젝트·조직 상황) 명시.
5. **결과 검증**
   - 분석 결과의 통계적 유의성·합리성 점검.
   - 한계점·가정 명시.
6. **분석 보고서 작성 (SP2.2 후반)**
   - 결과 표현 양식에 따라 보고서 작성.
   - 의사결정 권고사항 포함.

### 5.3 완료 조건
- [ ] 모든 계획 측정값 수집 (누락 ≤ 5%)
- [ ] 무결성 점검 통과 (부적합 ≤ 1%)
- [ ] 데이터셋 메타데이터 레지스터 등록
- [ ] 분석 명세에 따른 분석 완료
- [ ] 분석 결과의 한계·가정 명기
- [ ] 분석 보고서 작성·검토 완료

## 6. 인터페이스 부서
- **Data Owner**: 수집 책임
- **MA**: 분석 수행
- **QA**: 검증
- **이해관계자**: 분석 결과의 수요자
- **PMC**: 프로젝트 진척 분석 입력 ([[PRO-CMMI-02-02]])

## 7. 주의사항 / 예외 처리

### 7.1 수집 누락 (시스템 장애·휴가 등)
- 누락 ≥ 5% 발생 시 즉시 보고 → 백필 수집 또는 분석 시 한계 명시.
- 반복 누락 발생 시 자동화·이중화 검토.

### 7.2 이상치(outlier) 식별
- 통계적 기준(3σ, IQR×1.5 등) 적용.
- 이상치 제거 전 원인 분석 — 단순 제거 금지.

### 7.3 분석 결과가 비즈니스 가설과 상충
- 결과를 임의 조정 금지.
- 데이터·방법론 재검증 → 결과가 일관되면 그대로 보고하고 해석을 분리 제공.
