# 확인 준비 — 대상·환경·절차 (WI-CMMI-03-05-01)

> 상위 절차: [[PRO-CMMI-03-05_확인_절차]]

## 1. 업무 목적
"올바른 것을 만들었는가" 평가에 필요한 확인 대상·방법·환경·절차·기준을 사용자·고객 관점에서 수립한다.

## 2. 수행 주체
- **주 수행자**: Validation Lead
- **검토자**: Chief Engineer, 고객, Test Engineer
- **승인자**: Project Manager

## 3. 범위
PRO-CMMI-03-05 §5 의 **SP1.1 (Select Products for Validation) ~ SP1.2 (Establish the Validation Environment) ~ SP1.3 (Establish Validation Procedures and Criteria)**.

## 4. 입력 자료 / 산출물
- **Input**
  - VER 통과 산출물 (PRO-CMMI-03-04 SP3.2 인계)
  - 고객 요구사항·운영 시나리오 (RD)
  - 운영 환경 정보 (현장 조사·매뉴얼)
- **Output**
  - [[TMP-CMMI-03-05-01-01_확인_계획서]]

## 5. 수행 절차

### 5.1 사전 준비
1. VER 인계 산출물·결과 확보.
2. 운영 환경 또는 운영 모사 환경 가용성 점검.
3. 사용자·이해관계자 참여 일정 조율.

### 5.2 수행 단계
1. **확인 대상·방법 선택 (SP1.1)**
   - 산출물 / 컴포넌트 단위 + 라이프사이클 단계 단위 우선순위.
   - 방법 — 시연 / 시뮬레이션 / 프로토타입 / 사용자 시험 / 운영 시범.
2. **확인 환경 (SP1.2)**
   - 운영 환경(또는 모사) + 사용자 단말·통신 + 데이터.
   - 환경의 사용자 친화성·접근성 확인.
3. **확인 절차·기준 (SP1.3)**
   - 절차서 — 사용자 행동 단계 + 기대 결과 + 합격 기준.
   - 부분 충족·미충족 처리 흐름.
4. **검토·승인**

### 5.3 완료 조건 체크리스트
- [ ] 확인 대상 목록 (라이프사이클 단계별)
- [ ] 각 대상 확인 방법 매핑
- [ ] 환경 명세 (운영 또는 모사)
- [ ] 사용자 참여 일정 확보
- [ ] 절차·기준 (사용자 행동·기대·합격)
- [ ] 부분 충족·미충족 흐름
- [ ] PM 승인 결재

## 6. 인터페이스 부서
- **VAL 팀**: 주 수행
- **VER Lead**: 통과 산출물 인계 ([[PRO-CMMI-03-04]] SP3.2)
- **고객 / 사용자 대표**: 참여
- **PI Lead**: 운영 환경 정보 ([[PRO-CMMI-03-03]])
- **PPQA**: 절차 준수 감사

## 7. 주의사항 / 예외 처리

### 7.1 운영 환경 사용 불가
- 실 운영 환경 사용 불가:
  - 운영 모사 환경 + 신뢰도 등급.
  - 향후 운영 환경에서 보완 확인 일정.

### 7.2 사용자 참여 부족
- 충분한 사용자 표본 확보 어려움:
  - 대리인 + 사후 사용자 피드백.
  - 표본 크기 부족 시 결과 일반화 한계 명시.

### 7.3 절차 기준 정성 표현
- 사용자 만족도 등 정성 기준:
  - 5점 척도·NPS 등 정량화.
  - 임계값(예: 평균 ≥ 4.0) 사전 합의.
