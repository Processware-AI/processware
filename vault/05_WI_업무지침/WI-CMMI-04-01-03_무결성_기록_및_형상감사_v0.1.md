# 무결성 — CM 기록·형상감사 (WI-CMMI-04-01-03)

> 상위 절차: [[PRO-CMMI-04-01_형상_관리_절차]] · 표준: CMMI-DEV V1.3 CM SG3

## 1. 업무 목적
CM 활동(식별·통제·변경)의 기록을 작성·유지하고, 정기·수시 형상감사(FCA·PCA)를 수행하여 형상 정보의 무결성을 확인하고 부적합을 시정한다.

## 2. 수행 주체
- **주 수행자**: Configuration Manager (기록 작성), PPQA (감사 주관)
- **검토자**: CCB
- **승인자**: CEO/CTO (감사 결과보고서)

## 3. 범위
PRO-CMMI-04-01 §5 의 **SP3.1(기록 작성·유지) ~ SP3.2(형상감사)** 적용. 변경통제([[WI-CMMI-04-01-02]]) 누적 결과를 입력으로 사용.

## 4. 입력 자료 / 산출물
- **Input**
  - CR 처리 기록 ([[WI-CMMI-04-01-02]] 산출)
  - 베이스라인 기술서 ([[WI-CMMI-04-01-01]] 산출)
  - CM 시스템 운영 로그
- **Output**
  - [[TMP-CMMI-04-01-03-01_CM_기록부]] (작성본)
  - [[TMP-CMMI-04-01-03-02_형상감사_체크리스트_및_결과보고]] (작성본)
  - 부적합 시정조치 입력 → [[PRO-CMMI-04-03]]

## 5. 수행 절차

### 5.1 사전 준비
1. 감사 일정 확정 (분기 1회 정기 + 수시).
2. 감사 범위 결정 (대상 베이스라인·프로젝트).
3. 감사자 지명 — 평가 대상 작성자 배제(객관성).

### 5.2 수행 단계
1. **CM 기록 작성·유지 (SP3.1)**
   - CR 처리 결과를 변경 로그에 누적 기록.
   - CI 상태(현재 버전·승인·잠금) 갱신.
   - 베이스라인별 통계(CI 수·변경 건수) 산출.
2. **감사 계획 수립 (SP3.2 전반)**
   - FCA(기능 형상감사)·PCA(물리 형상감사) 범위·체크리스트 설계.
   - 표본 추출(전수 곤란 시 위험 가중 표본).
3. **감사 수행**
   - 체크리스트 항목별 증적 확인.
   - 부적합·관찰사항 분류·우선순위 부여.
4. **감사 결과보고 (SP3.2 후반)**
   - 부적합·관찰사항·개선권고를 보고서로 정리.
   - CCB/경영진 보고.
5. **시정조치 발행 및 추적**
   - 부적합 발견 시 PPQA 부적합 보고서로 핸드오프 ([[PRO-CMMI-04-03]]).
   - 시정조치 완료까지 본 WI 가 추적 대장 유지.

### 5.3 완료 조건
- [ ] CM 기록부가 정기적으로 갱신 (월 1회 이상)
- [ ] 분기 형상감사 1회 이상 실시·보고
- [ ] 모든 부적합이 PPQA 시정조치로 등록
- [ ] 시정 완료 또는 기한 연장 사유 명기
- [ ] 감사 결과보고서 경영진 결재
- [ ] MAT-001 등록

## 6. 인터페이스 부서
- **CM**: 기록 작성·유지 주 수행
- **PPQA**: 감사 주관 ([[PRO-CMMI-04-03]] §3 RACI: SP3.2 R)
- **CCB**: 감사 결과 검토·시정 의사결정
- **MA**: 감사 통계를 측정 저장소에 입력 ([[PRO-CMMI-04-02]])
- **경영진**: 결과 보고 수령·결재

## 7. 주의사항 / 예외 처리

### 7.1 감사 결과 부적합 다수 발생 (임계 초과)
- 분기 감사에서 부적합 비율 ≥ 5% 시:
  - 차기 감사 주기 단축(월 1회로 임시 전환).
  - OPF([[PRO-CMMI-01-02]]) 개선 안건으로 등록.

### 7.2 감사자 객관성 침해
- 감사 대상 작성자·관리자가 감사자로 지명된 경우 즉시 교체.
- 외부 감사 가능성 검토(중대 결함 시).

### 7.3 기록 손실/위변조
- CM 기록 로그 위변조 의심 시: 백업 로그 대조 + Senior Mgmt 보고 + 보안팀 협조.
- 위변조 확인 시 변경 롤백 + 인사 조치 검토.
