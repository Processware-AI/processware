# 요구사항 일관성 확인 (WI-CMMI-003-01-06)

> 상위 절차: [[PRO-CMMI-301_요구사항_개발_및_관리_절차_v1.0]]

## 1. 업무 목적
요구사항 간 충돌·중복·누락과 산출물 정합성을 정기 점검하여 결함 누수를 방지한다.

## 2. 수행 주체
- 주 수행자: RDM PO
- 검토자: QA, 아키텍트
- 승인자: PM

## 3. 범위
- 베이스라인 시·변경 시

## 4. 입력 / 산출물
- Input: 등록부, 설계·테스트
- Output: [[TMP-CMMI-003-01-06_요구사항_일관성_점검표_v1.0]]

## 5. 수행 절차

### 5.1 사전 준비
1. 점검 항목 정의.
2. 자동 검증 스크립트.

### 5.2 수행 단계
1. **충돌 점검**.
2. **중복 점검**.
3. **누락 점검**.
4. **산출물 정합 점검**.
5. **시정**.
6. **재점검**.

### 5.3 완료 조건
- [ ] 점검표 완료
- [ ] 부적합 0건 또는 시정 완료
- [ ] 재점검

## 6. 인터페이스 부서
- 설계, QA

## 7. 주의사항 / 예외 처리

### 7.1 의도된 중복
- 의도된 중복은 사유 기록.

### 7.2 충돌 미해결
- 의사결정 회의로 해결.

### 7.3 자동화 실패
- 수동 점검 보완.
