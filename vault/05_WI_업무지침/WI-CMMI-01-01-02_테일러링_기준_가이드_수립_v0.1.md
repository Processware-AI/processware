# 테일러링 기준·가이드 수립 (WI-CMMI-01-01-02)

> 상위 절차: [[PRO-CMMI-01-01_조직_표준프로세스_수립_유지_절차]]

## 1. 업무 목적
프로젝트가 OSSP를 자신의 정의 프로세스(Defined Process)로 변환할 때 적용할 수정·제거·추가 기준과 절차를 문서화하여, 일관된 테일러링 품질을 보장한다.

## 2. 수행 주체
- **주 수행자**: EPG
- **검토자**: Process Owner, 대표 프로젝트 PM
- **승인자**: CEO/CTO

## 3. 범위
PRO-CMMI-01-01 §5 의 **SP1.3** 단계에 한정. 본 가이드는 OSSP 수립 완료 후 적용. waiver 절차 포함.

## 4. 입력 자료 / 산출물
- **Input**: 승인된 OSSP, 라이프사이클 모델, 기존 프로젝트 사례
- **Output**: [[TMP-CMMI-01-01-02-01_테일러링_가이드라인]], waiver 양식

## 5. 수행 절차

### 5.1 사전 준비
1. OSSP·라이프사이클 모델 베이스라인 ID 확인.
2. 최근 12개월 내 프로젝트 정의 프로세스 사례 5개 이상 수집.

### 5.2 수행 단계
1. **테일러링 차원(Dimension) 식별** — 규모/기간/리스크/도메인/팀 경험 등
2. **차원별 기준 도출** — 예: 1인월 미만 프로젝트는 PMC SP1.5 생략 가능
3. **수정·제거·추가 규칙 작성** — 각 18 PA 별로 허용·금지 항목 명시
4. **문서화 표준 명시** — 정의 프로세스 기록 형식·필수 섹션
5. **waiver 절차 정의** — 가이드 범위 초과 시 신청·심의·승인 흐름
6. **피어 리뷰** — EPG + PM 대표 + PPQA
7. **승인 및 CM 베이스라인 등록**

### 5.3 완료 조건
- [ ] 테일러링 가이드라인 작성·승인 완료
- [ ] 18 PA 각각의 허용·금지 항목 명시
- [ ] waiver 절차 정의·승인 라인 명시
- [ ] 피어 리뷰 결함 해소
- [ ] CM 베이스라인 + MAT-001 등록

## 6. 인터페이스 부서
- **EPG**: 본 지침 주 수행
- **CM**: 베이스라인 통제 ([[PRO-CMMI-04-01]])
- **IPM**: 본 가이드를 적용하여 정의 프로세스 수립 ([[PRO-CMMI-02-05]] SP1.3)

## 7. 주의사항 / 예외 처리

### 7.1 신규 도메인 프로젝트
- 기존 가이드 미적용 영역 → 즉시 임시 테일러링 + 1개월 내 정식 등재.

### 7.2 강제 표준 충돌
- 외부 규제(예: 의료 21 CFR Part 11) 가 더 엄격 시 가이드 범위 자동 축소 — waiver 불가 영역 표기.

### 7.3 가이드 위반 사후 발견
- PPQA 가 위반 감지 시 NCR 발행 → 시정조치 [[PRO-CMMI-02-02]] SG2 흐름.
