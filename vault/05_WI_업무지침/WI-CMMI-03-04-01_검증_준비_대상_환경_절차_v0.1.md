# 검증 준비 — 대상·환경·절차 (WI-CMMI-03-04-01)

> 상위 절차: [[PRO-CMMI-03-04_검증_절차]]

## 1. 업무 목적
"올바르게 만들었는가" 평가에 필요한 검증 대상·방법·환경·절차·기준을 사전에 수립하여 검증 활동의 객관성·재현성을 확보한다.

## 2. 수행 주체
- **주 수행자**: Verification Lead
- **검토자**: Chief Engineer, Test Engineer, RD/TS Owner
- **승인자**: Project Manager

## 3. 범위
PRO-CMMI-03-04 §5 의 **SP1.1 (Select Work Products for Verification) ~ SP1.2 (Establish the Verification Environment) ~ SP1.3 (Establish Verification Procedures and Criteria)**.

## 4. 입력 자료 / 산출물
- **Input**
  - 요구사항 베이스라인 (RD/REQM)
  - 설계·TDP·ICD (TS)
  - 컴포넌트 / 통합 일정 (PI)
- **Output**
  - [[TMP-CMMI-03-04-01-01_검증_계획서]] (대상·방법·환경·절차·기준 통합)

## 5. 수행 절차

### 5.1 사전 준비
1. 요구사항·설계 베이스라인 확정.
2. 가용 검증 도구·시험 데이터·시뮬레이터 목록 확보.
3. 시험 데이터의 개인정보 / 라이선스 검토.

### 5.2 수행 단계
1. **검증 대상·방법 선택 (SP1.1)**
   - 산출물별 검증 우선순위 (요구·설계·코드·문서·통합 결과).
   - 방법 — 시험(Test) / 분석(Analysis) / 검사(Inspection) / 시연(Demonstration) — 매핑.
2. **검증 환경 수립 (SP1.2)**
   - 시험 환경(HW/SW), 시뮬레이터, 시험 데이터.
   - 환경의 격리·재현성 보장.
3. **검증 절차·기준 (SP1.3)**
   - 절차서: 단계·입력·기대 출력·합격 기준.
   - 부적합 처리 흐름.
4. **검토·승인**

### 5.3 완료 조건 체크리스트
- [ ] 검증 대상 목록 (산출물 유형별)
- [ ] 각 대상에 검증 방법 매핑 (T/A/I/D)
- [ ] 검증 환경 명세 + 재현성 점검
- [ ] 시험 데이터 라이선스·개인정보 검토 결과
- [ ] 절차서·기준 표 (입력·기대·합격)
- [ ] 부적합 처리 흐름
- [ ] PM 승인 결재

## 6. 인터페이스 부서
- **VER 팀**: 주 수행
- **RD / TS Owner**: 요구·설계 정보 공급, 검증가능성 확인
- **Test Engineer**: 시험 실행 협업
- **CM**: 검증 절차 베이스라인 ([[PRO-CMMI-04-01]])
- **PPQA**: 절차 준수 감사 ([[PRO-CMMI-04-03]])

## 7. 주의사항 / 예외 처리

### 7.1 검증 가능성 부재
- 요구사항이 검증 가능하지 않음 (정성):
  - RD WI-04 (분석 5축) 회귀 — 정량화 요청.
  - 본 WI 진행 보류.

### 7.2 시험 환경 미구축
- 시험 환경이 일정 내 미구축:
  - 단계 분리 — 우선 가능 영역 시험 후 환경 완성 시 잔여 시험.
  - RSKM 등록.

### 7.3 시험 데이터 개인정보 우려
- 실제 데이터 사용 불가:
  - 합성 데이터 / 익명 데이터.
  - 합성률 100% 시 운영 시나리오 대표성 별도 검증.
