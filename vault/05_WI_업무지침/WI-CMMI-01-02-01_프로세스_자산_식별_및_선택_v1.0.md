# 프로세스 자산 식별 및 선택 (WI-CMMI-01-02-01)

> 상위 절차: [[PRO-CMMI-01-02_프로세스_자산_개발_절차_v1.0]]

## 1. 업무 목적
PAL(프로세스 자산 라이브러리)에 등재할 자산 후보를 식별하고 비즈니스 가치·재사용성 기반으로 우선순위를 결정·선택하여, 자산 개발 자원을 효과적으로 배분한다.

## 2. 수행 주체
- 주 수행자: SEPG 위원
- 검토자: PCB
- 승인자: SEPG Lead

## 3. 범위
- 표준 PA(20종) 의 POL/PRO/WI/TMP/MAT 후보
- 외부 표준(예: CMMI-DEV-ML3) 인용 가능 자산 포함

## 4. 입력 / 산출물
- Input: 비즈니스 목표, 격차 분석, 외부 표준 매핑
- Output: [[TMP-CMMI-01-02-01-01_프로세스_자산_식별표_v1.0]] 승인본

## 5. 수행 절차

### 5.1 사전 준비
1. 기존 자산 목록(MAT-001) 확인.
2. PA별 격차 보고 수집.

### 5.2 수행 단계
1. **후보 수집** — SEPG 위원이 PA별 자산 후보를 식별표에 등재.
2. **가치 평가** — 비즈니스 정렬, 재사용성, 변동성 3축 점수화(1~5).
3. **우선순위 결정** — 점수 합 기반 H/M/L 분류.
4. **PCB 심의** — 자원 배분 적정성 확인.
5. **SEPG Lead 승인** — 식별표 확정.
6. **개발 백로그 등재** — 우선순위에 따라 작업 큐 등록.

### 5.3 완료 조건
- [ ] 식별표 승인본 보관
- [ ] 백로그 등재
- [ ] [[MAT-001_문서관리대장]] 동기화

## 6. 인터페이스 부서
- PCB, Process Owner, QA

## 7. 주의사항 / 예외 처리

### 7.1 외부 표준 라이선스
- 외부 표준 원문 직접 복사 금지(20단어 일치 금지). 인용·paraphrase 만 허용.

### 7.2 후보 누락
- PA별 최소 1개 후보 식별을 강제. 0개 보고 시 SEPG Lead 가 검토 코멘트 부여 후 재식별.

### 7.3 우선순위 충돌
- 동점 시 비즈니스 정렬 점수 우선, 차순위는 변동성(낮을수록 우선).
