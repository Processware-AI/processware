# 팀 구성·운영 규칙 수립 (WI-CMMI-01-01-05)

> 상위 절차: [[PRO-CMMI-01-01_조직_표준프로세스_수립_유지_절차]]

## 1. 업무 목적
프로젝트가 일관된 기준 위에서 팀을 구성·운영할 수 있도록, 조직 차원의 팀 구조 가이드(역할·책임·권한·의사결정 메커니즘·해체 절차)와 권한위임(empowerment) 규칙을 정의·유지한다. 이는 통합 프로젝트 관리(IPM SG2 IPPD 분기)와 직접 연계된다.

## 2. 수행 주체
- **주 수행자**: EPG (규칙 정의), HR (역할·평가 연동)
- **검토자**: Process Owner, 법무(노동/개인정보), 대표 PM
- **승인자**: 경영진 (CEO/CTO)

## 3. 범위
PRO-CMMI-01-01 §5 의 **SP1.7 (Establish Rules and Guidelines for Teams)** 단계에 적용. 본 지침은 조직 표준 규칙을 정의하며, **프로젝트별 팀 인스턴스화는 [[PRO-CMMI-02-05]] IPM SG2 (IPPD 보조 목표)** 에서 본 규칙을 인용해 적용한다.

## 4. 입력 자료 / 산출물
- **Input**
  - OSSP (WI-CMMI-01-01-01 산출)
  - HR 직무체계·평가체계
  - 조직 의사결정 정책 (POL-CMMI-01)
  - 노동·개인정보 관련 법규
- **Output**
  - [[TMP-CMMI-01-01-05-01_팀_운영_규칙서]] (작성본)
  - CM 베이스라인 등록
  - PAL 등록

## 5. 수행 절차

### 5.1 사전 준비
1. OSSP 정의서 §3 프로세스 요소표 확인 (각 프로세스의 "역할" 항목 추출).
2. 기존 팀 구성 사례 인터뷰 (지난 12개월 종료 프로젝트 3건 이상 회고).
3. HR 부서와 직무 체계 정합성 점검 일정 확정.

### 5.2 수행 단계
1. **권한위임(Empowerment) 메커니즘 정의** (Subpractice 1)
   - 의사결정 권한 4단계 정의: 팀 자율 / PM 결정 / 부서장 결정 / 경영진 결정.
   - 의사결정 시한(SLA): 단계별 응답 시한 명시 (예: 부서장 24h, 경영진 5BD).
   - Escalation 트리거 정의.
2. **팀 구조 가이드 수립** (Subpractice 2)
   - 팀 유형 분류: Feature 팀 / Component 팀 / Platform 팀 / Tiger 팀 (한시).
   - 유형별 권장 구성: 최소·최대 인원, 핵심 역할, 다기능성 비율.
   - 신규 팀 생성 절차: 제안 → 검토 → 승인 → 킥오프 → 운영 → 해체.
3. **운영 규칙 수립** (Subpractice 3)
   - 회의 표준: 주기·아젠다·기록 보관.
   - 합의 기준 (Working Agreements): 정의·시작·종료(DoR/DoD)·코드리뷰·근무·휴가·On-call.
   - 갈등 해결 절차: 1차 팀 내 / 2차 PM / 3차 HR + 부서장.
   - 성과 평가 연동: 개인 + 팀 KPI 가중치 (HR 합의).
4. **법무 검토**
   - 노동법(휴게·연장근로) / 개인정보(원격근무 모니터링) / 차별금지 적합성.
5. **피어 리뷰**
   - EPG + HR + 대표 PM + 노조/협의회(있을 시) 참여.
   - 결함 항목 수정·재검토.
6. **경영진 승인 및 CM 베이스라인 등록**
   - 승인 후 [[PRO-CMMI-04-01_형상_관리_절차]] 따라 베이스라인화.
   - PAL 등록 → MAT-001 행 추가.
   - HR 시스템 직무·역할 매핑 동기화.

### 5.3 완료 조건 (체크리스트)
- [ ] 권한위임(Empowerment) 4단계 의사결정 모델 정의
- [ ] 의사결정 시한 SLA 단계별 명시
- [ ] 팀 유형 분류(최소 3유형) + 유형별 구성 가이드 작성
- [ ] 신규 팀 생성·해체 절차 6단계 명시
- [ ] 회의/Working Agreement/갈등해결 운영 규칙 작성
- [ ] 법무 검토 결재 (노동·개인정보·차별금지 적합)
- [ ] HR 직무체계 매핑 동기화 완료
- [ ] 피어 리뷰 결함 0건 또는 모두 해소
- [ ] CEO/CTO 승인 결재 첨부
- [ ] CM 베이스라인 ID 부여 + MAT-001 등록 + PAL 등재

## 6. 인터페이스 부서
- **EPG**: 본 지침 주 수행
- **HR**: 직무·역할·평가 연동, 노동법 1차 검토
- **법무**: 노동·개인정보·차별금지 검토
- **CM**: 베이스라인 등록 ([[PRO-CMMI-04-01]])
- **IPM (PRO-CMMI-02-05)**: 본 규칙을 인용해 프로젝트별 팀 인스턴스화
- **OT (PRO-CMMI-01-03)**: 팀 리더십·갈등해결 교육 니즈 입력

## 7. 주의사항 / 예외 처리

### 7.1 노동법 위반 우려 발생
- 운영 규칙(특히 회의·근무·On-call) 도입 전 HR + 법무 합동 검토 의무.
- 시범 운영 중 노동법 이슈 제기 시: 즉시 해당 조항 보류 → 30일 내 재검토.

### 7.2 노조/협의회 협의 필요
- 근무·평가·휴가 관련 조항은 노사협의회 사전 협의 의무 (법령에 따른 경우).
- 협의 미완료 조항은 운영 보류, 잠정 조항 명시.

### 7.3 다국가/원격 팀
- 시차 ≥ 4시간 또는 다국가 팀: On-call 분배·회의 시간대 별도 부록 작성.
- 개인정보 국외이전 시 별도 동의·근거 마련.

### 7.4 한시 Tiger 팀 (Crisis Response)
- 6주 이하 한시 팀은 SP1.7 운영 규칙 일부 면제 가능 (회의 표준·평가 연동).
- 단, 권한위임 모델·갈등해결 절차는 의무 적용.

### 7.5 해체 시 노하우 손실
- 팀 해체 시 회고 워크숍 + 교훈 등록 의무 ([[PRO-CMMI-01-02]] OPF 환류).
- 핵심 인력 1명 이상 PAL 회고 기여 의무.
