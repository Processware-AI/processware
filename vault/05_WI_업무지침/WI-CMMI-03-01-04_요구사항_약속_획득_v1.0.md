# 요구사항 약속 획득 (WI-CMMI-03-01-04)

> 상위 절차: [[PRO-CMMI-03-01_요구사항_개발_및_관리_절차_v1.0]]

## 1. 업무 목적
요구사항에 대한 책임 있는 약속을 이해관계자로부터 획득·기록하여 후속 검증·인수의 기준으로 사용한다.

## 2. 수행 주체
- 주 수행자: RDM PO
- 검토자: PM
- 승인자: Sponsor

## 3. 범위
- 베이스라인 직전, 변경 시 갱신

## 4. 입력 / 산출물
- Input: 등록부, 변환 매트릭스
- Output: [[TMP-CMMI-03-01-04-01_요구사항_약속서_v1.0]]

## 5. 수행 절차

### 5.1 사전 준비
1. 약속 대상자 확인.
2. 회의 일정.

### 5.2 수행 단계
1. **검토 회의**.
2. **합의 확인**.
3. **변경 요구 처리**.
4. **약속 서명**.
5. **공유**.
6. **이행 추적 시작**.

### 5.3 완료 조건
- [ ] 약속서 서명본
- [ ] 미합의 항목 별도 처리
- [ ] 추적 시작

## 6. 인터페이스 부서
- 법무, 영업

## 7. 주의사항 / 예외 처리

### 7.1 미합의
- 미합의 항목은 별도 등록·재협상.

### 7.2 변경
- 후속 변경 시 재약속.

### 7.3 강요된 약속
- 압박으로 강요된 약속은 리스크 등록.
