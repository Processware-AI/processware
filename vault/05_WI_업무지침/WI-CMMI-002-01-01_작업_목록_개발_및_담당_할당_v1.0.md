# 작업 목록 개발 및 담당 할당 (WI-CMMI-002-01-01)

> 상위 절차: [[PRO-CMMI-201_프로젝트_계획_절차_v1.0]]

## 1. 업무 목적
프로젝트 산출물·작업을 WBS 로 분해하고 담당자를 명확히 할당하여 책임 공백 없이 실행되도록 한다.

## 2. 수행 주체
- 주 수행자: 프로젝트 PM
- 검토자: PMO, 부서장
- 승인자: 프로젝트 PM

## 3. 범위
- 모든 신규 프로젝트 + 중대 변경 시

## 4. 입력 / 산출물
- Input: 프로젝트 헌장, 요구사항, 라이프사이클 모델
- Output: [[TMP-CMMI-002-01-01_WBS_및_담당할당표_v1.0]] 승인본

## 5. 수행 절차

### 5.1 사전 준비
1. 라이프사이클 결정.
2. 산출물 인벤토리 작성.

### 5.2 수행 단계
1. **WBS 분해** — 산출물 → 작업 → 활동 (3~5 레벨).
2. **작업 명명** — 동사+명사 형식.
3. **담당 할당** — Primary + Backup.
4. **공수 추정 입력** — [[WI-CMMI-002-03-03_작업량_일정_자원_추정_v1.0]] 와 연동.
5. **검토** — PMO·부서장 검토.
6. **승인** — PM 승인 후 베이스라인.

### 5.3 완료 조건
- [ ] WBS 모든 노드에 담당자 1명 이상
- [ ] 베이스라인 등록
- [ ] [[MAT-001_문서관리대장]] Row

## 6. 인터페이스 부서
- PMO, 부서장, EST

## 7. 주의사항 / 예외 처리

### 7.1 미할당 작업
- 미할당 노드 발견 시 24시간 내 할당 또는 PM 직접 수행.

### 7.2 부서 간 충돌
- 동일 자원 다중 할당 시 부서장 협의·우선순위 결정.

### 7.3 변경
- 베이스라인 후 WBS 변경은 [[PRO-CMMI-402_형상관리_절차_v1.0]] CCB 적용.
