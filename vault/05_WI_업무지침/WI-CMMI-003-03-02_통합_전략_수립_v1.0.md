# 통합 전략 수립 (WI-CMMI-003-03-02)

> 상위 절차: [[PRO-CMMI-303_제품통합_절차_v1.0]]

## 1. 업무 목적
컴포넌트 통합 순서·환경·기준·도구를 사전 정의하여 통합의 일관성·예측가능성을 높인다.

## 2. 수행 주체
- 주 수행자: Integration Lead
- 검토자: PM
- 승인자: PM

## 3. 범위
- 신규 프로젝트 베이스라인 시

## 4. 입력 / 산출물
- Input: 아키텍처, 의존도
- Output: [[TMP-CMMI-003-03-02_통합전략서_v1.0]]

## 5. 수행 절차

### 5.1 사전 준비
1. 의존도 분석.
2. 환경 옵션.

### 5.2 수행 단계
1. **순서 결정** — Top-down/Bottom-up/Big-bang.
2. **환경 정의**.
3. **기준 정의**.
4. **도구 선택**.
5. **검토·승인**.
6. **배포**.

### 5.3 완료 조건
- [ ] 전략서 승인본
- [ ] 도구 셋업
- [ ] 환경 준비

## 6. 인터페이스 부서
- 개발, QA, CM

## 7. 주의사항 / 예외 처리

### 7.1 의존 순환
- 의존 순환 발견 시 아키텍처 보정.

### 7.2 환경 부족
- 가상화 활용.

### 7.3 변경
- 단계 게이트 시 갱신.
