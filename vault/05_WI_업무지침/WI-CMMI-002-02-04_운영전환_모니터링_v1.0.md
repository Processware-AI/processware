# 운영전환 모니터링 (WI-CMMI-002-02-04)

> 상위 절차: [[PRO-CMMI-202_프로젝트_모니터링_및_통제_절차_v1.0]]

## 1. 업무 목적
프로젝트 산출물의 운영 전환 진척과 hypercare 기간 안정성을 모니터링하여 운영 사고를 예방한다.

## 2. 수행 주체
- 주 수행자: PM
- 검토자: 운영팀
- 승인자: PM

## 3. 범위
- 운영 인계 진행 + hypercare 기간(통상 30일)

## 4. 입력 / 산출물
- Input: 전환 계획서, 운영 사고 로그
- Output: [[TMP-CMMI-002-02-04_전환_모니터링표_v1.0]]

## 5. 수행 절차

### 5.1 사전 준비
1. 모니터링 지표 정의.
2. 사고 분류 기준.

### 5.2 수행 단계
1. **일일 사고 점검** — hypercare 기간.
2. **운영팀 데일리 미팅**.
3. **사고 분석**.
4. **시정 조치**.
5. **종료 검토** — hypercare 종료 시 안정성 확인.
6. **공식 종결**.

### 5.3 완료 조건
- [ ] hypercare 기간 사고 < 임계값
- [ ] 운영팀 종결 서명
- [ ] 교훈 등재

## 6. 인터페이스 부서
- 운영팀, CS

## 7. 주의사항 / 예외 처리

### 7.1 중대 사고
- 즉시 hypercare 연장.

### 7.2 인수 거절
- 운영팀 인수 거절 시 사유 보완 후 재인계.

### 7.3 데이터 부족
- 사고 로그 부재 시 인터뷰로 보완.
