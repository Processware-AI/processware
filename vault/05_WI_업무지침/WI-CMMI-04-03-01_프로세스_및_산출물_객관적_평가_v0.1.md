# 프로세스·산출물 객관적 평가 (WI-CMMI-04-03-01)

> 상위 절차: [[PRO-CMMI-04-03_프로세스_제품_품질보증_절차]] · 표준: CMMI-DEV V1.3 PPQA SG1

## 1. 업무 목적
프로젝트·조직의 프로세스 수행과 산출물에 대해 정의된 기준(정책·표준·절차) 대비 준수 여부를 객관적·독립적으로 평가하여 부적합·관찰사항을 식별한다.

## 2. 수행 주체
- **주 수행자**: QA Engineer (PPQA) — 평가 대상 작성자 배제 원칙
- **검토자**: Process Owner
- **승인자**: QA Director (정기) / Senior Mgmt (중대 부적합)

## 3. 범위
PRO-CMMI-04-03 §5 의 **SP1.1(프로세스 객관적 평가) ~ SP1.2(산출물 객관적 평가)** 적용. 부적합 의사소통·해결은 [[WI-CMMI-04-03-02]] 참조.

## 4. 입력 자료 / 산출물
- **Input**
  - 적용 정책·표준·절차 (조직 OPA)
  - 평가 계획 (분기 단위)
  - 평가 대상 프로세스 수행 증적·산출물
- **Output**
  - [[TMP-CMMI-04-03-01-01_프로세스_평가_체크리스트_및_보고서]]
  - [[TMP-CMMI-04-03-01-02_산출물_평가_체크리스트_및_부적합_보고서]]

## 5. 수행 절차

### 5.1 사전 준비
1. 평가 계획 확정 (분기 단위, 대상 프로세스·산출물·일정).
2. 평가 기준(정책·표준·절차) 최신 버전 확인.
3. 평가자 지정 — 평가 대상 작성·실행자 배제(객관성).
4. 평가 체크리스트 준비·검토.

### 5.2 수행 단계
1. **프로세스 객관적 평가 (SP1.1)**
   - 대상 프로세스 식별: PRO 단위 또는 PA 단위.
   - 체크리스트 항목별 증적 확인 (실행 로그·결재 문서·산출물).
   - 적합·부적합·관찰사항·N/A 판정.
2. **산출물 객관적 평가 (SP1.2)**
   - 대상 산출물 식별: TMP·REC·CI 등.
   - 산출물의 정확성·완전성·일관성·표준 부합성 확인.
   - 적합·부적합·관찰사항·N/A 판정.
3. **증적 보존**
   - 평가에 사용된 증적의 사본·링크 보존 (재현 가능성).
4. **평가 보고서 작성**
   - 평가 메타(대상·일자·평가자·기준)·결과·증적 링크 정리.
   - 부적합·관찰사항을 우선순위와 함께 보고.
5. **검토·승인**
   - Process Owner 검토 (사실관계 확인).
   - QA Director 승인.

### 5.3 완료 조건
- [ ] 평가 계획에 따라 모든 대상 프로세스·산출물 평가 완료
- [ ] 평가자 객관성 충족 (작성자·실행자 배제 확인)
- [ ] 체크리스트 항목별 판정 완료
- [ ] 증적 보존 (사본·링크)
- [ ] 평가 보고서 작성·검토·승인
- [ ] 부적합 발견 시 [[WI-CMMI-04-03-02]] 핸드오프

## 6. 인터페이스 부서
- **PPQA**: 본 지침 주 수행
- **Process Owner**: 평가 결과 사실관계 검토
- **Project Manager**: 평가 일정 협조
- **CM**: 형상감사 결과 ↔ PPQA 부적합 연계 ([[WI-CMMI-04-01-03]])
- **All PAs**: PPQA 는 전 PA 객관적 평가 지원 (PPQA-supports-all)

## 7. 주의사항 / 예외 처리

### 7.1 평가자 객관성 충족 불가
- 조직 규모상 평가 대상 작성자와 평가자가 동일 인물 외 대안 없는 경우:
  - 외부 평가자(자매 부서·외부 컨설턴트) 활용.
  - 또는 동료 검토(peer review) 보강.

### 7.2 평가 시간 부족 (정기 일정 충돌)
- 분기 평가 누락 우려 시 표본 평가로 전환:
  - 위험 가중 표본(중대 프로세스·산출물 우선).
  - 누락 영역은 다음 분기 우선 평가로 이월 등록.

### 7.3 평가 중 중대 부적합 발견
- 즉시 Project Manager 와 QA Director 에 통보.
- 시스템·제품 출시 영향 시 출시 보류 검토 권고.
