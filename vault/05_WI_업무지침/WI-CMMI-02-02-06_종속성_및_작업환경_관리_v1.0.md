# 종속성 및 작업환경 관리 (WI-CMMI-02-02-06)

> 상위 절차: [[PRO-CMMI-02-02_프로젝트_모니터링_및_통제_절차_v1.0]]

## 1. 업무 목적
프로젝트 진행에 영향을 주는 외부 종속성과 작업환경 변화를 등록·모니터링하여 일정·품질 영향 최소화.

## 2. 수행 주체
- 주 수행자: PM
- 검토자: IT, PMO
- 승인자: PM

## 3. 범위
- 외부 시스템·라이브러리·라이선스·환경

## 4. 입력 / 산출물
- Input: 아키텍처, 외부 인터페이스 명세
- Output: [[TMP-CMMI-02-02-06-01_종속성_등록부_v1.0]]

## 5. 수행 절차

### 5.1 사전 준비
1. 종속성 식별 워크숍.
2. 환경 정의 확인.

### 5.2 수행 단계
1. **등록**.
2. **상태 모니터링** — 격주.
3. **변경 알람**.
4. **영향 평가**.
5. **대응 결정**.
6. **이력 기록**.

### 5.3 완료 조건
- [ ] 등록부 갱신
- [ ] 변경 시 영향 분석 완료

## 6. 인터페이스 부서
- IT, 보안팀, 외부 벤더

## 7. 주의사항 / 예외 처리

### 7.1 외부 라이브러리 EOL
- EOL 6개월 전 대체 평가.

### 7.2 보안 취약점
- CVE 발견 시 24시간 내 패치/완화.

### 7.3 환경 차이
- 개발·테스트·운영 환경 차이 발견 시 즉시 정합화.
