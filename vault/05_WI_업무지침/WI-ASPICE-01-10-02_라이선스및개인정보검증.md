# 라이선스 및 개인정보 검증 (WI-ASPICE-01-10-02)

> 상위 절차: [[PRO-ASPICE-01-10_ML데이터관리프로세스]]
> ASPICE 4.0 SUP.11.BP2 — 라이선스/개인정보 검증

## 1. 업무 목적

본 지침은 [[WI-ASPICE-01-10-01_데이터수집및라벨링]] 을 통해 수집된 ML 학습 데이터셋의 **라이선스 적법성과 개인정보(PII) 처리 적합성** 을 검증하여, 상업용 ADAS 제품 개발 과정에서 저작권·개인정보보호법 위반을 사전 차단하는 데 목적이 있다. 모든 데이터셋은 본 검증을 통과한 경우에 한하여 후속 [[WI-ASPICE-01-10-03_데이터품질평가]] 단계로 인계된다.

## 2. 수행 주체

| 역할 | 담당 |
|---|---|
| 주수행자 | Data Engineer |
| 검토자 | Legal / Privacy Officer / QA |
| 승인자 | Data Manager |

## 3. 범위

VWAY Motors 가 자체 수집·구매·외부 라이선스 받은 모든 ML 데이터셋(이미지·LiDAR·Radar·CAN log·운전자 행동 데이터) 의 라이선스 검증과 PII 검증에 적용한다. 사내 통계용 데이터(개발 metric 등)와 [[PRO-ASPICE-01-10_ML데이터관리프로세스]] §2 비대상 데이터는 제외한다.

## 4. 입력 자료 / 산출물

- **Input**:
  - [[WI-ASPICE-01-10-01_데이터수집및라벨링]] 산출 Raw Dataset + Labels
  - 데이터 출처 증빙(라이선스 계약서, 구매 영수증, 공개 데이터셋 라이선스 텍스트)
  - PII 탐지 도구(얼굴 인식·번호판 인식 모듈) 결과
- **Output**:
  - [[TMP-ASPICE-01-10-02-01_데이터라이선스검증서]] 작성 산출물
  - 사용 승인 데이터셋 목록 (라이선스 대장)
  - 부적합 데이터셋 처리 로그 (제외/재정제)

## 5. 수행 절차

### 5.1 사전 준비

1. 검증 대상 데이터셋 목록을 [[PRO-ASPICE-01-10_ML데이터관리프로세스]] §5 단계 1 산출물에서 확보한다.
2. 라이선스 분류 카탈로그(CC-BY, CC-BY-SA, CC-BY-NC, MIT, Apache-2.0, 상용, 비공개 등)를 최신 상태로 갱신한다.
3. 금지 라이선스 목록을 확인한다 (예: 상업적 사용 금지 라이선스인 CC-BY-NC 계열은 양산 ADAS 제품용 학습에 사용 금지).
4. PII 자동 탐지 도구의 모델·룰셋 버전이 최신인지 확인한다 (얼굴 검출, 차량 번호판 검출, 음성 화자 식별 등).
5. Legal/Privacy Officer 일정·연락 채널을 확보한다.

### 5.2 수행 단계 (ASPICE SUP.11.BP2 참조)

1. **데이터셋별 출처 식별** — 각 데이터셋(또는 데이터 청크)에 대해 출처(공개 저장소·구매 벤더·내부 수집)와 라이선스 유형을 기록한다.
2. **라이선스 카탈로그 매핑** — 출처별 라이선스를 카탈로그에 매핑하고, 상업적 사용 가능 여부·재배포 가능 여부·파생물 라이선스 요구사항(copyleft) 을 명시한다.
3. **금지 라이선스 탐지** — 양산용 학습 데이터셋 중 금지 라이선스(CC-BY-NC 계열, GPL 계열 학습 가중치 전이 우려, Unknown) 사용 여부를 확인한다. 위반 시 즉시 제외 처리한다.
4. **PII 자동 탐지 실행** — 이미지·영상 데이터에 대해 얼굴 검출·차량 번호판 검출 모델을 실행하고, 음성 데이터에 대해 화자 식별 가능성을 점검한다.
5. **PII 처리 결정** — 탐지 결과별로 (a) 자동 블러/마스킹 적용, (b) 동의서 보유 시 원본 유지, (c) 동의서 없고 마스킹 불가 시 제외 등 처리 방침을 결정·실행한다.
6. **GDPR / 개인정보보호법 준수 확인** — 데이터 주체 동의서 보유 여부, 국외 이전 적법성, 보유기간 정당성, 가명·익명 처리 적정성을 확인한다.
7. **법무 검토 요청** — 라이선스 매핑 결과와 PII 처리 결과를 종합하여 Legal/Privacy Officer 에게 검토를 요청하고, 검토 의견을 기록한다.
8. **사용 승인 결재** — Data Manager 가 종합 결과를 검토하여 데이터셋별 사용 승인/조건부 승인/반려를 결정한다.
9. **결과 통보** — 승인 데이터셋은 [[WI-ASPICE-01-10-03_데이터품질평가]] 로 인계한다. 반려 데이터셋은 [[WI-ASPICE-01-10-01_데이터수집및라벨링]] 으로 회송하여 재수집·재정제한다.

### 5.3 완료 조건 체크리스트

- [ ] 검증 대상 데이터셋 전수에 대해 라이선스 유형이 카탈로그에 매핑되었다.
- [ ] 금지 라이선스 탐지 결과가 0건이거나, 발견 시 전수 제외 처리되었다.
- [ ] PII 자동 탐지가 전 이미지·영상 샘플에 대해 실행되었다.
- [ ] 탐지된 PII 는 마스킹·동의서 확인·제외 중 한 가지 방침으로 처리되었다.
- [ ] GDPR / 개인정보보호법 준수 확인 결과가 기록되었다.
- [ ] Legal/Privacy Officer 검토 의견이 첨부되었다.
- [ ] Data Manager 의 사용 승인 결재가 완료되었다.
- [ ] [[TMP-ASPICE-01-10-02-01_데이터라이선스검증서]] 가 작성·결재 완료되어 `08_REC_기록/` 에 보관되었다.
- [ ] [[MAT-001_문서관리대장]] 에 본 산출물이 등록되었다.

## 7. 주의사항 / 예외 처리

1. **불명 라이선스(Unknown)** — 출처는 확인되나 라이선스가 불명확한 데이터셋은 자동 반려 처리하고 Legal Team 에 추가 조사를 요청한다. 조사 미종결 시 학습에 사용 금지.
2. **공개 데이터셋의 라이선스 변경** — nuScenes·KITTI 등 외부 공개 데이터셋의 라이선스 약관이 사후 변경된 경우, 변경 시점 이후 신규 학습에는 사용 금지. 기존 베이스라인은 유지하되 [[PRO-ASPICE-01-11_프로세스개선및재사용]] 에 보고한다.
3. **마스킹 후에도 식별 가능성 잔존** — 얼굴 블러를 적용해도 의복·체형으로 재식별 가능한 경우, 추가 비식별 조치(전신 블러·합성 대체) 적용 후 Privacy Officer 재승인 필요.
4. **국외 이전 데이터** — 데이터 주체가 EU·중국 거주자일 경우 GDPR / PIPL(중국 개인정보보호법) 의 국외 이전 요건(SCC, 표준계약 등)을 충족하지 않으면 학습 사용을 보류한다.
