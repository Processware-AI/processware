# 테일러링 지침 운영 (WI-CMMI-01-02-03)

> 상위 절차: [[PRO-CMMI-01-02_프로세스_자산_개발_절차_v1.0]]

## 1. 업무 목적
프로젝트 특성(규모·도메인·라이프사이클·법규)에 맞춰 OSSP 를 안전하게 테일러링할 수 있는 기준·범위·승인 흐름을 운영하여, 프로젝트 효율과 표준 준수를 동시에 달성한다.

## 2. 수행 주체
- 주 수행자: 프로젝트 PM
- 검토자: SEPG, QA
- 승인자: SEPG Lead

## 3. 범위
- 신규 프로젝트 착수 시(필수) + 중대 변경 시(임시)
- 테일러링 가능 항목: 단계 생략·통합·도구 대체·문서 통합

## 4. 입력 / 산출물
- Input: 프로젝트 헌장, 위험 등록부, OSSP
- Output: [[TMP-CMMI-01-02-03-01_테일러링_요청서_v1.0]] 승인본, 프로젝트 정의 프로세스(PDP)

## 5. 수행 절차

### 5.1 사전 준비
1. 프로젝트 특성 점검표 작성.
2. 테일러링 후보 항목 식별.

### 5.2 수행 단계
1. **요청서 작성** — PM 이 항목·사유·리스크·대안 기재.
2. **SEPG 검토** — 표준 무결성·리스크 평가.
3. **QA 검토** — 검증 가능성 확인.
4. **SEPG Lead 승인** — PDP 등록.
5. **프로젝트 적용** — 프로젝트 계획서에 PDP 명시.
6. **종료 시 리뷰** — 테일러링 효과 회고.

### 5.3 완료 조건
- [ ] 요청서 SEPG Lead 서명본
- [ ] PDP 프로젝트 계획서에 첨부
- [ ] 종료 회고 자료 PAL 등재

## 6. 인터페이스 부서
- 프로젝트 PMO, SEPG, QA

## 7. 주의사항 / 예외 처리

### 7.1 핵심 통제 생략 금지
- QA·CM·VV 핵심 통제는 테일러링 대상 아님.

### 7.2 사후 발견
- 미승인 테일러링이 운영 중 발견되면 30일 내 정식 요청 또는 원복.

### 7.3 표준 변경
- OSSP 가 변경되면 진행 중 프로젝트 PDP 정합성 점검 의무.
