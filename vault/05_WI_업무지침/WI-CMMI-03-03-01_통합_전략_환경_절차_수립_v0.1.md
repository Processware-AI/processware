# 통합 전략·환경·절차 수립 (WI-CMMI-03-03-01)

> 상위 절차: [[PRO-CMMI-03-03_제품_통합_절차]]

## 1. 업무 목적
제품 컴포넌트를 안전·체계적으로 조립·평가·인도하기 위한 통합 전략, 통합 환경, 통합 절차·기준을 수립한다.

## 2. 수행 주체
- **주 수행자**: Integration Lead
- **검토자**: Architect, Engineer, CM, Test Eng
- **승인자**: Project Manager

## 3. 범위
PRO-CMMI-03-03 §5 의 **SP1.1 (Determine Integration Sequence) ~ SP1.2 (Establish the Product Integration Environment) ~ SP1.3 (Establish Product Integration Procedures and Criteria)**.

## 4. 입력 자료 / 산출물
- **Input**
  - 설계 베이스라인 (TS WI-02 TDP)
  - 컴포넌트 목록·인수 일정 (TS WI-04)
  - ICD (TS WI-03)
- **Output**
  - [[TMP-CMMI-03-03-01-01_제품_통합_전략_환경_절차서]]

## 5. 수행 절차

### 5.1 사전 준비
1. TDP·ICD·인수 일정 확보.
2. 통합 환경 가용성 점검 (랙·네트워크·라이선스 등).
3. 관계자(개발팀·CM·시험) 워크숍 일정.

### 5.2 수행 단계
1. **통합 순서 결정 (SP1.1)**
   - Bottom-up / Top-down / Big-bang / Sandwich 중 결정.
   - 의존성 그래프 작성 — 데드락·순환 확인.
2. **통합 환경 수립 (SP1.2)**
   - 하드웨어·소프트웨어·네트워크·테스트 데이터.
   - 모의 외부 시스템(Mock/Stub) 정의.
3. **통합 절차·기준 (SP1.3)**
   - 각 통합 라운드의 진입/종료 기준.
   - 합격/불합격 기준 (smoke/sanity/regression).
   - 부적합 처리 흐름.
4. **검토·승인**

### 5.3 완료 조건 체크리스트
- [ ] 통합 순서 결정 (의존성 그래프 첨부)
- [ ] 통합 환경 명세 (HW/SW/N/W/Data/Mock)
- [ ] 라운드별 진입·종료 기준 표
- [ ] 합격/불합격 판정 기준
- [ ] 부적합 처리 흐름도
- [ ] PM 승인 결재

## 6. 인터페이스 부서
- **Integration 팀**: 주 수행
- **Architect**: 의존성 검토
- **CM**: 베이스라인 통제
- **Test Eng**: 시험 절차 협업 ([[PRO-CMMI-03-04]])

## 7. 주의사항 / 예외 처리

### 7.1 환경 가용성 부족
- 통합 환경이 부족(예: 라이선스 미확보):
  - 우선 영역에 한정 통합 + 나머지는 클라우드 임시.
  - 후속 인수 일정 명시.

### 7.2 의존성 순환
- 컴포넌트 간 순환 의존:
  - 인터페이스 재설계 (TS SP2.3 회귀).
  - 임시 stub 으로 끊어 통합 후 정식 인터페이스.

### 7.3 외부 시스템 미가용
- 외부 시스템(ERP 등) 시험 환경 부재:
  - Mock 서버 도입 + 인터페이스 계약 시험.
  - 정식 환경 시점에 통합 시험 재실시.
