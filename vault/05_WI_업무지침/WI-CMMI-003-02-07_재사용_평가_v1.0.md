# 재사용 평가 (WI-CMMI-003-02-07)

> 상위 절차: [[PRO-CMMI-302_기술솔루션_절차_v1.0]]

## 1. 업무 목적
기존 자산·OSS·상용 컴포넌트의 재사용 가능성·라이선스를 평가하여 비용·기간·리스크 최적화.

## 2. 수행 주체
- 주 수행자: 아키텍트
- 검토자: 법무, 보안팀
- 승인자: PM

## 3. 범위
- 신규 컴포넌트 도입 직전

## 4. 입력 / 산출물
- Input: 후보 컴포넌트
- Output: [[TMP-CMMI-003-02-07_재사용_평가서_v1.0]]

## 5. 수행 절차

### 5.1 사전 준비
1. 후보 인벤토리.
2. 평가 기준.

### 5.2 수행 단계
1. **기능 적합성**.
2. **품질·신뢰성**.
3. **라이선스**.
4. **보안**.
5. **TCO**.
6. **결정**.

### 5.3 완료 조건
- [ ] 평가서 승인본
- [ ] 라이선스 확인
- [ ] 보안 통과

## 6. 인터페이스 부서
- 법무, 보안, 구매

## 7. 주의사항 / 예외 처리

### 7.1 라이선스 충돌
- GPL/카피레프트 등 호환성 검토.

### 7.2 EOL 위험
- EOL 6개월 내 컴포넌트 회피.

### 7.3 보안 취약
- CVE 활성 컴포넌트 회피.
