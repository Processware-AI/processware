# 확인 실행 (WI-CMMI-03-05-02)

> 상위 절차: [[PRO-CMMI-03-05_확인_절차]]

## 1. 업무 목적
확인 계획서에 따라 사용자·이해관계자가 직접 또는 모사 환경에서 제품을 사용·시연·시뮬레이션하여 의도된 사용 충족 여부를 확인한다.

## 2. 수행 주체
- **주 수행자**: Validation Lead + 사용자/이해관계자
- **검토자**: Chief Engineer, Test Engineer
- **승인자**: Project Manager

## 3. 범위
PRO-CMMI-03-05 §5 의 **SP2.1 (Perform Validation)**.

## 4. 입력 자료 / 산출물
- **Input**
  - 확인 계획서 (WI-CMMI-03-05-01)
  - 환경 구축 완료 상태
  - VER 통과 산출물
- **Output**
  - [[TMP-CMMI-03-05-02-01_확인_실행_보고서_교차참조_매트릭스]]

## 5. 수행 절차

### 5.1 사전 준비
1. 환경 가동 점검 (smoke 1회).
2. 참여 사용자 가용성·NDA 확인.
3. 결과 수집 방법(기록·녹화·설문) 준비.

### 5.2 수행 단계
1. **세션 개시**
   - 세션 개요·범위·기록 동의 안내.
2. **사용자 시험·시연**
   - 절차서 따라 사용자 행동 수행.
   - VAL Lead·관찰자가 결과 기록.
   - 사용자 의견(긍정·부정·개선) 별도 캡처.
3. **교차참조 매트릭스 작성**
   - 시험 시나리오 ↔ 고객 요구사항 매핑.
   - 커버리지 점검.
4. **현장 이슈 처리**
   - 세션 중 발견된 차단(blocker) 이슈 즉시 등록.
5. **데이터 정리**
   - 세션별 결과 + NPS / 5점 척도 평균 + 부적합 등록.

### 5.3 완료 조건 체크리스트
- [ ] 모든 계획 절차 실행 (커버리지 ≥ 95%)
- [ ] 시나리오 ↔ CR 교차참조 100%
- [ ] 사용자 의견 기록 (긍정·부정·개선)
- [ ] 세션별 결과·NPS·5점 척도 측정
- [ ] Blocker 이슈 즉시 등록
- [ ] PM 승인 결재 (실행 종료)

## 6. 인터페이스 부서
- **VAL 팀 + 사용자 / 고객**: 주 수행
- **PM**: 진행 모니터링
- **RD / TS / PI Owner**: 부적합 환류 (WI-CMMI-03-05-03 에서)
- **PPQA**: 절차 준수 감사

## 7. 주의사항 / 예외 처리

### 7.1 세션 중 환경 장애
- 환경 장애로 진행 불가:
  - 잔여 시나리오를 재일정.
  - 부분 결과는 별도 기록.

### 7.2 사용자 미참여
- 일부 사용자 미참석:
  - 대리인 + 비동기 회람.
  - 표본 부족 시 결과 일반화 한계 명시.

### 7.3 결정적 부적합 발견
- Critical 부적합 발견:
  - 즉시 진행 일시 중지 + PM 보고.
  - WI-CMMI-03-05-03 환류 절차 우선 가동.
