# 조직 측정저장소 운영 (WI-CMMI-001-02-06)

> 상위 절차: [[PRO-CMMI-102_프로세스_자산_개발_절차_v1.0]]

## 1. 업무 목적
조직 차원의 측정 데이터(역사·기준선·벤치마크) 를 단일 저장소에 적재·관리하여 추정·계획·개선 활동의 근거 자료로 활용한다.

## 2. 수행 주체
- 주 수행자: SEPG 데이터 담당
- 검토자: PCB
- 승인자: SEPG Lead

## 3. 범위
- PA별 측정값(GOV·PLAN·MPM·QA·CM 등), 프로젝트 종료 데이터

## 4. 입력 / 산출물
- Input: 프로젝트 측정값, [[PRO-CMMI-204_성과_및_측정_관리_절차_v1.0]] 산출
- Output: 적재 데이터, 통계 요약, 기준선

## 5. 수행 절차

### 5.1 사전 준비
1. 스키마 문서 확인.
2. 데이터 정의(단위·주기) 일치 검증.

### 5.2 수행 단계
1. **적재 요청 접수** — 프로젝트가 적재요청서 제출.
2. **품질 점검** — 결측·이상치·단위 일관성.
3. **변환·표준화** — 표준 단위·코드로 변환.
4. **저장** — 측정저장소 DB 적재.
5. **공표** — 분기 통계 보고서 게시.
6. **이력 보존** — 5년 보존, 익명화 후 학술 활용 가능.

### 5.3 완료 조건
- [ ] 적재 결과 무결성 점검 통과
- [ ] 분기 통계 게시
- [ ] 데이터 사전(Data Dictionary) 갱신

## 6. 인터페이스 부서
- IT(DB), 각 Process Owner, MPM

## 7. 주의사항 / 예외 처리

### 7.1 데이터 누락
- 프로젝트가 데이터 미제출 시 SEPG 가 PCB 에 보고, 시정조치 발의.

### 7.2 이상치
- 통계적 3시그마 초과 데이터는 별도 분류, 정상 데이터 통계와 구분.

### 7.3 개인정보
- 인력 식별 가능한 데이터는 익명화 후 적재.
