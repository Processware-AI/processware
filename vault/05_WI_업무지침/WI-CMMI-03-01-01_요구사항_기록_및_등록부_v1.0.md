# 요구사항 기록 및 등록부 (WI-CMMI-03-01-01)

> 상위 절차: [[PRO-CMMI-03-01_요구사항_개발_및_관리_절차_v1.0]]

## 1. 업무 목적
요구사항을 등록부에 표준 형식으로 기록·관리하여 추적성·변경관리의 기반을 확보한다.

## 2. 수행 주체
- 주 수행자: RDM PO
- 검토자: 고객
- 승인자: PM

## 3. 범위
- 모든 프로젝트, 요구 식별 즉시

## 4. 입력 / 산출물
- Input: 인터뷰·문서·시스템 분석
- Output: [[TMP-CMMI-03-01-01-01_요구사항_등록부_v1.0]]

## 5. 수행 절차

### 5.1 사전 준비
1. ID 채번 규칙 확인.
2. 분류 체계 확인.

### 5.2 수행 단계
1. **식별**.
2. **속성 기록** — ID·서술·우선순위·소스·상태.
3. **분류**.
4. **검토 의견 반영**.
5. **승인**.
6. **공유**.

### 5.3 완료 조건
- [ ] 등록부 모든 필드 채움
- [ ] 고객 검토 완료
- [ ] PM 승인

## 6. 인터페이스 부서
- 영업, 고객, 아키텍트

## 7. 주의사항 / 예외 처리

### 7.1 모호 요구
- 모호 시 검토자 회의로 명확화.

### 7.2 충돌 요구
- 충돌 시 우선순위 협상.

### 7.3 변경
- 등록 후 변경은 [[PRO-CMMI-04-02_형상관리_절차_v1.0]] CCB.
