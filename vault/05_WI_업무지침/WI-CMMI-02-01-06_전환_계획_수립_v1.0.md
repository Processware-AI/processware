# 전환 계획 수립 (WI-CMMI-02-01-06)

> 상위 절차: [[PRO-CMMI-02-01_프로젝트_계획_절차_v1.0]]

## 1. 업무 목적
프로젝트 산출물을 운영조직에 안전하게 인계하기 위한 교육·문서·병렬운영 계획을 사전 수립한다.

## 2. 수행 주체
- 주 수행자: 프로젝트 PM
- 검토자: 운영팀, 보안팀
- 승인자: 프로젝트 PM

## 3. 범위
- 운영 단계가 있는 모든 프로젝트

## 4. 입력 / 산출물
- Input: 산출물 목록, 운영 절차
- Output: [[TMP-CMMI-02-01-06-01_전환계획서_v1.0]] 승인본

## 5. 수행 절차

### 5.1 사전 준비
1. 운영 인계 항목 식별.
2. 운영팀 사전 협의.

### 5.2 수행 단계
1. **인계 범위 정의** — 코드·문서·계정.
2. **교육 계획** — 운영자 교육.
3. **병렬운영** — 1~3개월.
4. **단계 인계** — 모니터링→대응→소유.
5. **인수 검증** — 운영팀 인수 서명.
6. **종결 후 지원** — 30일 hypercare.

### 5.3 완료 조건
- [ ] 전환계획서 승인본
- [ ] 운영팀 인수 서명
- [ ] hypercare 계획

## 6. 인터페이스 부서
- 운영팀, 고객지원팀, 보안팀

## 7. 주의사항 / 예외 처리

### 7.1 운영팀 미준비
- 30일 전 운영 역량 점검, 미달 시 일정 조정.

### 7.2 긴급 인계
- 긴급 인계 시 hypercare 90일로 연장.

### 7.3 인수 거절
- 운영팀 거절 시 사유·수정안 협의 후 재인계.
