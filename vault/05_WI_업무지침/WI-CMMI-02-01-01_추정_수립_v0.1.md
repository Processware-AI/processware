# 추정 수립 (WI-CMMI-02-01-01)

> 상위 절차: [[PRO-CMMI-02-01_프로젝트_계획_절차]]

## 1. 업무 목적
프로젝트 헌장으로부터 작업분해구조(WBS)를 도출하고, 산출물·작업 속성·라이프사이클 단계를 정의하며, 노력·비용을 정량적으로 추정하여 후속 계획 수립(WI-02)의 기초 데이터를 확보한다.

## 2. 수행 주체
- **주 수행자**: Project Manager
- **검토자**: PMO Director, 도메인 시니어 엔지니어
- **승인자**: CEO/CTO (추정 베이스라인 확정)

## 3. 범위
PRO-CMMI-02-01 §5 흐름의 **SP1.1(범위) ~ SP1.4(노력·비용)** 단계. SG2 (일정·예산·리스크 등) 는 [[WI-CMMI-02-01-02]] 에서 처리.

## 4. 입력 자료 / 산출물
- **Input**
  - 프로젝트 헌장 (kick-off 승인 문서)
  - OSSP 라이프사이클 모델 ([[WI-CMMI-01-01-01]] 산출물)
  - 과거 유사 프로젝트 측정값 (PAL 조회)
  - 고객·내부 요구사항 초안 (RD 인계 또는 헌장 첨부)
- **Output**
  - [[TMP-CMMI-02-01-01-01_WBS_작업패키지_기술서]] (작성본)
  - [[TMP-CMMI-02-01-01-02_추정_근거서]] (작성본)
  - 산출물·작업 속성 추정표 (별표)
  - 라이프사이클 단계 정의서 (별표)

## 5. 수행 절차

### 5.1 사전 준비
1. 프로젝트 헌장 + 헌장 승인 결재 확인.
2. PAL 에서 유사 프로젝트 측정값 (effort, 일정, defect rate) 조회.
3. OSSP 라이프사이클 모델 옵션 (Waterfall/Iterative/Agile/Hybrid) 후보 정리.
4. 추정 워크숍 일정 확정 (필수 참여: PM, 도메인 시니어, EPG 옵저버).

### 5.2 수행 단계
1. **범위 추정 (SP1.1)**
   - 프로젝트 헌장의 목표·산출물을 WBS 로 분해 (최대 5 수준)
   - 각 작업패키지(WP) 에 ID·설명·완료 기준 부여
   - [[TMP-CMMI-02-01-01-01]] WBS 표 작성
2. **산출물·작업 속성 추정 (SP1.2)**
   - 각 WP 별 size 속성 (LOC/FP/Story Point 등) 추정
   - 복잡도·재사용율·기술 친숙도 등 정성 속성 부여
   - 별표: 산출물 속성 추정표
3. **라이프사이클 단계 정의 (SP1.3)**
   - OSSP 모델 후보 중 본 프로젝트 적합 모델 선택
   - 단계 (Phase) · 게이트 (Gate) · 단계별 산출물 명시
   - 별표: 라이프사이클 단계 정의서
4. **노력·비용 추정 (SP1.4)**
   - WBS × 속성 × 라이프사이클 단계 기반 effort 추정
   - 추정 기법 명시 (Wide-band Delphi / Analogy / COCOMO / Story Point velocity 등)
   - 비용 = effort × 단가 + 직접비
   - [[TMP-CMMI-02-01-01-02]] 추정 근거서 작성
5. **추정 검증**
   - 도메인 시니어 review — overrun risk 시그널 검토
   - PAL 측정값 대비 편차 ±30% 초과 시 재추정
6. **승인 및 베이스라인화**
   - 추정 베이스라인을 SG2 계획 수립의 입력으로 인계
   - WI-02 입력 큐 등록

### 5.3 완료 조건
- [ ] WBS 작성 완료 — 모든 작업패키지에 ID·완료기준·예상 effort 부여
- [ ] 산출물 속성 추정표 작성 완료 (size + 정성 속성)
- [ ] 라이프사이클 단계 정의서 작성 완료 (게이트 조건 포함)
- [ ] 추정 근거서 작성 완료 — 사용 기법·가정·근거 데이터 명시
- [ ] 도메인 시니어 review 통과 — overrun risk 시그널 0건
- [ ] PAL 편차 ±30% 이내 또는 초과 시 정당화 근거 첨부

## 6. 인터페이스 부서
- **EPG**: OSSP 라이프사이클 모델 제공, 테일러링 시 자문
- **PAL 운영**: 유사 프로젝트 측정값 제공 ([[WI-CMMI-01-01-03]])
- **MA**: 추정 측정 항목 표준화 ([[PRO-CMMI-04-02]])
- **RD**: 헌장 첨부 요구사항 초안 검토 ([[PRO-CMMI-03-01]])

## 7. 주의사항 / 예외 처리

### 7.1 PAL 데이터 부재 (신규 도메인)
- 유사 측정값 없는 신기술·신도메인의 경우: 외부 벤치마크 데이터 또는 PoC 수행 후 재추정.
- 외부 데이터 사용 시 source_citation 에 출처·라이선스 명시 의무.

### 7.2 추정 편차 초과 (PAL 대비 ±30% 초과)
- 1차: 도메인 시니어 2명 이상 재검토.
- 2차: 그래도 초과 시 PMO Director 결재 + RSKM 리스크 등록 ([[PRO-CMMI-02-06]] SP2.1 입력).
- 3차: ±50% 초과 시 헌장 재검토 요청 (sponsor 에게 escalation).

### 7.3 라이프사이클 모델 미정의 영역
- OSSP 카탈로그 외 모델 필요 시 EPG 에게 임시 모델 승인 요청 ([[WI-CMMI-01-01-01]] §7.3).
- 6개월 내 OSSP 정식 등재 의무.
