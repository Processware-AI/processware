# 프로세스 방향성 수립 및 전달 (WI-CMMI-01-01-01)

> 상위 절차: [[PRO-CMMI-01-01_거버넌스_운영_절차_v1.0]]
> 상위 정책: [[POL-CMMI-01_거버넌스_및_프로세스자산_정책_v1.0]]

## 1. 업무 목적
조직의 비즈니스 전략과 정합되는 **프로세스 방향성**(중요 활동·우선순위·기대 결과)을 고위 경영진이 직접 정의하고 전사에 전달하여, 모든 인력이 동일한 기준으로 프로세스 활동에 임하게 한다.

## 2. 수행 주체
- **주 수행자**: CEO 또는 위임받은 임원(전략기획 임원)
- **검토자**: PCB(Process Control Board)
- **승인자**: CEO
- **지원 조직**: SEPG(초안 작성·문구 정제), QA(이행 확인)

## 3. 범위
- 전사 모든 PA(20종) 의 방향성 선언
- 연 1회 정기 갱신 + 비즈니스 전략 변경 시 임시 갱신
- 자회사·해외법인은 본 방향성을 준용하되 지역 테일러링 가능

## 4. 입력 자료 / 산출물
- **Input**
  - 회사 비즈니스 전략서(연간)
  - 전년도 거버넌스 검토 결과
  - 시장·법규·기술 환경 분석 보고
- **Output**
  - [[TMP-CMMI-01-01-01-01_프로세스_방향성_선언서_v1.0]] 작성본
  - [[POL-CMMI-01_거버넌스_및_프로세스자산_정책_v1.0]] 갱신(필요 시)
  - 사내 공지·CEO 메시지

## 5. 수행 절차

### 5.1 사전 준비
1. SEPG 가 비즈니스 전략 문서·전년도 거버넌스 회의록을 수집한다.
2. 시장·법규·기술 환경의 변화 항목을 표로 정리한다.
3. 작년 방향성과의 차이점(Gap)을 식별한다.

### 5.2 수행 단계
1. **방향성 초안 작성** — SEPG 가 [[TMP-CMMI-01-01-01-01_프로세스_방향성_선언서_v1.0]] 에 비즈니스 정렬·우선 PA·기대 결과·KPI 임계값을 채운다.
2. **경영진 워크숍** — CEO 주재, 임원 전원 참석. 초안을 반영하여 표현·우선순위를 확정한다.
3. **PCB 검토** — 측정 가능성·자원 가용성·리스크 정합성을 확인한다.
4. **CEO 승인** — 최종본에 서명한다. 승인 일자를 frontmatter `updated` 에 기록한다.
5. **전사 전달** — 사내 공지·CEO 메시지·전사 미팅 3개 채널로 전달한다. 핵심 인력에는 1:1 브리핑을 추가한다.
6. **이해도 점검** — 전달 후 2주 내 핵심 인력 대상 5문항 자가진단을 실시한다(목표 이해도 ≥ 90%).

### 5.3 완료 조건
- [ ] 방향성 선언서 CEO 서명본 보관
- [ ] [[POL-CMMI-01_거버넌스_및_프로세스자산_정책_v1.0]] 의 §1 목적·§2 범위 갱신 반영
- [ ] 사내 공지 게시 + 전사 미팅 의사록 보존
- [ ] 핵심 인력 이해도 ≥ 90% 검증
- [ ] [[MAT-001_문서관리대장]] Row 갱신

## 6. 인터페이스 부서
- **CEO Office**: 워크숍 주재
- **SEPG**: 초안·문구 정제
- **PCB**: 검토
- **HR**: 사내 전달 채널
- **QA**: 이행 확인

## 7. 주의사항 / 예외 처리

### 7.1 비즈니스 전략 미확정
- 전략이 미확정 상태이면 방향성 갱신을 보류한다. 전년 방향성을 잠정 유지하되 PCB 가 분기 검토 시 격차 점검을 수행한다.

### 7.2 긴급 환경 변화
- 법규 변경·중대 보안 사고·M&A 등이 발생한 경우 임시 워크숍을 30일 내 개최하여 부분 갱신한다.

### 7.3 경영진 의견 충돌
- CEO 가 단독 결정권을 가진다. 충돌 사항은 회의록 부속서에 보존한다.
