# 리스크 관리 준비 (WI-CMMI-02-06-01)

> 상위 절차: [[PRO-CMMI-02-06_리스크_관리_절차]]

## 1. 업무 목적
프로젝트의 리스크 관리 기반을 마련한다 — 리스크 출처·카테고리를 정의하고, 평가 파라미터 (확률·영향·임계치) 를 결정하고, 리스크 관리 전략을 수립한다.

## 2. 수행 주체
- **주 수행자**: Risk Manager
- **검토자**: Project Manager, 도메인 시니어
- **승인자**: Senior Management (전략서)

## 3. 범위
PRO-CMMI-02-06 §5 의 **SG1 SP1.1~1.3**.

## 4. 입력 자료 / 산출물
- **Input**: 프로젝트 헌장, 조직 RSKM 정책, 과거 RSKM PAL 자산
- **Output**:
  - [[TMP-CMMI-02-06-01-01_리스크_출처_카테고리]]
  - [[TMP-CMMI-02-06-01-02_리스크_파라미터_정의서]]
  - [[TMP-CMMI-02-06-01-03_리스크_관리_전략서]]

## 5. 수행 절차

### 5.1 사전 준비
1. 프로젝트 헌장 + 조직 RSKM 정책 입수.
2. 도메인 시니어 자문 일정.

### 5.2 수행 단계
1. **출처·카테고리 결정 (SP1.1)** — 요구사항·기술·일정·외주·환경·인적·법규 등 출처 분류.
2. **파라미터 정의 (SP1.2)** — 확률 (H/M/L 또는 0~1), 영향 (H/M/L 또는 비용·일정 수치), 임계치 (escalate 기준).
3. **전략 수립 (SP1.3)** — 식별·분석·완화 절차, 검토 주기, 책임 매트릭스, 도구 정의.

### 5.3 완료 조건
- [ ] 출처·카테고리 목록 (≥ 5 카테고리)
- [ ] 파라미터 정의서 (확률·영향·임계치)
- [ ] 전략서 작성·승인
- [ ] Senior Mgmt 결재
- [ ] CM 등록

## 6. 인터페이스 부서
- **PMO**: 정책 정합성 검토
- **CM**: 전략서 베이스라인 등록
- **PP**: PP SP2.2 리스크 식별 입력 ([[WI-CMMI-02-01-02]])

## 7. 주의사항 / 예외 처리

### 7.1 도메인 특수 출처
- 헬스/금융/방산 등 도메인은 법규·규제 출처가 강조 필요 — 별도 카테고리 추가.

### 7.2 파라미터 정량화 어려움
- 정량 데이터 없을 시 정성 (H/M/L) 시작 → 6개월 후 정량 전환 시도.

### 7.3 전략 갱신 시점
- 베이스라인 변경 또는 분기 1회 정기 review.
- 빈번 갱신 (월 ≥ 1회) 시 본 WI 자동화 도구 검토.
