# 운영개념 및 시나리오 개발 (WI-CMMI-003-01-08)

> 상위 절차: [[PRO-CMMI-301_요구사항_개발_및_관리_절차_v1.0]]

## 1. 업무 목적
실제 운영에서의 사용 흐름·이상 흐름·예외 시나리오를 명문화하여 요구·설계·검증의 기반으로 사용한다.

## 2. 수행 주체
- 주 수행자: RDM PO
- 검토자: 고객, UX
- 승인자: PM

## 3. 범위
- 모든 신규 시스템

## 4. 입력 / 산출물
- Input: 요구·도메인 자료
- Output: [[TMP-CMMI-003-01-08_운영개념_시나리오서_v1.0]]

## 5. 수행 절차

### 5.1 사전 준비
1. 사용자 페르소나 정의.
2. 환경 가정.

### 5.2 수행 단계
1. **정상 시나리오**.
2. **이상 시나리오**.
3. **예외 시나리오**.
4. **고객 검증**.
5. **승인**.
6. **공유**.

### 5.3 완료 조건
- [ ] 시나리오서 승인본
- [ ] 고객 확인
- [ ] 검증 케이스 입력

## 6. 인터페이스 부서
- UX, 운영

## 7. 주의사항 / 예외 처리

### 7.1 페르소나 누락
- 5명 이내 핵심 페르소나 필수.

### 7.2 표현 다의
- 다이어그램·예시.

### 7.3 변경
- 운영 변경 시 시나리오 갱신.
