# 양방향 추적성 관리 (WI-CMMI-003-01-05)

> 상위 절차: [[PRO-CMMI-301_요구사항_개발_및_관리_절차_v1.0]]

## 1. 업무 목적
요구사항 → 설계 → 코드 → 테스트 → 인수 의 양방향 추적성을 매트릭스로 운영하여 누락·과잉 산출물을 방지한다.

## 2. 수행 주체
- 주 수행자: RDM PO
- 검토자: QA
- 승인자: PM

## 3. 범위
- 모든 프로젝트, 단계별 갱신

## 4. 입력 / 산출물
- Input: 등록부, 설계·테스트 산출물
- Output: [[TMP-CMMI-003-01-05_추적성_매트릭스_v1.0]]

## 5. 수행 절차

### 5.1 사전 준비
1. 도구 선택(JIRA·DOORS).
2. ID 규약 확인.

### 5.2 수행 단계
1. **링크 생성**.
2. **자동·수동 검증**.
3. **고립 항목 식별**.
4. **시정**.
5. **베이스라인 시 점검**.
6. **변경 시 재검증**.

### 5.3 완료 조건
- [ ] 매트릭스 무결성 검증
- [ ] 고립 항목 0건
- [ ] 베이스라인 시 점검 완료

## 6. 인터페이스 부서
- 설계, 개발, QA

## 7. 주의사항 / 예외 처리

### 7.1 자동 누락
- 도구 자동화율 < 95% 시 수동 보완.

### 7.2 고립 항목
- 발견 시 24h 내 분석.

### 7.3 ID 충돌
- 중복 ID 즉시 변경.
