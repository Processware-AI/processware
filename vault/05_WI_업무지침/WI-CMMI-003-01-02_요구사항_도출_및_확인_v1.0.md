# 요구사항 도출 및 확인 (WI-CMMI-003-01-02)

> 상위 절차: [[PRO-CMMI-301_요구사항_개발_및_관리_절차_v1.0]]

## 1. 업무 목적
이해관계자로부터 요구사항을 체계적으로 도출하고 의도와 맞는지 확인한다.

## 2. 수행 주체
- 주 수행자: RDM PO
- 검토자: 고객
- 승인자: PM

## 3. 범위
- 모든 프로젝트, 단계별 갱신

## 4. 입력 / 산출물
- Input: 비즈니스 케이스
- Output: [[TMP-CMMI-003-01-02_요구사항_도출_워크숍_보고서_v1.0]]

## 5. 수행 절차

### 5.1 사전 준비
1. 이해관계자 식별.
2. 도출 기법 선택(인터뷰·워크숍·관찰·프로토타입).

### 5.2 수행 단계
1. **워크숍 진행**.
2. **요구 후보 수집**.
3. **명확화**.
4. **고객 확인**.
5. **등록**.
6. **변경 알람**.

### 5.3 완료 조건
- [ ] 보고서 승인본
- [ ] 고객 서명
- [ ] 등록부 갱신

## 6. 인터페이스 부서
- 영업, UX

## 7. 주의사항 / 예외 처리

### 7.1 도출 부족
- 추가 워크숍 일정.

### 7.2 모호
- 프로토타입으로 확인.

### 7.3 변경
- CCB 적용.
