# 기본 통합 수행 (WI-CMMI-03-03-01)

> 상위 절차: [[PRO-CMMI-03-03_제품통합_절차_v1.0]]

## 1. 업무 목적
컴포넌트 단위 통합을 표준 절차로 수행하여 통합 결함을 조기 발견한다.

## 2. 수행 주체
- 주 수행자: Integration Lead
- 검토자: QA
- 승인자: PM

## 3. 범위
- 모든 빌드 통합

## 4. 입력 / 산출물
- Input: 컴포넌트, 설계
- Output: 통합 빌드, [[TMP-CMMI-03-03-01-01_통합_체크리스트_v1.0]]

## 5. 수행 절차

### 5.1 사전 준비
1. 환경 확인.
2. CI 파이프라인 점검.

### 5.2 수행 단계
1. **사전 검증**.
2. **결합**.
3. **스모크 테스트**.
4. **이슈 식별**.
5. **시정**.
6. **베이스라인**.

### 5.3 완료 조건
- [ ] 스모크 통과
- [ ] 이슈 종결
- [ ] 베이스라인

## 6. 인터페이스 부서
- 개발, QA, CM

## 7. 주의사항 / 예외 처리

### 7.1 빌드 실패
- 즉시 차단·롤백.

### 7.2 회귀 결함
- 회귀 케이스 추가.

### 7.3 환경 차이
- 표준 이미지로 재현.
