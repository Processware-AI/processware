# 배포 모니터링·경험 통합 (WI-CMMI-01-02-04)

> 상위 절차: [[PRO-CMMI-01-02_조직_프로세스_개선_배포_절차]]

## 1. 업무 목적
배포된 OSSP·OPA가 프로젝트에서 실제로 적용되는지 모니터링하고, PPQA 컴플라이언스 감사 결과·프로젝트 측정값·교훈을 수집하여 OPA에 환류하여 지속적 개선 사이클을 닫는다.

## 2. 수행 주체
- **주 수행자**: EPG Lead
- **검토자**: PPQA Lead, Measurement Analyst
- **승인자**: CEO/CTO (분기 모니터링 결과 보고 승인)

## 3. 범위
PRO-CMMI-01-02 §5 의 **SP3.3(실행 모니터링) ~ SP3.4(경험 통합)** 단계. 본 결과는 차기 사이클 SP1.1 의 입력으로 환류된다.

## 4. 입력 자료 / 산출물
- **Input**
  - 배포 가이드·발효 OSSP·테일러링 기록부
  - PPQA 컴플라이언스 감사 결과 ([[PRO-CMMI-04-03]])
  - 프로젝트 측정 데이터 ([[PRO-CMMI-04-02]] MA)
  - 프로젝트 회고·종결 보고서
- **Output**
  - [[TMP-CMMI-01-02-04-01_배포_실행_모니터링_보고서]] 작성본 (분기)
  - [[TMP-CMMI-01-02-04-02_교훈_등록부]] 갱신본
  - OPA 갱신 권고 입력 (차기 SP1.3)

## 5. 수행 절차

### 5.1 사전 준비
1. 배포 완료 후 1분기 경과 시점 기준 모니터링 시작.
2. PPQA 감사 일정·결과·NCR 목록 확보.
3. 분기 측정 데이터 추출 (PAL 측정저장소).

### 5.2 수행 단계
1. **OSSP 적용 모니터링** (SP3.3)
   - 프로젝트별 OSSP 적용 여부·테일러링 합리성 검토.
   - 적용률(active project 중 적용 비율) 산정.
2. **프로젝트 산출물 표본 검토** (SP3.3)
   - 프로젝트별 핵심 산출물(계획서·요구사항 명세서 등)을 표본 검토.
   - OSSP 부합 여부 확인.
3. **PPQA 감사 결과 통합** (SP3.3)
   - PPQA NCR 목록 수집·분류·종결 추적.
   - 반복 발생 NCR 패턴 분석.
4. **이슈 식별·문서화·종결 추적** (SP3.3)
   - 미준수·결함 식별 → 시정 요청 → 종결까지 추적.
   - [[TMP-CMMI-01-02-04-01]] 양식 활용.
5. **OSSP·OPA 효과성 정기 검토** (SP3.4)
   - 프로세스 성능 목표 대비 실측값 비교 (분기·반기).
   - 적합성·효과성 판단 보고.
6. **교훈 도출 및 등록** (SP3.4)
   - 프로젝트 회고·종결 보고서에서 교훈 추출.
   - 일반화 가능한 교훈을 [[TMP-CMMI-01-02-04-02]] 등록부에 등재.
   - 접근권한 부여 (PAL 공개).
7. **개선 제안 관리** (SP3.4)
   - 프로젝트·이해관계자의 개선 제안을 수집·분류·평가.
   - 채택된 제안은 차기 SP1.3 개선 후보로 입력.
8. **결과 보고 및 환류** (SP3.4)
   - 분기 보고서 CEO/CTO 보고.
   - 차기 사이클 SP1.1 입력으로 자동 전달.

### 5.3 완료 조건
- [ ] 분기 모니터링 보고서 작성·승인 완료
- [ ] PPQA NCR 통합·종결 상태 추적 완료
- [ ] 표본 검토 결과 + OSSP 부합률 산정
- [ ] OSSP·OPA 효과성 평가 결과 등재 (분기 1회 이상)
- [ ] 교훈 등록부 갱신 (분기 최소 5건 이상)
- [ ] 개선 제안 분류·평가 + 차기 사이클 입력 인계
- [ ] CM 베이스라인 + MAT-001 등록

## 6. 인터페이스 부서
- **선행 WI**: [[WI-CMMI-01-02-03]] 배포 완료 인수
- **환류 WI (차기 사이클)**: [[WI-CMMI-01-02-01]] SP1.1 의 입력
- **PPQA**: 감사 결과 공급 ([[PRO-CMMI-04-03]])
- **MA**: 측정 데이터 공급 ([[PRO-CMMI-04-02]])
- **프로젝트**: 회고·교훈 공급
- **OT**: 교훈 기반 교육 니즈 갱신 ([[PRO-CMMI-01-03]] SP1.1)

## 7. 주의사항 / 예외 처리

### 7.1 컴플라이언스 부합률 90% 미달
- KPI 미달 시 즉시 PCB 안건 상정.
- 미준수 프로젝트 별도 개입 + 시정 액션 플랜 별도 수립.

### 7.2 교훈 등록 부진 (분기 5건 미만)
- 프로젝트 회고 회의 의무화 + EPG 가 사례 발굴 워크숍 개최.
- OT 와 연계하여 교훈 공유 세미나 분기 1회.

### 7.3 모니터링 데이터 누락·품질 이슈
- MA 측정저장소 데이터 결손 발견 시 [[WI-CMMI-01-01-03]] §7.2 절차로 정규화.
- 누락 분기 보고서는 다음 분기에 통합 보고.
