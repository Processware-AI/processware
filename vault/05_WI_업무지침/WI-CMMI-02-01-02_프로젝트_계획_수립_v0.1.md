# 프로젝트 계획 수립 — 일정·예산·리스크·데이터관리 (WI-CMMI-02-01-02)

> 상위 절차: [[PRO-CMMI-02-01_프로젝트_계획_절차]]

## 1. 업무 목적
WI-01 의 추정 베이스라인을 입력으로 받아 일정·예산·프로젝트 리스크·데이터관리 계획을 수립하여 통합 계획서의 핵심 4 축을 작성한다.

## 2. 수행 주체
- **주 수행자**: Project Manager
- **검토자**: PMO Director, Risk Manager (리스크 부분), Data Custodian (데이터관리 부분)
- **승인자**: CEO/CTO

## 3. 범위
PRO-CMMI-02-01 §5 의 **SP2.1(예산·일정) ~ SP2.4(자원)** 중 SP2.1~2.3 (일정·리스크·데이터). 자원·지식·이해관계자는 [[WI-CMMI-02-01-03]] 에서.

## 4. 입력 자료 / 산출물
- **Input**
  - [[WI-CMMI-02-01-01]] 산출물: WBS, 추정 근거서
  - RSKM 전략서 ([[WI-CMMI-02-06-01]] 산출물)
  - 조직 데이터관리 정책 ([[POL-CMMI-04_지원_품질보증_정책]])
- **Output**
  - [[TMP-CMMI-02-01-02-01_일정_예산_계획서]]
  - [[TMP-CMMI-02-01-02-02_프로젝트_리스크_목록]]
  - [[TMP-CMMI-02-01-02-03_데이터관리_계획서]]

## 5. 수행 절차

### 5.1 사전 준비
1. WI-01 산출 WBS·추정 베이스라인 확보 (entry_gate 충족).
2. RSKM 전략서 (리스크 파라미터·임계치) 확보.
3. 조직 데이터관리 정책·개인정보 보호 요건 확인.

### 5.2 수행 단계
1. **일정 수립 (SP2.1)**
   - WBS × effort × 자원 가용성 기반 일정 스케줄링 (Gantt 또는 백로그)
   - 마일스톤·게이트 일정 명시
   - 임계 경로(Critical Path) 식별
2. **예산 수립 (SP2.1)**
   - 단계별 비용 분배 (S-curve 또는 단계별 cost plan)
   - 예비비 (contingency reserve) ≥ 10% 설정
3. **프로젝트 리스크 식별 (SP2.2)**
   - WBS·일정·예산 검토 시 발견된 리스크 등록
   - [[TMP-CMMI-02-01-02-02]] 작성
   - RSKM SP2.1 입력 큐로 전달 ([[WI-CMMI-02-06-02]])
4. **데이터관리 계획 (SP2.3)**
   - 보유 데이터 유형 식별 (요구사항·설계·시험·고객PII 등)
   - 분류·보관 기간·접근 권한·폐기 정책 명시
   - [[TMP-CMMI-02-01-02-03]] 작성
5. **검토 및 베이스라인 후보화**
   - PMO + Risk Manager + Data Custodian review
   - WI-03 입력 큐로 인계

### 5.3 완료 조건
- [ ] 일정·예산 계획서 작성 — 임계 경로 식별·예비비 ≥ 10% 포함
- [ ] 프로젝트 리스크 목록 ≥ 10건 등록 (Top-N high-priority 명시)
- [ ] 데이터관리 계획서 — 데이터 유형별 보관·접근·폐기 명시
- [ ] 모든 출력이 RSKM·CM·MA 인터페이스 부서 통보 완료
- [ ] PMO 검토 의견 전체 해소

## 6. 인터페이스 부서
- **RSKM**: 리스크 목록 수신 ([[PRO-CMMI-02-06]] SP2.1)
- **MA**: 일정 진척·예산 EVM 측정 항목 등록 ([[PRO-CMMI-04-02]])
- **CM**: 데이터관리 계획서 베이스라인 등록 ([[PRO-CMMI-04-01]])
- **Sponsor**: 예산 한도 협의

### 7.1 일정 단축 요구 (Sponsor escalation)
- Sponsor 가 일정 단축 요구 시: critical path 영향분석 후 RSKM 등록.
- Crashing/Fast-tracking 적용 시 비용 증가 한도 ≤ 20% 권고.

### 7.2 예비비 부족
- 추정 불확실성 H 영역이 30% 초과면 예비비 ≥ 15% 로 상향.
- 예비비 사용 시 PMC SP2.2 (시정조치) 와 연결하여 사용 사유 기록.

### 7.3 데이터 보호 강화 요건 (개인정보·민감 데이터)
- 개인정보/PII 포함 시 별도 보안 계획 첨부 + 법무 검토 필수.
- 외부 표준 (GDPR/개인정보보호법) 요건 충족 체크리스트 추가.
