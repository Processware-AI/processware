# 인터페이스 요구사항 관리 (WI-CMMI-03-01-10)

> 상위 절차: [[PRO-CMMI-03-01_요구사항_개발_및_관리_절차_v1.0]]

## 1. 업무 목적
내·외부 인터페이스 요구사항을 별도 명세화·통제하여 통합 시 발생하는 호환성 문제를 사전 방지한다.

## 2. 수행 주체
- 주 수행자: 아키텍트
- 검토자: RDM PO, 외부 인터페이스 책임자
- 승인자: PM

## 3. 범위
- 모든 외부 시스템 인터페이스, 주요 내부 모듈 간 인터페이스

## 4. 입력 / 산출물
- Input: 시스템 아키텍처
- Output: [[TMP-CMMI-03-01-10-01_인터페이스_요구명세서_v1.0]]

## 5. 수행 절차

### 5.1 사전 준비
1. 인터페이스 인벤토리.
2. 외부 협력사 점검.

### 5.2 수행 단계
1. **인터페이스 식별**.
2. **명세 작성**.
3. **외부 협의**.
4. **버전 관리**.
5. **호환 검증**.
6. **변경 통제**.

### 5.3 완료 조건
- [ ] 명세서 승인본
- [ ] 외부 합의 서명
- [ ] 추적성 등록

## 6. 인터페이스 부서
- 외부 벤더, IT

## 7. 주의사항 / 예외 처리

### 7.1 외부 변경
- 외부 변경 시 30일 사전 통보 요구.

### 7.2 보안
- 보안 인터페이스는 별도 보호.

### 7.3 호환 깨짐
- 즉시 통합 정지·시정.
