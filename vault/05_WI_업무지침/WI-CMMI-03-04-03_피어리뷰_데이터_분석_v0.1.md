# 피어리뷰 데이터 분석 (WI-CMMI-03-04-03)

> 상위 절차: [[PRO-CMMI-03-04_검증_절차]]

## 1. 업무 목적
누적된 피어리뷰 데이터(시간·결함밀도·등급분포·재리뷰율)를 분석하여 트렌드를 도출하고, 프로세스 개선·교육 영역을 식별한다.

## 2. 수행 주체
- **주 수행자**: Verification Lead 또는 MA Owner
- **검토자**: EPG, Chief Engineer
- **승인자**: Project Manager (프로젝트 보고용), EPG Lead (조직 보고용)

## 3. 범위
PRO-CMMI-03-04 §5 의 **SP2.3 (Analyze Peer Review Data)**.

## 4. 입력 자료 / 산출물
- **Input**
  - 피어리뷰 기록부 누적 (WI-CMMI-03-04-02 산출)
  - 조직 기준선(과거 평균·산업 벤치마크)
- **Output**
  - [[TMP-CMMI-03-04-03-01_피어리뷰_결함_분석_보고서]]
  - EPG 측 개선 권고 (OPF 입력)

## 5. 수행 절차

### 5.1 사전 준비
1. 모든 세션 기록부 수집.
2. 조직 기준선·과거 평균 수치 확보.

### 5.2 수행 단계
1. **메트릭 집계**
   - 시간·LOC/페이지·결함 수·결함 밀도·등급 분포·재리뷰율.
   - 산출물 유형별 분리(요구·설계·코드·문서).
2. **트렌드 분석**
   - 시계열 그래프 (최근 5분기 등).
   - 기준선 대비 편차.
3. **결함 유형 분류**
   - 결함 카테고리(논리·성능·보안·테스트성·문서 등) 비율.
4. **개선 영역 식별**
   - 빈번 결함 유형 → 교육·체크리스트 갱신·툴 도입 권고.
5. **보고**
   - 프로젝트 PM 결재 + EPG 인계 (OPF 개선 입력).

### 5.3 완료 조건 체크리스트
- [ ] 모든 세션 데이터 집계
- [ ] 산출물 유형별 분리 메트릭
- [ ] 시계열 그래프
- [ ] 기준선 대비 편차
- [ ] 결함 유형 분류 비율
- [ ] 개선 권고 항목 1개 이상
- [ ] PM 결재 + EPG 인계 송부

## 6. 인터페이스 부서
- **VER / MA 팀**: 주 수행
- **EPG**: 개선 권고 수신 ([[PRO-CMMI-01-02]] SP1)
- **OT**: 교육 영역 입력 ([[PRO-CMMI-01-03]])
- **PPQA**: 절차 준수

## 7. 주의사항 / 예외 처리

### 7.1 데이터 부족
- 세션 수 < 5건 등 통계적 의의 부족:
  - 정성 분석 위주 + 다음 분기 누적 후 정량 분석.

### 7.2 메트릭 정의 변경
- 메트릭 정의가 분기 사이에 변경된 경우:
  - 시계열을 재산정하지 말고 변경점 표기.
  - 변경 사유·결정 기록.

### 7.3 외주·외부 리뷰 데이터
- 외부 리뷰 데이터 포함 여부:
  - 별도 분리 표기 (조직 평균에 영향 가능).
  - 외부 리뷰 비교는 정성으로 한정.
