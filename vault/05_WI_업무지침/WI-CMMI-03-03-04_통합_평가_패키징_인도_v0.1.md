# 통합 평가·패키징·인도 (WI-CMMI-03-03-04)

> 상위 절차: [[PRO-CMMI-03-03_제품_통합_절차]]

## 1. 업무 목적
조립된 제품을 평가하고, 패키징(릴리스 자료)을 작성하여 고객에게 인도한다. 인도 후 결함 모니터링 책임도 포함한다.

## 2. 수행 주체
- **주 수행자**: Integration Lead
- **검토자**: Chief Engineer, CM, QA, 고객
- **승인자**: Project Manager + 고객 (Accept)

## 3. 범위
PRO-CMMI-03-03 §5 의 **SP3.3 (Evaluate Assembled Product Components) ~ SP3.4 (Package and Deliver the Product or Product Component)**.

## 4. 입력 자료 / 산출물
- **Input**
  - 최종 통합 베이스라인 (WI-CMMI-03-03-03)
  - 통합 절차·기준 (WI-CMMI-03-03-01)
  - 지원 문서 패키지 (TS WI-04)
- **Output**
  - [[TMP-CMMI-03-03-04-01_통합_평가_패키징_인도_보고서]]
  - 릴리스 패키지 (CM 베이스라인)
  - 인도 확인서 (고객 수령 서명)

## 5. 수행 절차

### 5.1 사전 준비
1. VER 통과 산출물 확인 ([[PRO-CMMI-03-04]] 와 병행 가능).
2. 패키징 도구·릴리스 파이프라인 점검.
3. 인도 환경(고객 측 환경) 확인.

### 5.2 수행 단계
1. **통합 평가 (SP3.3)**
   - 통합 절차 기준에 따른 평가 시험 (smoke + sanity + 회귀).
   - VER 결과·VAL 결과 종합.
   - 비기능(성능·보안) 시험 결과 반영.
2. **릴리스 노트 작성**
   - 변경 사항·기능·결함 수정·알려진 이슈·업그레이드 안내.
3. **패키징 (SP3.4)**
   - 릴리스 아티팩트(코드·SBOM·매뉴얼·라이선스) 묶음.
   - 보안 서명 + 무결성 해시.
4. **인도 (SP3.4)**
   - 고객 환경 배포 또는 인도.
   - 인수 시험(UAT) 지원.
5. **인도 확인서**
   - 고객 수령 서명·전자결재 첨부.
6. **사후 모니터링**
   - 인도 후 30일 결함 모니터링 책임.

### 5.3 완료 조건 체크리스트
- [ ] 통합 평가 합격(smoke/sanity/회귀 모두 Pass)
- [ ] 비기능 시험 결과 반영
- [ ] 릴리스 노트 작성 (변경·기능·결함·알려진이슈·업그레이드)
- [ ] 패키지 무결성 해시·보안 서명
- [ ] CM 릴리스 베이스라인 등록
- [ ] 고객 인수 시험(UAT) 지원·결과 첨부
- [ ] 고객 인도 확인서 (서명/결재)
- [ ] 인도 후 30일 모니터링 책임자 지정

## 6. 인터페이스 부서
- **Integration 팀**: 주 수행
- **CM**: 릴리스 베이스라인 등록 ([[PRO-CMMI-04-01]])
- **VER / VAL**: 결과 공급 ([[PRO-CMMI-03-04]], [[PRO-CMMI-03-05]])
- **PPQA**: 절차 준수 감사 ([[PRO-CMMI-04-03]])
- **고객**: 인수 시험·인도 확인

## 7. 주의사항 / 예외 처리

### 7.1 인수 시험 불합격 (UAT Fail)
- 고객 UAT 미통과:
  - 차감 항목 정리 → 영향평가 → 시정 → 재UAT.
  - 일정 영향 시 PM + 고객 합의.

### 7.2 잠재 결함 다수 인지
- 알려진 이슈가 영향도 High 인 경우:
  - 인도 보류 또는 조건부 인도 (mitigation 명시).
  - 고객 동의·서명 필수.

### 7.3 환경 호환성 문제
- 고객 환경과 패키지 비호환:
  - 환경 매트릭스 점검 → 패치 또는 환경 변경.
  - 인도 일정 재조정.
