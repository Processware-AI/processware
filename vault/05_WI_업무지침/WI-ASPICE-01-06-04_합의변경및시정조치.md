# WI-ASPICE-01-06-04 — 합의 변경 및 시정 조치 (ACQ.4)

## 1. 업무 목적
계약 이탈·성과 미달·중대 결함이 탐지된 경우 공급사에 시정조치(SCAR, Supplier Corrective Action Request)를 발행하고 근본 원인 분석·시정 계획·검증을 추적하여 재발을 방지한다. ASPICE 4.0 ACQ.4 의 변경·시정 활동을 충족한다.

## 2. 수행 주체
- **주 수행자**: Supply Chain Engineer
- **검토자**: Project Manager / QA / Legal
- **승인자**: Procurement Manager

## 3. 범위
- 트리거: 성과 보고서 Yellow/Red, 중대 결함, 계약 의무 위반, 보안 사고
- 활동: SCAR 발행 → 공급사 RCA → 시정조치 → 검증 → 종결
- 제외: 일상적 산출물 결함은 SUP.9 결함관리로 처리

## 4. 입력 자료 / 산출물
**입력 자료**
- 이탈 증적 (성과 보고서·결함·감사 결과)
- 계약서 / SOW / SLA
- 공급사 의사소통 채널·담당자 정보

**산출물**
- 시정조치 요구서(TMP-ASPICE-01-06-04-01)
- 공급사 근본 원인 분석 보고서 (8D/5-Why)
- 시정조치 계획 (공급사 작성)
- 검증 결과 / 종결 확인서
- (필요 시) 계약 변경 합의서

## 5. 수행 절차

### 5.1 사전 준비
1. 이탈 사실의 객관적 증적(데이터·로그·문서) 확보.
2. 계약상 SCAR 발행 권한 및 응답 기한 확인.
3. 영향 분석(일정/품질/비용) 사전 수행.
4. 법무 사전 검토 (계약 위반 수준 판단).

### 5.2 수행 단계
1. **SCAR 발행** (ACQ.4 BP7) — 시정조치 요구서를 공식 채널로 공급사에 발행한다.
2. **공급사 RCA 수령** — 공급사가 8D 또는 5-Why 로 근본 원인을 분석·제출한다(통상 10영업일).
3. **시정조치 계획 검토** (ACQ.4 BP8) — 시정조치의 실효성·일정·재발 방지 적정성을 검토한다.
4. **시정조치 실행 모니터링** — 마일스톤 별 진행 점검.
5. **검증** — 시정조치 결과의 효과를 측정 가능한 기준으로 검증한다.
6. **종결 확인서** — 효과 입증 후 SCAR 종결 확인서를 발행한다.
7. **계약 변경(필요 시)** — 시정조치 결과 계약 변경이 필요하면 변경 합의서를 체결.
8. **레슨드런 등록** — 공급망 SUP.7 lessons learned 등록.

### 5.3 완료 조건 체크리스트
- [ ] 이탈 증적이 객관적으로 기록되었다.
- [ ] SCAR 가 공식 채널로 발행되고 공급사 수령 확인이 보존되었다.
- [ ] 공급사 RCA 가 수령·검토되었다.
- [ ] 시정조치 계획·일정·완료 기준이 합의되었다.
- [ ] 시정조치 결과가 측정 가능한 기준으로 검증되었다.
- [ ] 종결 확인서가 발행되었다.
- [ ] 필요 시 계약 변경이 체결되었다.
- [ ] 레슨드런이 등록되었다.
- [ ] [[MAT-001_문서관리대장]] 갱신 완료.

## 6. 인터페이스 부서
- 법무팀: 계약 위반 판정·변경 합의서 작성
- QA: RCA 적정성 감사·검증 독립성
- 프로젝트 관리: 일정/리스크 통합 관리
- 보안팀: 보안 사고 SCAR 의 추가 조치
- CM: 변경 산출물·합의서 베이스라인 관리

## 7. 주의사항 / 예외 처리

### 7.1 공급사 응답 거부 또는 지연
공식 응답 기한 초과 시 1차 독촉, 2차 에스컬레이션(공급사 임원), 3차 계약 위반 통지 절차를 따른다.

### 7.2 RCA 부실 (피상적 원인)
RCA 가 표면 원인만 식별한 경우 재제출을 요구하며, QA 가 적정성 판정 권한을 가진다.

### 7.3 동일 결함 재발
시정조치 종결 후 동일/유사 결함이 재발하면 즉시 신규 SCAR + 계약 변경(예: 페널티·공급선 다변화) 검토.

### 7.4 보안 사고 SCAR
보안 사고는 별도 보안 인시던트 대응 절차와 병행하며, SCAR 의 RCA 도 보안팀 합동 검토를 거친다.
