# 작업 접근방식 정의 (WI-CMMI-002-01-02)

> 상위 절차: [[PRO-CMMI-201_프로젝트_계획_절차_v1.0]]

## 1. 업무 목적
프로젝트 특성에 적합한 라이프사이클·방법론·도구를 정의하여 작업 접근의 일관성을 확보한다.

## 2. 수행 주체
- 주 수행자: 프로젝트 PM
- 검토자: PMO, SEPG
- 승인자: 프로젝트 PM

## 3. 범위
- 모든 신규 프로젝트

## 4. 입력 / 산출물
- Input: 프로젝트 헌장, OSSP, 테일러링 가이드
- Output: [[TMP-CMMI-002-01-02_프로젝트_접근방식서_v1.0]] 승인본

## 5. 수행 절차

### 5.1 사전 준비
1. 프로젝트 특성 점검(규모·도메인·법규).
2. OSSP 라이프사이클 옵션 검토.

### 5.2 수행 단계
1. **라이프사이클 선택** — Waterfall/Iterative/Agile 등.
2. **방법론 결정** — Scrum/Kanban/SAFe 등.
3. **도구 세트** — IDE/CI/이슈관리 등.
4. **테일러링 요청** — [[WI-CMMI-001-02-03_테일러링_지침_운영_v1.0]] 적용.
5. **접근방식서 작성** — 결정·근거 기록.
6. **승인 및 배포** — 팀 공유.

### 5.3 완료 조건
- [ ] 접근방식서 승인본
- [ ] 테일러링 등록(필요 시)
- [ ] 팀 공유 의사록

## 6. 인터페이스 부서
- SEPG, PMO, IT

## 7. 주의사항 / 예외 처리

### 7.1 라이프사이클 변경
- 진행 중 변경은 PM 승인 + PMO 통보.

### 7.2 도구 미가용
- 라이선스 부족 시 대체 도구 + 보안 검토.

### 7.3 외부 강제
- 고객 강제 라이프사이클은 사유 기록.
