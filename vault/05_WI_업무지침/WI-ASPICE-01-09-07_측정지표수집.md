# 측정 지표 수집 (WI-ASPICE-01-09-07)

> 상위 절차: [[PRO-ASPICE-01-09_프로젝트관리프로세스]]

## 1. 업무 목적
GQM(Goal-Question-Metric) 기반으로 정의된 KPI/지표를 정기적으로 수집하고 데이터 품질을 검증한 뒤 측정 DB에 적재하여, 의사결정·트렌드 분석의 신뢰 가능한 원천을 제공한다. ASPICE 4.0 MAN.6 BP1(측정 정보 요구 정의), BP2(측정 절차 정의), BP3(데이터 수집)에 부합한다.

## 2. 수행 주체
- **주수행자**: Project Manager
- **검토자**: QA Manager, PMO
- **승인자**: Program Director

## 3. 범위
프로젝트 전 기간의 일정·품질·자원 KPI를 포함한다. 도구 자동 수집을 우선하고, 수기 수집은 자동화가 불가한 항목으로 제한한다.

## 4. 입력 자료 / 산출물
- **Input**: GQM 목표 정의서, 지표 카탈로그, 도구 데이터(Jira, SonarQube, TestRail, Polarion), 수기 수집 양식
- **Output**: 측정 지표 수집 양식(TMP-ASPICE-01-09-07-01), 측정 DB 적재 로그

## 5. 수행 절차 (단계별)

### 5.1 사전 준비
1. GQM 목표 정의서와 지표 카탈로그 최신본 확인.
2. 도구별 API/Export 권한 점검(Jira, SonarQube, TestRail).
3. 측정 DB 적재 스크립트(또는 ETL 파이프라인) 정상 동작 확인.

### 5.2 수행 단계 (ASPICE BP 참조)
1. **BP1 측정 정보 요구 검증** — GQM 목표가 변경되었는지(단계 전환·고객 요구 변동) 점검.
2. **BP2 절차 적용** — 지표별 수집원·수집 주기·단위·산출 공식을 카탈로그대로 적용.
3. **BP3 데이터 수집** — 자동(API)/수기 방법으로 측정값 수집, 수집 일시 기록.
4. **데이터 품질 확인** — 완전성(필수 지표 누락), 정확성(이상치/단위 오류), 적시성(주차 마감일 준수) 3축 검증.
5. **이상값 처리** — 단위 오류는 수정, 결측은 사유 기재 후 재수집 일정 등록, 통계 이상치는 원인 추적.
6. **측정 DB 적재** — 검증 완료 데이터를 측정 DB에 적재, 적재 로그 보존.
7. **수집 양식 작성·검토** — 검토자(QA/PMO) 확인 후 적재 완료 표시.

### 5.3 완료 조건 체크리스트
- [ ] GQM 목표 변경 사항이 반영되었는가
- [ ] 정의된 모든 지표가 수집되었는가(누락 없음)
- [ ] 데이터 품질 3축(완전성/정확성/적시성) 검증이 기록되었는가
- [ ] 이상값 처리 결과가 기재되어 있는가
- [ ] 측정 DB 적재 로그가 보존되어 있는가
- [ ] [[MAT-001_문서관리대장]] 갱신이 완료되었는가

## 6. 인터페이스 부서
- 도메인 팀(SW/HW/QA) — 도구 사용·수기 수집 협조
- IT/DevOps — 도구 API·DB 운영
- PMO — 지표 카탈로그·GQM 목표 관리

## 7. 주의사항 / 예외 처리
1. **도구 장애 시**: 수동 백업 수집 양식으로 전환, 도구 복구 후 차이 검증.
2. **결측 데이터**: 추정값 사용 금지, 결측 사유와 재수집 일정을 양식에 기재.
3. **단위 혼동(예: KLoC vs LoC)**: 카탈로그 단위 표기를 강제, 단위 미일치 시 적재 거부.
4. **개인정보 포함 지표**: 익명화·집계 후 적재(개별 식별 금지).
