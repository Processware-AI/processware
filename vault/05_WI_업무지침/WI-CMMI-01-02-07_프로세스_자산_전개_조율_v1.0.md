# 프로세스 자산 전개 조율 (WI-CMMI-01-02-07)

> 상위 절차: [[PRO-CMMI-01-02_프로세스_자산_개발_절차_v1.0]]

## 1. 업무 목적
개발/개정된 자산이 실무에 정착되도록 단계적 전개 계획·교육·피드백·정착 평가를 통합 운영하여, 자산이 PAL 에 등재된 후에도 사장되지 않도록 한다.

## 2. 수행 주체
- 주 수행자: SEPG 전개 담당
- 검토자: OT, PCB
- 승인자: SEPG Lead

## 3. 범위
- 신규/개정 OSSP, 영향받는 모든 부서·프로젝트

## 4. 입력 / 산출물
- Input: 신규 자산, 변경 영향 범위
- Output: [[TMP-CMMI-01-02-07-01_자산_전개계획서_v1.0]], 전개 보고

## 5. 수행 절차

### 5.1 사전 준비
1. 영향받는 부서·프로젝트 식별.
2. 교육 콘텐츠 초안 작성.

### 5.2 수행 단계
1. **전개 계획 수립** — 단계·일정·교육·평가 지표.
2. **파일럿 운영** — 1~2 부서 시범 적용.
3. **피드백 수집** — 양식·절차 개선점.
4. **전사 전개** — 단계별 확산.
5. **교육 시행** — [[PRO-CMMI-05-01_조직_훈련_절차_v1.0]] 와 연동.
6. **정착 평가** — 90일 후 사용률·이해도·만족도.

### 5.3 완료 조건
- [ ] 전개 계획 승인본
- [ ] 파일럿 결과 보고
- [ ] 교육 이수율 ≥ 90%
- [ ] 90일 정착 평가 보고

## 6. 인터페이스 부서
- OT, 각 부서, QA

## 7. 주의사항 / 예외 처리

### 7.1 저항
- 변경 저항 발생 시 변화관리 워크숍, 1:1 면담.

### 7.2 정착 실패
- 90일 사용률 < 70% 시 자산 재설계 검토.

### 7.3 우선순위 충돌
- 동시 다수 전개 시 PCB 가 우선순위 결정.
