# 운영 개념·기능·품질 정의 (WI-CMMI-03-01-03)

> 상위 절차: [[PRO-CMMI-03-01_요구사항_개발_절차]]

## 1. 업무 목적
요구사항을 실 운영 환경에서 사용·운용하기 위한 **운영 개념(Operational Concept)**, **시나리오·use case**, **기능 정의**, **품질 속성**을 도출하고, 요구사항을 분석하여 결함을 식별·제거한다.

## 2. 수행 주체
- **주 수행자**: Requirements Engineer, Engineer/Architect
- **검토자**: Chief Engineer, Test Engineer, VAL Lead
- **승인자**: Project Manager

## 3. 범위
PRO-CMMI-03-01 §5 의 **SP3.1 (Establish Operational Concepts and Scenarios) ~ SP3.2 (Establish a Definition of Required Functionality and Quality Attributes) ~ SP3.3 (Analyze Requirements)** 단계.

## 4. 입력 자료 / 산출물
- **Input**
  - 고객 요구사항 명세서 (WI-CMMI-03-01-01)
  - 제품·컴포넌트 요구사항 명세서 (WI-CMMI-03-01-02)
  - 환경 정보(현장 조사, 운영 매뉴얼 등)
- **Output**
  - [[TMP-CMMI-03-01-03-01_운영_개념_시나리오_정의서]]
  - [[TMP-CMMI-03-01-03-02_기능_품질속성_정의서]]
  - 요구사항 분석 결함 보고

## 5. 수행 절차

### 5.1 사전 준비
1. 제품 요구사항 베이스라인 확보 (WI-02 산출).
2. 운영 환경 조사 — 현장 방문·관찰 일정 확정.
3. 시나리오 작성 워크숍 일정 확정 (고객·사용자 대표 포함).

### 5.2 수행 단계
1. **운영 개념 수립 (SP3.1)**
   - 제품이 작동할 환경·사용자·운영 모드(정상/예외/실패) 기술.
   - 라이프사이클 단계별(설치·운영·유지보수·폐기) 운영 개념 분리.
2. **운영 시나리오·use case 작성**
   - 정상 시나리오 + 대안 시나리오 + 예외 시나리오 3개 묶음 단위.
   - 각 use case 에 actor·전제·트리거·기본 흐름·대안 흐름·종료조건 명시.
3. **기능 정의 (SP3.2 - 기능 부분)**
   - 시나리오를 기능 목록으로 정제.
   - 각 기능을 IPO(Input/Process/Output) 단위로 분해.
4. **품질 속성 정의 (SP3.2 - 품질 부분)**
   - 성능·신뢰성·가용성·보안·유지보수성·이식성·확장성·접근성 등 항목 평가.
   - 각 항목에 정량 목표(예: 가용성 99.9%) 부여.
5. **요구사항 분석 (SP3.3)**
   - 완전성 · 일관성 · 검증가능성 · 실현가능성 · 추적성 5축 점검.
   - 모호 / 누락 / 중복 / 모순 결함 식별 → 결함 보고서.
6. **결함 처리**
   - Open 결함을 분류 (해결/이관-WI-04 균형/Drop).
   - 해결 결함은 WI-01/02 산출물 갱신.
7. **검토·승인**
   - Chief Eng + Test Eng + VAL Lead 합동 검토.
   - PM 승인.

### 5.3 완료 조건 체크리스트
- [ ] 운영 개념 라이프사이클 4단계 모두 기술
- [ ] 시나리오 묶음(정상/대안/예외) 모든 핵심 기능에 작성
- [ ] 기능 IPO 분해 완료
- [ ] 품질 속성 8축 평가 + 적용 항목 정량 목표
- [ ] 분석 5축 점검 완료 (결함 보고서 발행)
- [ ] Open 결함 처리 상태 기록 (해결/이관/Drop)
- [ ] PM 승인 결재

## 6. 인터페이스 부서
- **요구사항 엔지니어링팀**: 주 수행
- **Test Engineer**: 시나리오·기능 정의의 시험 가능성 검토 ([[PRO-CMMI-03-04]])
- **VAL Lead**: 시나리오 운영 적합성 검토 ([[PRO-CMMI-03-05]])
- **고객·사용자 대표**: 시나리오 검증 참여
- **CM**: 베이스라인 등록

## 7. 주의사항 / 예외 처리

### 7.1 운영 환경 접근 불가
- 현장 방문이 불가한 경우(보안·원거리 등):
  - 영상·사진·매뉴얼·고객 인터뷰로 대체.
  - 대체 사용 시 신뢰도 등급 + PM 결재.

### 7.2 품질 속성 측정 불가
- 정량 목표 부여가 곤란한 품질 속성(예: 신규 도메인):
  - 정성 → 정량 마일스톤 (예: 베타 후 측정 후 정량화) 도입.
  - 마일스톤 일자·책임자 명시.

### 7.3 결함 다발 발생
- 결함이 요구사항 수의 20% 초과:
  - WI-CMMI-03-01-01 elicit 재실시 또는 WI-02 PR 재정렬 권고.
  - 무리한 진행 시 후속 단계 위험 증대 → PM 의사결정 필수.
