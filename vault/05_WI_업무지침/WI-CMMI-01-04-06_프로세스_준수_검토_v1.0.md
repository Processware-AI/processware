# 프로세스 준수 검토 (WI-CMMI-01-04-06)

> 상위 절차: [[PRO-CMMI-01-04_구현_인프라_운영_절차_v1.0]]

## 1. 업무 목적
조직 전체의 프로세스 준수도를 반기 단위로 감사·보고하여 거버넌스에 입력한다.

## 2. 수행 주체
- 주 수행자: QA Lead
- 검토자: SEPG, PCB
- 승인자: CEO

## 3. 범위
- 반기 1회 + 임시(중대 부적합 다발 시)

## 4. 입력 / 산출물
- Input: QA 부적합 데이터, 내재화 점검, 시정조치
- Output: [[TMP-CMMI-01-04-06-01_프로세스_준수_검토_보고서_v1.0]] 승인본

## 5. 수행 절차

### 5.1 사전 준비
1. 검토 범위·기준 확정.
2. 데이터 수집 일정.

### 5.2 수행 단계
1. **데이터 통합** — QA·내재화·시정조치.
2. **준수도 산출** — PA별 점수화.
3. **격차 분석** — 부서·PA별.
4. **권고 작성** — 표준 갱신·교육·시정.
5. **PCB·CEO 보고**.
6. **차기 입력** — 거버넌스 회의 의제 등재.

### 5.3 완료 조건
- [ ] 보고서 승인본
- [ ] 거버넌스 회의 의제 등재
- [ ] 권고사항 백로그 등재

## 6. 인터페이스 부서
- QA, SEPG, PCB, 각 부서

## 7. 주의사항 / 예외 처리

### 7.1 데이터 무결성
- 시스템 로그·증빙 부재 시 부서 자체 응답 점수 가중치 하향.

### 7.2 외부 감사 결과
- 외부 감사 보고서 결과를 본 보고서에 통합 반영.

### 7.3 비공개 사안
- 법적 분쟁 등 비공개 사안은 별도 부속서로 분리.
