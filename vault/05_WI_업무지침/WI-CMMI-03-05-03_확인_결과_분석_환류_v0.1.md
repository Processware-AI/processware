# 확인 결과 분석·환류 (WI-CMMI-03-05-03)

> 상위 절차: [[PRO-CMMI-03-05_확인_절차]]

## 1. 업무 목적
확인 실행 결과를 종합 분석하여 의도 충족 여부를 판정하고, 미충족·부분 충족 항목을 RD/TS/PI 로 customer needs feedback 으로 환류한다.

## 2. 수행 주체
- **주 수행자**: Validation Lead
- **검토자**: Chief Engineer, PM, RD/TS/PI Owner
- **승인자**: Project Manager + 고객 (최종 수락)

## 3. 범위
PRO-CMMI-03-05 §5 의 **SP2.2 (Analyze Validation Results)**.

## 4. 입력 자료 / 산출물
- **Input**
  - 확인 실행 보고서 (WI-CMMI-03-05-02)
  - 사용자 의견 / NPS / 5점 척도 데이터
- **Output**
  - [[TMP-CMMI-03-05-03-01_확인_결과_분석_환류_보고서]]
  - RD/TS/PI 환류 변경요청 (CR)
  - 인도 진행 여부 결정

## 5. 수행 절차

### 5.1 사전 준비
1. 실행 보고서 + 사용자 의견 데이터 정리.
2. 합격 기준·임계값(NPS·5점) 재확인.

### 5.2 수행 단계
1. **결과 종합·판정**
   - 충족/부분/미충족 분류 + 충족율·NPS 산출.
   - 정량 기준 (예: NPS ≥ 40 + 충족율 ≥ 80%) 대비 판정.
2. **결함·니즈 환류**
   - 미충족·부분 충족 항목을 책임 부서(RD/TS/PI)별 분류.
   - 변경요청(CR) 발행 — customer needs feedback 으로 표기.
3. **의도 충족 종합 판정**
   - Pass / Conditional Pass / Fail 결정.
   - Conditional Pass 의 경우 조건·마일스톤 명시.
4. **인도 진행 결정**
   - Pass → 인도 진행 ([[PRO-CMMI-03-03]] WI-04 SP3.4 인계).
   - Conditional → 인도 + 후속 조치 합의.
   - Fail → 인도 보류 + 재VAL.
5. **고객 수락**
   - 고객 측 수락 서명·결재 확보.

### 5.3 완료 조건 체크리스트
- [ ] 충족/부분/미충족 분류 + 충족율·NPS 산출
- [ ] 정량 기준 대비 판정
- [ ] 미충족·부분 충족 항목 CR 발행
- [ ] RD/TS/PI 환류 송부 기록
- [ ] 종합 판정 (Pass/Conditional/Fail)
- [ ] 인도 진행 결정
- [ ] 고객 수락 서명·결재 (Pass·Conditional)
- [ ] PM 승인 결재

## 6. 인터페이스 부서
- **VAL 팀**: 주 수행
- **RD / TS / PI Owner**: customer needs feedback 수신
- **PI Lead**: 인도 진행 시 인계 ([[PRO-CMMI-03-03]] WI-04)
- **고객**: 수락 서명
- **PPQA**: 절차 준수 감사

## 7. 주의사항 / 예외 처리

### 7.1 정량 기준 미달 (Fail)
- NPS·충족율 임계값 미달:
  - 인도 보류 + 근본 원인 분석 (CAR).
  - 재VAL 마일스톤 명시.

### 7.2 사용자 의견 vs 정량 결과 충돌
- 정량은 충족이나 사용자 부정 의견 다수:
  - 정성 분석 강화 — UX·사용성 관점.
  - 추가 라운드 또는 부분 수정 후 인도.

### 7.3 환류 후 변경 영향
- 환류된 CR 이 기존 베이스라인에 크게 영향:
  - 일정·비용 재계획 (PP).
  - PM + 고객 + 경영진 합의 후 진행.
