# 검증 실행·결과 분석 (WI-CMMI-03-04-04)

> 상위 절차: [[PRO-CMMI-03-04_검증_절차]]

## 1. 업무 목적
검증 계획서에 따라 시험·분석·검사·시연을 실행하고, 결과를 분석하여 부적합을 식별·환류한다.

## 2. 수행 주체
- **주 수행자**: Verification Lead, Test Engineer
- **검토자**: Chief Engineer, RD/TS Owner (부적합 피드백 수신)
- **승인자**: Project Manager

## 3. 범위
PRO-CMMI-03-04 §5 의 **SP3.1 (Perform Verification) ~ SP3.2 (Analyze Verification Results)**.

## 4. 입력 자료 / 산출물
- **Input**
  - 검증 계획서 (WI-CMMI-03-04-01)
  - 검증 환경 (구축 완료)
  - 검증 대상 베이스라인
- **Output**
  - [[TMP-CMMI-03-04-04-01_검증_결과_보고서]] (결과 + 분석 + 부적합)

## 5. 수행 절차

### 5.1 사전 준비
1. 검증 계획서 + 환경 헬스체크.
2. 시험 데이터 셋업 + 검증 도구 가동 확인.

### 5.2 수행 단계
1. **검증 실행 (SP3.1)**
   - 절차서 따라 시험/분석/검사/시연 수행.
   - 결과 자동 수집 (CI/CD 통합 권장).
2. **결과 분석 (SP3.2)**
   - 결과 vs 기대 비교.
   - 부적합 식별·등급 부여 (Critical/High/Med/Low).
3. **부적합 환류**
   - 책임 부서(RD/TS/PI) 별 분류 → 시정 요청.
   - 결함 추적 ID 할당.
4. **재검증**
   - 시정 완료 시 재시험.
5. **VAL 인계 준비**
   - VER 통과 산출물을 VAL ([[PRO-CMMI-03-05]]) 입력으로 송부.

### 5.3 완료 조건 체크리스트
- [ ] 모든 계획 절차 실행 (커버리지 100%)
- [ ] 결과 자동 수집·증빙 첨부
- [ ] 부적합 등급 부여·할당
- [ ] 시정 완료(또는 PM 결재 보류) — Critical/High 0건
- [ ] 재검증 결과 첨부
- [ ] VAL 인계 송부 기록
- [ ] PM 승인 결재

## 6. 인터페이스 부서
- **VER 팀**: 주 수행
- **RD / TS / PI Owner**: 부적합 시정 수신
- **VAL Lead**: 통과 산출물 수신 ([[PRO-CMMI-03-05]])
- **CM**: 베이스라인 통제
- **PPQA**: 절차 준수 감사

## 7. 주의사항 / 예외 처리

### 7.1 환경 불안정
- 시험 중 환경 장애 다발:
  - 재시험 횟수 기준 적용 (예: 동일 시나리오 3회 재시도 후 보류).
  - 환경 안정화 후 회귀 시험.

### 7.2 결과 비결정성 (Flaky)
- 동일 시나리오 결과가 매번 다름:
  - 시험 격리 점검 (병렬·외부 의존 등).
  - 격리 후 안정화될 때까지 결과 미합산.

### 7.3 Critical 부적합 다수
- Critical 등급 부적합이 10건 이상:
  - 즉시 PM 보고 + 진행 일시 중지.
  - 근본 원인 분석 (CAR) 가동 — [[PRO-CMMI-04-04]] DAR 또는 CAR 연계.
