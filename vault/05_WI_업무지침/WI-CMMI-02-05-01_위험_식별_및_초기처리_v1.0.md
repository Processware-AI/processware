# 위험 식별 및 초기처리 (WI-CMMI-02-05-01)

> 상위 절차: [[PRO-CMMI-02-05_리스크_및_기회_관리_절차_v1.0]]

## 1. 업무 목적
새로운 위험·기회를 식별하고 초기 분류·등록·통보를 신속히 수행한다.

## 2. 수행 주체
- 주 수행자: Risk Manager (또는 PM 위임)
- 검토자: PMO
- 승인자: PM

## 3. 범위
- 모든 진행 프로젝트, 신규 발견 즉시

## 4. 입력 / 산출물
- Input: 회의·이슈·외부 정보
- Output: [[TMP-CMMI-02-05-01-01_위험_초기등록서_v1.0]]

## 5. 수행 절차

### 5.1 사전 준비
1. 위험 분류 체계 확인.
2. 등록부 접근 확보.

### 5.2 수행 단계
1. **식별** — 워크숍·체크리스트.
2. **초기 분류** — 카테고리·중요도.
3. **즉시 통보** — High 등급은 24h.
4. **등록**.
5. **임시 대응** — 필요 시.
6. **상세 분석 회부**.

### 5.3 완료 조건
- [ ] 등록서 보관
- [ ] High 등급 24h 내 통보
- [ ] 등록부 갱신

## 6. 인터페이스 부서
- PMO, Sponsor

## 7. 주의사항 / 예외 처리

### 7.1 식별 누락
- 사후 발견 시에도 등록 의무.

### 7.2 외부 정보
- 외부 보안·시장 정보 모니터링.

### 7.3 기회 식별
- 위험뿐 아니라 기회도 동일 절차.
