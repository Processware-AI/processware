# 신기술 및 도구 평가 (WI-CMMI-001-03-02)

> 상위 절차: [[PRO-CMMI-103_프로세스_관리_및_개선_절차_v1.0]]

## 1. 업무 목적
프로세스 효율·품질을 향상시킬 수 있는 신기술·도구를 체계적으로 평가하여 비즈니스 가치 검증 후 도입한다.

## 2. 수행 주체
- 주 수행자: CTO 또는 위임 기술팀
- 검토자: SEPG, IT
- 승인자: CEO

## 3. 범위
- AI/자동화/CI-CD/보안 도구 등

## 4. 입력 / 산출물
- Input: 격차 보고, 외부 트렌드 보고
- Output: [[TMP-CMMI-001-03-02_신기술_평가보고서_v1.0]], 도입 결정문

## 5. 수행 절차

### 5.1 사전 준비
1. 후보 발굴(컨퍼런스·논문·벤더 RFI).
2. 평가 기준 사전 정의.

### 5.2 수행 단계
1. **PoC 계획** — 범위·기간·평가지표.
2. **PoC 실행** — 4~8주 시범 운영.
3. **결과 분석** — 비용·효과·리스크.
4. **DAR 적용** — 중대 도입은 [[PRO-CMMI-404_의사결정_분석_및_해결_절차_v1.0)]] 적용.
5. **CEO 결정** — 도입 또는 보류.
6. **도입 시 표준 갱신** — [[WI-CMMI-001-02-05_작업환경_표준_관리_v1.0]] 연동.

### 5.3 완료 조건
- [ ] 평가보고서 승인본
- [ ] 도입 결정문
- [ ] 표준 작업환경 갱신(도입 시)

## 6. 인터페이스 부서
- IT, 구매팀, 보안팀

## 7. 주의사항 / 예외 처리

### 7.1 라이선스·보안
- 도입 전 라이선스·보안 검토 필수.

### 7.2 PoC 실패
- 실패 사유·대안 후보 보고.

### 7.3 벤더 락인
- 단일 벤더 종속 위험 시 대안 보유 후 도입.
