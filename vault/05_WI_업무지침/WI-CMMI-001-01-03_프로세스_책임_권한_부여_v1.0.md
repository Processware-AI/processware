# 프로세스 책임 권한 부여 (WI-CMMI-001-01-03)

> 상위 절차: [[PRO-CMMI-101_거버넌스_운영_절차_v1.0]]

## 1. 업무 목적
프로세스 활동의 의사결정·이행·검증 책임이 누구에게 있는지를 RACI 매트릭스로 명시·발령하여, 책임 공백 없이 프로세스가 운영되도록 한다.

## 2. 수행 주체
- **주 수행자**: HR Director, SEPG Lead
- **검토자**: PCB
- **승인자**: CEO

## 3. 범위
- 전사 PA(20종) 의 거버넌스 역할(Process Owner·검토자·승인자)
- 프로젝트 차원 R&R 은 [[PRO-CMMI-201_프로젝트_계획_절차_v1.0]] 별도 정의

## 4. 입력 자료 / 산출물
- **Input**: 조직도, 직무기술서, 자원요청서 승인본
- **Output**: [[TMP-CMMI-001-01-03_역할_권한_RACI_v1.0]] 승인본, 인사 발령 문서

## 5. 수행 절차

### 5.1 사전 준비
1. 현 RACI 와 직무기술서의 일치성 점검.
2. 결원·중복·과부하 항목 식별.

### 5.2 수행 단계
1. **RACI 초안 작성** — SEPG 가 PA별 단계와 역할을 매트릭스화한다.
2. **HR 검토** — 직무기술서·인사 발령 정합성 확인.
3. **PCB 심의** — 권한 충돌·견제 원칙 확인(예: QA 와 Process Owner 분리).
4. **CEO 승인** — 인사 발령으로 효력 발생.
5. **전사 공지** — 사내 인트라넷·조직도 갱신.
6. **온보딩 연동** — [[PRO-CMMI-501_조직_훈련_절차_v1.0]] 신규 입사자 교육 콘텐츠 갱신.

### 5.3 완료 조건
- [ ] RACI CEO 서명본 보관
- [ ] 인사 발령 문서 발행
- [ ] 조직도·인트라넷 갱신
- [ ] 신규 입사자 교육 콘텐츠 반영
- [ ] [[MAT-001_문서관리대장]] Row 갱신

## 6. 인터페이스 부서
- **HR**: 인사 발령
- **법무**: 권한 위임 적법성
- **IT**: 인트라넷·시스템 권한 동기화

## 7. 주의사항 / 예외 처리

### 7.1 권한 충돌
- 동일인이 R 과 A 를 동시에 갖지 않도록(견제 원칙) 분리. 불가피 시 PCB 가 보완통제 정의.

### 7.2 결원
- Process Owner 결원 시 30일 내 직무대행 지정. 90일 초과 시 정식 채용·발령.

### 7.3 조직 개편
- 본부 통폐합 등으로 RACI 가 무효화될 경우 30일 내 갱신 의무.
