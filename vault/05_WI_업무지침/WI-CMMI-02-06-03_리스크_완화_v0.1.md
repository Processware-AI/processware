# 리스크 완화 (WI-CMMI-02-06-03)

> 상위 절차: [[PRO-CMMI-02-06_리스크_관리_절차]]

## 1. 업무 목적
임계치 초과 리스크에 대한 완화 계획 (mitigation + contingency)을 수립하고 실행하여 발생 확률·영향을 감소시킨다.

## 2. 수행 주체
- **주 수행자**: Risk Manager (계획), PM (실행)
- **검토자**: 도메인 엔지니어
- **승인자**: Senior Management (Critical 완화 계획)

## 3. 범위
PRO-CMMI-02-06 §5 의 **SG3 SP3.1~3.2**.

## 4. 입력 자료 / 산출물
- **Input**: WI-02 우선순위 결과
- **Output**: [[TMP-CMMI-02-06-03-01]], [[TMP-CMMI-02-06-03-02]], [[TMP-CMMI-02-06-03-03_비상계획서]] (Critical 리스크 시)

## 5. 수행 절차

### 5.1 사전 준비
1. 임계치 초과 리스크 목록.

### 5.2 수행 단계
1. **완화 계획 수립 (SP3.1)** — 발생 확률 감소·영향 감소 활동 + 비상 계획 (contingency) + 발동 조건 (trigger).
2. **완화 계획 실행 (SP3.2)** — 책임자·자원 할당, 정기 status 추적.
3. **리스크 status 갱신** — Active/Mitigated/Triggered/Closed.
4. **종결** — 완화 후 우선순위 재평가; Low/Medium 으로 강하 시 모니터링 큐.

### 5.3 완료 조건
- [ ] 임계치 초과 리스크 100% 완화 계획 보유
- [ ] Critical 완화 계획 Senior Mgmt 결재
- [ ] 실행 추적 (status 갱신)
- [ ] 종결 시 우선순위 재평가
- [ ] PAL 등록 (완화 노하우)

## 6. 인터페이스 부서
- **PM**: 완화 실행 책임 (RACI)
- **PMC**: 시정조치와 연계 ([[WI-CMMI-02-02-03]])
- **MA**: 완화 측정값 ([[PRO-CMMI-04-02]])
- **EPG/PAL**: 완화 노하우 PAL 등록

## 7. 주의사항 / 예외 처리

### 7.1 완화 비용 > 리스크 영향
- 비용 비효율 시 acceptance (수용) 결정 가능 — Senior Mgmt 승인 필요.

### 7.2 비상 계획 발동 (Triggered)
- 즉시 PMC SP2 시정조치 분기.
- 발동 후 root cause 분석 → 향후 RSKM 개선.

### 7.3 완화 효과 미달
- 정기 review 에서 효과 미달 발견 시: 완화 계획 강화 또는 acceptance.
