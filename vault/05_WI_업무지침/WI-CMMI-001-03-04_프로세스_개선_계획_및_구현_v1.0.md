# 프로세스 개선 계획 및 구현 (WI-CMMI-001-03-04)

> 상위 절차: [[PRO-CMMI-103_프로세스_관리_및_개선_절차_v1.0]]

## 1. 업무 목적
백로그에 등재된 개선 항목 중 실행할 항목의 계획서를 작성하고 단계적으로 구현·검증한다.

## 2. 수행 주체
- 주 수행자: 개선 PM(SEPG 또는 Process Owner)
- 검토자: PCB, QA
- 승인자: CEO(중대) / SEPG Lead(경미)

## 3. 범위
- 백로그 우선순위 High/Med 항목

## 4. 입력 / 산출물
- Input: 개선기회 등록부, 자원·일정 가용성
- Output: [[TMP-CMMI-001-03-04_개선_계획서_v1.0]] 승인본, 구현 결과 보고

## 5. 수행 절차

### 5.1 사전 준비
1. 백로그 검토.
2. 자원·일정 협의.

### 5.2 수행 단계
1. **계획 작성** — 목표·범위·일정·자원·KPI.
2. **승인** — 중대=CEO, 경미=SEPG Lead.
3. **파일럿** — 1~2 부서 시범.
4. **전사 구현** — 단계적 확산.
5. **검증** — 목표 KPI 달성 확인.
6. **종결** — 등록부 종결 처리, 교훈 PAL 등재.

### 5.3 완료 조건
- [ ] 계획서 승인본
- [ ] KPI 검증 보고
- [ ] 교훈 PAL 등재

## 6. 인터페이스 부서
- PCB, QA, OT, 각 부서

## 7. 주의사항 / 예외 처리

### 7.1 KPI 미달
- 1차 미달 시 원인 분석·계획 수정. 2차 미달 시 종결 또는 재기획 결정.

### 7.2 구현 지연
- 일정 30% 초과 시 PCB 보고, 자원 재배정.

### 7.3 교훈 누락
- 종결 전 교훈 등재 필수, 미등재 시 종결 보류.
