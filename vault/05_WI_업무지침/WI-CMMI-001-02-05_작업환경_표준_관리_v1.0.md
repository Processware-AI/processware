# 작업환경 표준 관리 (WI-CMMI-001-02-05)

> 상위 절차: [[PRO-CMMI-102_프로세스_자산_개발_절차_v1.0]]

## 1. 업무 목적
프로젝트가 일관 환경에서 작업하도록 표준 도구·라이선스·구성을 정의·갱신·배포하여, 환경 차이로 인한 결함과 재작업을 줄인다.

## 2. 수행 주체
- 주 수행자: IT Director
- 검토자: SEPG, QA, 보안팀
- 승인자: CEO

## 3. 범위
- 개발·테스트·문서화·CI/CD·이슈관리 도구 표준 세트

## 4. 입력 / 산출물
- Input: 라이선스 현황, 도구 평가 보고, 보안 요구사항
- Output: [[TMP-CMMI-001-02-05_표준_작업환경_정의서_v1.0]] 승인본, 표준 이미지

## 5. 수행 절차

### 5.1 사전 준비
1. 현 도구 사용 현황 조사.
2. 라이선스·보안 점검.

### 5.2 수행 단계
1. **표준 정의서 작성** — 도구·버전·라이선스·구성.
2. **SEPG 검토** — 프로세스 정합.
3. **보안 검토** — 보안 정책 충족.
4. **CEO 승인** — 표준 이미지 배포.
5. **온보딩 연동** — 신규 입사자 자동 배포.
6. **연 1회 갱신** — 신기술 평가 결과 반영.

### 5.3 완료 조건
- [ ] 정의서 승인본
- [ ] 표준 이미지 배포 확인
- [ ] [[MAT-001_문서관리대장]] Row

## 6. 인터페이스 부서
- IT, 보안팀, 구매팀, OT

## 7. 주의사항 / 예외 처리

### 7.1 라이선스 만료
- 만료 30일 전 갱신 협상 시작, 미갱신 시 대체 도구 평가.

### 7.2 도구 추가 요청
- [[WI-CMMI-001-03-02_신기술_및_도구_평가_v1.0]] 와 연동.

### 7.3 보안 취약점
- CVE 발견 시 24시간 내 패치 또는 대체.
