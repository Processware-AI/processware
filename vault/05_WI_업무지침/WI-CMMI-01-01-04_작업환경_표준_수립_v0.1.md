# 작업환경 표준 수립 (WI-CMMI-01-01-04)

> 상위 절차: [[PRO-CMMI-01-01_조직_표준프로세스_수립_유지_절차]]

## 1. 업무 목적
조직 내 모든 프로젝트가 사용할 공통 작업환경(개발·빌드·테스트·문서·협업·보안 도구 스택과 인프라 표준)을 평가·채택·유지하여, 일관된 생산성·재현성·도구 호환성·라이선스 통제 기반을 제공한다.

## 2. 수행 주체
- **주 수행자**: EPG (작업환경 표준 운영)
- **검토자**: Process Owner, IT/DevOps Lead, 보안 책임자
- **승인자**: 경영진 (CEO/CTO)

## 3. 범위
PRO-CMMI-01-01 §5 의 **SP1.6 (Establish Work Environment Standards)** 단계에 적용. 상업적 가용 표준 평가, 조직 채택, 격차 영역 신규 정의 모두를 포함. 프로젝트별 환경 변형은 [[WI-CMMI-01-01-02]] 테일러링 가이드라인에서 처리.

## 4. 입력 자료 / 산출물
- **Input**
  - OSSP (WI-CMMI-01-01-01 산출)
  - 프로세스 니즈·목표 (PRO-CMMI-01-02 SP1.1)
  - 상업적 가용 작업환경 표준 후보 카탈로그 (예: ISO/IEC 12207, CNCF stacks, Cloud landing zones, 사내 IT 표준)
  - 라이선스·보안 정책 (POL-CMMI-04)
- **Output**
  - [[TMP-CMMI-01-01-04-01_작업환경_표준서]] (작성본)
  - CM 베이스라인 등록
  - PAL 등록 (WI-CMMI-01-01-03 와 연동)

## 5. 수행 절차

### 5.1 사전 준비
1. OSSP 정의서 확인 (WI-CMMI-01-01-01 완료본 §3 프로세스 요소표).
2. 후보 작업환경 표준 수집: 사내 IT 표준, 업계 표준, 클라우드 권장 아키텍처, 오픈소스 거버넌스 가이드.
3. EPG·IT·보안 합동 워크숍 일정 확정 (2회 권장: 평가 1회, 합의 1회).

### 5.2 수행 단계
1. **상업적 가용 표준 평가** (Subpractice 1)
   - 후보 표준 별 적합성 기준 평가: 적용범위·라이선스·통합성·보안·총비용·운영부담.
   - 평가 점수표 작성, 채택·부분채택·기각 권고.
2. **채택 영역 확정**
   - 채택 카테고리 분류: 개발도구(IDE/SDK), 빌드/CI, 형상관리, 테스트, 협업/문서, 보안/감사, 인프라/클라우드.
   - 카테고리별 조직 표준 1개 또는 다중 옵션 + 선정기준 명시.
3. **격차 영역 신규 정의** (Subpractice 2)
   - 채택 표준이 다루지 않는 영역(예: AI/ML 실험환경) 식별.
   - 격차 영역 별 내부 표준 신규 작성 — 최소 요구사항·권장 구성·예외 절차 포함.
4. **라이선스·보안 검토**
   - 채택 도구 라이선스 적합성 점검 (오픈소스 라이선스 호환성·상용 라이선스 좌석수).
   - 보안 책임자 검토 — 데이터 분류·접근통제·로그/감사 요구사항 정합성.
5. **변경 영향평가**
   - 기존 프로젝트 영향도 산정(이전 비용·중단 위험).
   - 마이그레이션 계획 초안 작성 (필요 시).
6. **피어 리뷰**
   - EPG + IT/DevOps + 대표 프로젝트 PM + 보안 책임자 참여.
   - 결함 항목 수정·재검토.
7. **경영진 승인 및 CM 베이스라인 등록**
   - 승인 후 [[PRO-CMMI-04-01_형상_관리_절차]] 따라 베이스라인화.
   - PAL 등록 ([[WI-CMMI-01-01-03]] 연동) → MAT-001 행 추가.

### 5.3 완료 조건 (체크리스트)
- [ ] 후보 작업환경 표준 평가표 작성·서명 (최소 3개 후보 평가)
- [ ] 카테고리별 조직 채택 표준 1개 이상 확정 (개발/빌드/테스트/협업/보안/인프라 6개 영역 모두 커버)
- [ ] 격차 영역 신규 표준 식별 및 정의 또는 "없음" 명시
- [ ] 라이선스 적합성 점검 통과 (위반 0건)
- [ ] 보안 책임자 검토 결재 첨부
- [ ] 피어 리뷰 결함 0건 또는 모두 해소
- [ ] CEO/CTO 승인 결재 첨부
- [ ] CM 베이스라인 ID 부여 + MAT-001 등록
- [ ] PAL 카탈로그 등재

## 6. 인터페이스 부서
- **EPG**: 본 지침 주 수행, PAL 등록 협조
- **IT/DevOps**: 도구·인프라 표준 실 운영 책임
- **보안 책임자**: 라이선스·접근통제·감사 요구사항 검토
- **CM**: 베이스라인 등록·버전 통제 ([[PRO-CMMI-04-01]])
- **OPF**: 작업환경 변경 니즈 공급 ([[PRO-CMMI-01-02]] SP1.1)
- **OT**: 채택 도구 교육 니즈 전달 ([[PRO-CMMI-01-03]] SP1.1)

## 7. 주의사항 / 예외 처리

### 7.1 라이선스 위반 발견
- 평가/운영 중 라이선스 위반 발견 시: 즉시 사용 중지 → 보안 책임자 보고 → 대체 도구 평가.
- 위반이 6개월 이상 미해결 시 해당 도구 강제 제거 + 사후 감사 실시.

### 7.2 신규 도구 긴급 도입 요청
- 프로젝트가 표준 외 신규 도구 도입 긴급 요청 시:
  - 임시 사용 허용 기간 최대 3개월 + 보안 임시 평가 의무.
  - 3개월 내 EPG 평가 거쳐 표준 편입 또는 폐기 결정.
  - 기간 연장은 1회만 (총 6개월), 그 이상은 waiver 절차.

### 7.3 상용 라이선스 좌석 부족
- 채택 도구 좌석이 프로젝트 수요 대비 부족 시:
  - 우선순위 기준(영업/매출 임팩트·고객 약속) 적용.
  - 미배정 프로젝트는 대체 도구 임시 사용 + 차기 예산 반영.

### 7.4 격차 영역 표준 부재로 임시 사용 발생
- 격차 영역 임시 사용 도구는 6개월 내 정식 표준 등재 의무.
- 미등재 시 PPQA 비준수 이슈로 보고.
