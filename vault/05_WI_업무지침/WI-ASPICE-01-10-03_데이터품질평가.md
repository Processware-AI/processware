# 데이터 품질 평가 (WI-ASPICE-01-10-03)

> 상위 절차: [[PRO-ASPICE-01-10_ML데이터관리프로세스]]
> ASPICE 4.0 SUP.11.BP3 — 데이터 품질 평가 (분포·편향·노이즈·라벨 일치율)

## 1. 업무 목적

본 지침은 [[WI-ASPICE-01-10-02_라이선스및개인정보검증]] 을 통과한 데이터셋에 대해 **완전성·정확성·균형성·ODD 대표성** 의 4개 차원에서 품질을 정량 평가하고, 기준 미달 항목을 식별하여 정제·보완 계획을 수립함으로써 학습용 데이터의 신뢰성을 확보하는 데 목적이 있다.

## 2. 수행 주체

| 역할 | 담당 |
|---|---|
| 주수행자 | Data Engineer |
| 검토자 | ML Engineer / QA |
| 승인자 | Data Manager |

## 3. 범위

라이선스 검증을 통과한 모든 ML 학습·검증·테스트용 데이터셋에 적용한다. 단, 외부 벤치마크 비교용으로만 사용되는 표준 공개 데이터셋은 본 지침의 분포 보완 단계만 면제하고 나머지 평가는 동일하게 수행한다.

## 4. 입력 자료 / 산출물

- **Input**:
  - [[WI-ASPICE-01-10-02_라이선스및개인정보검증]] 사용 승인 데이터셋
  - 데이터셋 라벨 파일·메타데이터
  - ODD(Operational Design Domain) 정의 문서 (시스템 요구사항 산출물)
  - 자동 품질 검사 도구(Great Expectations, 사내 QC Pipeline) 설정
- **Output**:
  - [[TMP-ASPICE-01-10-03-01_데이터품질평가보고서]] 작성 산출물
  - 자동 검사 로그 파일
  - 미달 항목 보완 계획 (재수집 요청·라벨 재검수 요청)

## 5. 수행 절차

### 5.1 사전 준비

1. 평가 대상 데이터셋의 ID·총 샘플 수·클래스 목록을 정리한다.
2. 품질 지표 기준값을 확정한다 — 완전성 ≥ 95%, 어노테이션 오류율 ≤ 2%, 소수 클래스 비율 ≥ 5%, 야간/우천 등 ODD 시나리오 커버리지 ≥ 5% 등.
3. ODD 정의 문서에서 평가해야 할 시나리오 차원을 추출한다 (날씨·시간대·도로 종류·차량 밀도 등).
4. 자동 품질 검사 도구의 룰셋·기댓값(expectations) 을 데이터셋 스키마에 맞춰 갱신한다.
5. 수동 점검(샘플링) 인원·할당량을 사전 배정한다.

### 5.2 수행 단계 (ASPICE SUP.11.BP3 참조)

1. **자동 품질 지표 측정** — Great Expectations 또는 사내 QC Pipeline 을 실행하여 완전성(Null·Missing 비율), 정확성(스키마·범위 위반 비율), 라벨 일관성(중복·모순 라벨)을 정량 측정한다.
2. **클래스 분포 분석** — 각 클래스의 샘플 수와 비율을 산출하고, 소수 클래스(minority class) 가 기준 비율 미만인 경우 식별한다.
3. **ODD 커버리지 평가** — 메타데이터(촬영 시간·기상·도로 종류) 기반으로 ODD 시나리오별 샘플 분포를 산출하고, 미달 시나리오를 식별한다.
4. **수동 점검(샘플링)** — 무작위 샘플(전체의 0.5~1%) 을 추출하여 어노테이터 2명 이상이 라벨 정확성을 재평가한다. IoU 또는 Cohen κ 로 일치율을 계산한다.
5. **편향 분석** — 보호 속성(연령·성별·지역 등) 또는 ODD 속성(시간대·기상)에 대한 통계적 편향을 산출한다.
6. **종합 판정** — 4대 차원의 측정값을 기준값과 비교하여 통과/조건부 통과/미달 판정한다.
7. **보완 계획 수립** — 미달 항목별로 재수집·재라벨링·합성 데이터 보강 등 보완 방법과 일정·담당자를 결정한다.
8. **보고서 결재 및 인계** — Data Manager 승인 후 [[WI-ASPICE-01-10-04_데이터버전및베이스라인]] 으로 인계한다. 미달인 경우 보완 후 재평가 또는 조건부 인계 결정.

### 5.3 완료 조건 체크리스트

- [ ] 자동 품질 지표(완전성·정확성·일관성)가 전 데이터셋에 대해 측정되었다.
- [ ] 클래스별 분포와 소수 클래스 비율이 산출되었다.
- [ ] ODD 시나리오별 커버리지가 산출되었다.
- [ ] 수동 점검 샘플링이 기준 비율(0.5~1%) 이상 수행되었다.
- [ ] 라벨 일치율(IoU 또는 κ)이 ≥ 95% 이거나, 미달 시 재라벨링 계획이 첨부되었다.
- [ ] 미달 항목별 보완 계획(담당자·일정 포함) 이 수립되었다.
- [ ] Data Manager 승인 결재가 완료되었다.
- [ ] [[TMP-ASPICE-01-10-03-01_데이터품질평가보고서]] 가 결재 완료되어 `08_REC_기록/` 에 보관되었다.
- [ ] [[MAT-001_문서관리대장]] 에 본 산출물이 등록되었다.

## 7. 주의사항 / 예외 처리

1. **ODD 미달 시나리오의 합성 데이터 사용** — 야간·악천후 등 수집이 어려운 시나리오를 합성 데이터(시뮬·증강) 로 보완할 경우, 합성 비율을 명시하고 ML Engineer 가 학습 영향을 별도 검증해야 한다.
2. **라벨 일치율 미달** — IoU 또는 κ 가 기준 미달일 때, 어노테이터 가이드 갱신 + 재훈련 + 재라벨링의 3단계 보완을 거치고 재평가한다.
3. **편향 발견 시 처리** — 보호 속성 편향이 임계값을 초과하면 [[PRO-ASPICE-01-04_머신러닝공학프로세스]] 의 공정성(Fairness) 평가와 연계하여 학습 단계에서 보정 기법(re-weighting, sampling) 을 적용해야 함을 명시한다.
4. **외부 벤치마크 데이터셋의 분포 미달** — 외부 표준 데이터셋(예: KITTI) 은 분포 보완을 면제하되, 보완 면제 사유를 보고서에 명시하고 학습 산출물에서 별도 분리 추적한다.
