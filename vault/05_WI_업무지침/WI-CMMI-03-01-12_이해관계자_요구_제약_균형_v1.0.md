# 이해관계자 요구 제약 균형 (WI-CMMI-03-01-12)

> 상위 절차: [[PRO-CMMI-03-01_요구사항_개발_및_관리_절차_v1.0]]
> 상위 정책: [[POL-CMMI-03_엔지니어링_정책_v1.0]]

## 1. 업무 목적
서로 상충하는 이해관계자 요구·기술 제약·예산·일정·법규 제약 사이에서 정량적 가중치 평가로 합의 가능한 균형점을 도출하고, 채택된 트레이드오프 결정을 추적 가능 형태로 기록한다.

## 2. 수행 주체
- 주 수행자: BA
- 검토자: PM, 아키텍트, 재무 담당
- 승인자: PM (중대 트레이드오프는 CTO 또는 CEO 까지 에스컬레이션)
- 자문: 법무·보안·고객 대표

## 3. 범위
- 대상: 동일 객체에 대해 둘 이상의 이해관계자가 양립 불가한 요구를 제기한 모든 사례
- 시기: 요구사항 도출 단계 후반 ~ 베이스라인 직전, 또는 변경요청 심의 단계
- 제외: 단일 이해관계자 내부의 우선순위 조정은 일반 변경 통제(PRO-CMMI-04-02)로 처리

## 4. 입력 자료 / 산출물
- **Input**
  - 요구사항 등록부의 충돌 마킹 항목
  - 일관성 점검 결과의 모순 목록(WI-CMMI-03-01-06 산출)
  - 필요충분성 분석서의 모순 후보(WI-CMMI-03-01-11 산출)
  - 프로젝트 제약 명세(예산·일정·법규)
- **Output**
  - [[TMP-CMMI-03-01-12-01_요구_제약_균형_평가서_v1.0]] 작성본
  - 균형 결정 회의록 + 이해관계자 서명
  - 영향 받은 요구사항의 갱신 요청서

## 5. 수행 절차

### 5.1 사전 준비
1. 충돌 항목별로 관련 이해관계자(요구 제기자·영향 수령자)를 식별한다.
2. 평가 기준 가중치 표를 PM 과 합의한다(예: 사업 가치 30 / 비용 25 / 일정 20 / 리스크 15 / 규제 10).
3. 균형 결정 회의를 일정·장소·참석자 확정하여 통지한다.

### 5.2 수행 단계
1. **충돌 정의**
   - 충돌하는 요구사항 쌍(또는 그룹) 을 한 문장으로 정의한다.
   - 각 요구를 견지할 경우의 결과를 구체적으로 기술한다.
2. **대안 도출**
   - 최소 3가지 대안을 작성한다(요구A 우선 / 요구B 우선 / 절충안).
   - 각 대안에 대해 비용·일정·리스크·고객 만족도 등 영향을 추정한다.
3. **정량 평가**
   - 합의된 가중치 표를 적용하여 대안별 점수를 계산한다.
   - 점수 차이가 5% 이내면 정성 평가(이해관계자 우선순위) 로 보완한다.
4. **회의 운영**
   - 영향 받는 이해관계자 대표·PM·아키텍트가 참석한다.
   - 평가 결과를 공유하고, 채택 대안을 표결 또는 합의로 결정한다.
5. **결정 기록**
   - 채택 대안·기각 대안·기각 사유를 모두 기록한다.
   - 영향 받는 요구사항 ID 를 모두 나열하고 갱신 사항을 표로 정리한다.
6. **이해관계자 서명**
   - 핵심 이해관계자(요구 제기자)와 PM 의 서명을 수령한다.
   - 서명 거부 시 에스컬레이션 절차(7.3)에 따른다.
7. **추적성 갱신**
   - 결정으로 변경된 요구사항을 양방향 추적성 매트릭스(WI-CMMI-03-01-05)에 즉시 반영한다.

### 5.3 완료 조건 체크리스트
- [ ] 모든 충돌 항목 식별 (목록 첨부)
- [ ] 각 충돌별 대안 3가지 이상 도출
- [ ] 정량 평가 점수표 작성 완료
- [ ] 균형 결정 회의록 서명본 보관
- [ ] 핵심 이해관계자 서명 100% 수령
- [ ] 영향 받은 요구사항 갱신 요청서 발행
- [ ] 양방향 추적성 매트릭스 갱신 완료

## 6. 인터페이스 부서
- DAR 위원회(PRO-CMMI-04-04): 중대 트레이드오프의 의사결정 평가
- 재무팀: 비용 영향 추정 검증
- 법무·보안: 규제·보안 제약 해석
- 형상관리: 결정 후속 변경요청 발행

## 7. 주의사항 / 예외 처리

### 7.1 이해관계자 합의 결렬
- 표결·합의 모두 실패 시 PM 이 결정권을 행사한다.
- PM 결정에 핵심 이해관계자가 거부 의사를 표명하면 CTO·CEO 까지 에스컬레이션한다.
- 에스컬레이션 결정 사항은 균형 평가서에 별도 섹션으로 기록한다.

### 7.2 규제·법규로 인한 강제 결정
- 법규·인허가 요건이 우선시되어 정량 평가 결과를 무시해야 하는 경우, 강제 사유와 근거 법령을 명시하고 정량 평가는 참고 기록으로만 보존한다.
- 법무팀의 사전 검토 의견을 반드시 첨부한다.

### 7.3 이해관계자 서명 거부
- 서명 거부 사유를 면담 기록으로 보존하고, PM 단독 결재로 처리한다.
- 추후 분쟁 대비를 위해 거부 의사 진술서를 회의록 첨부 자료로 보관한다.
- 서명 거부 빈도가 누적되면 거버넌스 검토(PRO-CMMI-01-01)로 이관한다.
