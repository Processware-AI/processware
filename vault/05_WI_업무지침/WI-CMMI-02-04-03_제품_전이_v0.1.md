# 제품 전이 (WI-CMMI-02-04-03)

> 상위 절차: [[PRO-CMMI-02-04_공급자_협약_관리_절차]]

## 1. 업무 목적
인수 완료된 공급자 제품·서비스를 운영·유지보수 조직으로 통제된 방식으로 인계한다.

## 2. 수행 주체
- **주 수행자**: Project Manager
- **검토자**: 운영팀 Lead, CM, 보안 (해당 시)
- **승인자**: PMO Director

## 3. 범위
PRO-CMMI-02-04 §5 의 **SG2 SP2.3**.

## 4. 입력 자료 / 산출물
- **Input**: 인수 완료 제품, 인수 결과보고서, 운영 매뉴얼, 교육 자료
- **Output**: [[TMP-CMMI-02-04-03-01_전이_계획서]]

## 5. 수행 절차

### 5.1 사전 준비
1. 운영팀 인수 준비도 확인 (인력·도구·환경).
2. 보안 평가 (해당 시).

### 5.2 수행 단계
1. **전이 계획 수립** — 인수 항목, 인수자, 교육·문서, 일정, 책임 인계 시점.
2. **교육·문서 전달** — 운영 매뉴얼, 운영 가이드, 비상 연락처, 알려진 이슈 목록.
3. **운영 환경 설치/배포** — 인수 환경 → 운영 환경 이관.
4. **인수자 검증** — 운영팀이 인수 product 시연·운영 가능 확인.
5. **PMO Director 결재 + 책임 인계** — 인계 시점 이후 issue 는 운영팀 책임.

### 5.3 완료 조건
- [ ] 전이 계획서 작성·승인
- [ ] 교육·문서 전달 완료
- [ ] 운영 환경 정상 동작 확인
- [ ] 운영팀 검증 (sign-off)
- [ ] PMO Director 결재
- [ ] CM 등록 (전이 완료 record)

## 6. 인터페이스 부서
- **운영팀**: 인수자
- **CM**: 전이 베이스라인 등록 ([[PRO-CMMI-04-01]])
- **CISO**: 보안 영역 평가 (해당 시)
- **OT**: 운영팀 교육 ([[PRO-CMMI-01-03]])

## 7. 주의사항 / 예외 처리

### 7.1 운영팀 미준비
- 운영팀 인력·도구·환경 미준비 시: 전이 연기 + 보강 계획.
- 보강 일정 ≥ 2주 시 SAM 종료 일정 PP 영향평가.

### 7.2 운영 환경 설치 실패
- 인수 환경과 운영 환경 차이로 실패 시: 환경 차이 분석 + 보정 (공급자 협력 필요시 SOW §보증 조항 발동).

### 7.3 인수자 검증 실패
- 운영팀이 product 운영 불가 판단 시: 추가 교육 + 재검증. 2차 실패 시 PMO escalation.
