# 구현 및 지원 문서 작성 (WI-CMMI-03-02-04)

> 상위 절차: [[PRO-CMMI-03-02_기술_솔루션_설계_절차]]

## 1. 업무 목적
TDP·ICD·MBR 결정에 따라 컴포넌트를 구현하고, 사용자·운영·유지보수·교육 매뉴얼 등 지원 문서를 작성하여 후속 PI/VER/인도 단계의 기반을 제공한다.

## 2. 수행 주체
- **주 수행자**: Engineer (구현), Doc Writer (매뉴얼)
- **검토자**: Chief Engineer, VER Lead, OT Lead (교육 매뉴얼)
- **승인자**: Project Manager

## 3. 범위
PRO-CMMI-03-02 §5 의 **SP3.1 (Implement the Design) ~ SP3.2 (Develop Product Support Documentation)**.

## 4. 입력 자료 / 산출물
- **Input**
  - TDP (WI-CMMI-03-02-02)
  - ICD (WI-CMMI-03-02-03)
  - MBR 결정 (WI-CMMI-03-02-03)
- **Output**
  - [[TMP-CMMI-03-02-04-01_구현_결과_보고서]] (코드 베이스라인 + 단위시험 결과)
  - [[TMP-CMMI-03-02-04-02_지원_문서_패키지]] (사용자/운영/유지보수/교육 매뉴얼)
  - PI 인계용 컴포넌트 (인수문서 첨부)

## 5. 수행 절차

### 5.1 사전 준비
1. 구현 환경(빌드·CI/CD·테스트) 준비.
2. 단위시험 기준·코드 커버리지 목표 설정.
3. 매뉴얼 골격(WI-CMMI-03-02-02 TDP 인덱스) 확인.

### 5.2 수행 단계
1. **컴포넌트 구현 (SP3.1)**
   - Make 컴포넌트: 설계서·ICD 따라 코드/구성 작성.
   - Buy/Reuse 컴포넌트: 조달·통합 작업.
   - 코딩 표준·시큐어 코딩(OWASP) 적용.
2. **단위 시험·정적 분석**
   - 단위 시험 통과 + 코드 커버리지 목표 달성.
   - 정적 분석(SAST) 무결함(Critical·High).
3. **빌드 베이스라인 등록**
   - 구현 결과를 CM 베이스라인으로 등록.
   - SBOM 생성·등록(SAM 연계).
4. **지원 문서 작성 (SP3.2)**
   - 사용자 매뉴얼 (사용자 그룹별).
   - 운영 매뉴얼 (정상/예외/실패 모드).
   - 유지보수 매뉴얼 (백업·복원·패치·트러블슈팅).
   - 교육 매뉴얼 (OT 연계).
5. **문서 검토**
   - Chief Eng + 사용자 대표 + OT Lead 합동 검토.
6. **PI 인계 준비**
   - 컴포넌트 인수문서(인계서) 작성 — PI WI-CMMI-03-03-03 SP3.1 입력.
7. **PM 승인**

### 5.3 완료 조건 체크리스트
- [ ] 모든 Make 컴포넌트 단위시험 통과
- [ ] 코드 커버리지 목표 충족 (예: ≥ 70%)
- [ ] 정적 분석 Critical·High 결함 0건
- [ ] CM 빌드 베이스라인 + SBOM 등록
- [ ] 사용자/운영/유지보수/교육 매뉴얼 4종 작성
- [ ] 문서 검토 결함 0건 또는 해소
- [ ] PI 인수문서 작성
- [ ] PM 승인 결재

## 6. 인터페이스 부서
- **Engineer / Doc Writer**: 주 수행
- **CM**: 베이스라인·SBOM 등록 ([[PRO-CMMI-04-01]])
- **VER Lead**: 단위시험 결과 사전 점검 ([[PRO-CMMI-03-04]])
- **PI Lead**: 인수문서 수신 ([[PRO-CMMI-03-03]] SP3.1)
- **OT Lead**: 교육 매뉴얼 수신 ([[PRO-CMMI-01-03]])
- **SAM Owner**: Buy 컴포넌트 인수 ([[PRO-CMMI-02-04]])

## 7. 주의사항 / 예외 처리

### 7.1 단위시험 목표 미달
- 커버리지 70% 미만:
  - 부족 영역을 식별·시험 추가.
  - 시간 부족 시 PM 결재로 후속 마일스톤 등재.

### 7.2 매뉴얼 사용자 미참여 검토
- 사용자 대표가 검토 불참:
  - 대리인 검토 + 사후 사용자 피드백 의무.
  - 인도 후 30일 내 피드백 수렴·반영.

### 7.3 Buy 컴포넌트 인수 지연
- 공급업체 인도 지연:
  - 영향 평가 + 일정 재조정.
  - RSKM 등록 + PM 결정.
