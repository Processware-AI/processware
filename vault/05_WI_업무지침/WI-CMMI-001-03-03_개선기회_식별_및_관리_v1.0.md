# 개선기회 식별 및 관리 (WI-CMMI-001-03-03)

> 상위 절차: [[PRO-CMMI-103_프로세스_관리_및_개선_절차_v1.0]]

## 1. 업무 목적
다양한 채널(부적합·만족도·아이디어 박스)에서 개선기회를 수집·평가·등록부에 등재하여 개선 활동의 입력으로 활용한다.

## 2. 수행 주체
- 주 수행자: SEPG 운영자
- 검토자: PCB
- 승인자: SEPG Lead

## 3. 범위
- 부적합·KPI 미달·만족도 의견·신기술 평가·교훈 등 모든 입력

## 4. 입력 / 산출물
- Input: 부적합 보고, KPI 알람, 회고록
- Output: [[TMP-CMMI-001-03-03_개선기회_등록부_v1.0]] 갱신, 백로그

## 5. 수행 절차

### 5.1 사전 준비
1. 입력 채널별 수집 주기 정의.
2. 평가 기준(영향·노력·전략 정렬) 적용.

### 5.2 수행 단계
1. **수집** — 채널별 입력 통합.
2. **분류** — 즉시 조치 / 기획 / 보류.
3. **평가** — 영향·노력 매트릭스 점수화.
4. **백로그 등재** — 우선순위 정렬.
5. **분기 검토** — PCB 에서 진척 보고.
6. **종결** — 구현 완료 시 효과 평가 후 종결.

### 5.3 완료 조건
- [ ] 등록부 분기 갱신
- [ ] 백로그 우선순위 부여
- [ ] PCB 보고

## 6. 인터페이스 부서
- 각 부서, QA, OT

## 7. 주의사항 / 예외 처리

### 7.1 중복 아이디어
- 유사 항목 통합. 제안자 모두에게 통합 통보.

### 7.2 보류 장기화
- 6개월 보류 시 자동 폐기 또는 재검토.

### 7.3 즉시 조치
- 안전·법규 항목은 즉시 조치 후 사후 등재.

|---|---|---|

| 분기 종결 건수 | ≥ 5건 | 분기 |
