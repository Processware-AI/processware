# 변경 추적·통제 (WI-CMMI-04-01-02)

> 상위 절차: [[PRO-CMMI-04-01_형상_관리_절차]] · 표준: CMMI-DEV V1.3 CM SG2

## 1. 업무 목적
베이스라인 CI 에 대한 변경 요청을 표준 양식으로 접수·추적하고, 영향평가 후 CCB 심의로 통제하여 무결성을 유지한다.

## 2. 수행 주체
- **주 수행자**: Configuration Manager
- **검토자**: 영향 평가자(개발·QA·아키텍처)
- **승인자**: CCB

## 3. 범위
PRO-CMMI-04-01 §5 의 **SP2.1(변경 요청 추적) ~ SP2.2(CI 통제)** 적용. 베이스라인 수립 완료([[WI-CMMI-04-01-01]]) 후 적용.

## 4. 입력 자료 / 산출물
- **Input**
  - 변경 요청(고객·내부) 원천
  - 현 베이스라인 ([[WI-CMMI-04-01-01]] 산출)
  - CM 시스템 운영규정
- **Output**
  - [[TMP-CMMI-04-01-02-01_변경요청서_CR]] (작성본)
  - [[TMP-CMMI-04-01-02-02_CCB_의사록_및_변경통제_대장]] (작성본)
  - 갱신된 CI(개정 이력 포함)

## 5. 수행 절차

### 5.1 사전 준비
1. 베이스라인 릴리즈 완료 확인([[WI-CMMI-04-01-01]]).
2. CCB 정기 회의 주기 확정(주 1회 권장).
3. 변경요청 접수 채널(이메일·티켓·폼) 안내.

### 5.2 수행 단계
1. **변경요청 접수·등록 (SP2.1 전반)**
   - 요청자가 CR 양식 작성 → CM 시스템 등록.
   - CR ID 자동 부여 + 우선순위 임시 부여(요청자 기준).
2. **영향평가**
   - 기술 영향(아키텍처·인터페이스·성능) 평가.
   - 비용·일정·리스크 영향 산정.
   - 영향 받는 CI 목록 확정.
3. **CCB 심의 (SP2.1 후반)**
   - CCB 회의 안건 등록 → 회의 개최.
   - 결정: 승인 / 조건부 승인 / 보류 / 반려.
   - 의사록 작성·서명.
4. **CI 통제 — 개정 (SP2.2)**
   - 승인된 변경에 대해 CI 개정 작업 지시.
   - 작업 결과를 CM 시스템에 체크인 → 새 버전 부여.
   - 이전 버전과의 diff 기록.
5. **변경통제 대장 갱신**
   - 변경요청 → CCB 결정 → CI 버전 매핑을 변경통제 대장에 등록.
   - 요청자·이해관계자 통보.
6. **베이스라인 재라벨링(필요 시)**
   - 다수 CI 동시 개정으로 신규 베이스라인 필요 시 [[WI-CMMI-04-01-01]] §5.2.4 호출.

### 5.3 완료 조건
- [ ] 모든 CR 이 CM 시스템에 등록되어 추적 가능
- [ ] 영향평가 의견서 첨부 (기술·비용·일정·리스크)
- [ ] CCB 결정이 의사록에 명기·서명
- [ ] 승인 CR 은 해당 CI 가 개정·체크인되어 새 버전 부여
- [ ] 변경통제 대장 갱신 완료
- [ ] 요청자·이해관계자 통보 기록 보관

## 6. 인터페이스 부서
- **CM**: 본 지침 주 수행
- **CCB**: 변경 심의·승인 ([[PRO-CMMI-04-01]] §3 RACI)
- **개발/QA/아키텍처**: 영향평가 입력
- **PMC**: 변경이 일정·비용에 미친 영향 모니터링 ([[PRO-CMMI-02-02]])
- **REQM**: 요구사항 변경 시 REQM 양방향 추적성 갱신 ([[PRO-CMMI-02-03]])

## 7. 주의사항 / 예외 처리

### 7.1 긴급 변경 (Hotfix)
- 서비스 중단·법규 위반 등 24시간 내 처리 필요 시 긴급 변경 프로세스 적용:
  - 변경 적용 후 24시간 내 사후 CCB 보고·승인.
  - 사후 미승인 시 즉시 롤백.

### 7.2 영향평가 의견 불일치
- 평가자 간 의견 충돌 시 CCB 가 중재.
- 중재 불가 시 DAR ([[PRO-CMMI-04-04]]) 발동.

### 7.3 무단 변경 발견
- 형상감사([[WI-CMMI-04-01-03]])에서 CCB 미승인 변경 발견 시:
  - 즉시 PPQA 부적합 보고서 발행 ([[PRO-CMMI-04-03]]).
  - 변경 롤백 또는 사후 CCB 심의로 정식화.
