# 통합 계획·실행 (WI-CMMI-02-05-02)

> 상위 절차: [[PRO-CMMI-02-05_통합_프로젝트_관리_절차]]

## 1. 업무 목적
PP 의 통합 계획서를 다른 PA (MA·OT·CM·QA 등) 계획과 통합하여 상위 통합 계획서를 작성하고, 정의된 프로세스로 실제 실행하며 측정 actuals 를 수집한다.

## 2. 수행 주체
- **주 수행자**: Project Manager
- **검토자**: PMO Director, 인접 PA 책임자
- **승인자**: CEO/CTO

## 3. 범위
PRO-CMMI-02-05 §5 의 **SP1.4 (계획 통합) ~ SP1.5 (정의된 프로세스 실행)**.

## 4. 입력 자료 / 산출물
- **Input**
  - PP 통합 계획서 ([[TMP-CMMI-02-01-03-03]])
  - WI-01 산출 정의 프로세스 기술서
  - 인접 PA 계획 (MA Measurement Plan, OT 교육 계획, CM Plan, QA Plan)
- **Output**
  - [[TMP-CMMI-02-05-02-01_상위_통합_계획서]]
  - 정의된 프로세스 실행 산출물·측정 actuals (지속)

## 5. 수행 절차

### 5.1 사전 준비
1. PP 통합 계획서 + 인접 PA 계획 수집.
2. 정의 프로세스 (WI-01) 확정.

### 5.2 수행 단계
1. **계획 통합 (SP1.4)** — PP 의 7 섹션 + MA/OT/CM/QA 계획 = 상위 통합 계획서.
2. **상위 일관성 검증** — Cross-PA 충돌 (예: OT 일정 vs PP 일정) 검출 + 조정.
3. **CEO/CTO 결재 + CM 등록** — IPM 상위 통합 계획 베이스라인.
4. **정의된 프로세스로 실행 (SP1.5)** — WI-01 정의 프로세스의 단계 따라 작업 실행.
5. **측정 actuals 수집** — MA 계획에 따른 실측 데이터 수집 (PMC WI-01 과 연계).

### 5.3 완료 조건
- [ ] 상위 통합 계획서 작성 (PP + 인접 PA 통합)
- [ ] Cross-PA consistency check 통과
- [ ] CEO/CTO 결재
- [ ] CM 베이스라인 등록
- [ ] 정의된 프로세스 실행 시작 — 첫 측정 actuals 수집

## 6. 인터페이스 부서
- **MA**: 측정 계획·actuals 수집 ([[PRO-CMMI-04-02]])
- **OT**: 교육 계획 ([[PRO-CMMI-01-03]])
- **CM**: 통합 계획 베이스라인 ([[PRO-CMMI-04-01]])
- **PPQA**: 정의된 프로세스 audit ([[PRO-CMMI-04-03]])

## 7. 주의사항 / 예외 처리

### 7.1 Cross-PA 충돌
- OT 일정이 PP 일정 후 제공되는 경우: 사전 교육 또는 일정 조정.
- 충돌 해소 불가 시 정의 프로세스 수정 (WI-01 재진입).

### 7.2 정의 프로세스 미준수 (실행 단계)
- PPQA audit 에서 미준수 발견 시 시정조치 (PMC SP2).
- 빈번 미준수 시 정의 프로세스 적합성 재검토.

### 7.3 측정 actuals 미수집
- 도구 부재로 수집 누락 시 수동 집계 + MA 도구 보강 요청.
