# 해결안 선정 (WI-CMMI-04-04-03)

> 상위 절차: [[PRO-CMMI-04-04_의사결정_분석_결정_절차]] · 표준: CMMI-DEV V1.3 DAR SG1 (후반)

## 1. 업무 목적
[[WI-CMMI-04-04-02]] 평가 결과에 근거하여 해결안을 선정하고, 결정 근거를 문서화하여 결재함으로써 사안에 대한 최종 의사결정을 산출한다.

## 2. 수행 주체
- **주 수행자**: Decision Owner
- **검토자**: Decision Facilitator, 평가 패널
- **승인자**: Approver (사안별 위임권자)

## 3. 범위
PRO-CMMI-04-04 §5 의 **SP1.6(해결안 선정)** 적용. 후속 실행은 호출 PA(TS·SAM·OPF 등)로 위임.

## 4. 입력 자료 / 산출물
- **Input**
  - 대안 평가 결과서 ([[WI-CMMI-04-04-02]] 산출)
  - 평가 기준 정의서 ([[WI-CMMI-04-04-01]] 산출)
  - 사안 트리거 PA 의 후속 조건
- **Output**
  - [[TMP-CMMI-04-04-03-01_의사결정_근거서]]
  - [[TMP-CMMI-04-04-03-02_추천_해결안_보고서]]

## 5. 수행 절차

### 5.1 사전 준비
1. 평가 결과서 최종 본 확보.
2. 결재 라인·서식 확인.
3. 후속 실행 PA 의 인터페이스 요건 확인.

### 5.2 수행 단계
1. **평가 결과 검토**
   - 종합 순위·민감도 분석·강약점 종합 검토.
   - 1순위 대안과 차순위 대안의 점수 차이·안정성 평가.
2. **해결안 선정**
   - 평가 1순위를 기본 채택.
   - 단, 정성적 요소(전략 적합도·리스크 허용도)로 차순위 채택 가능 — 사유 명기 필수.
3. **의사결정 근거서 작성**
   - 사안·기준·대안·평가·선정 사유를 단일 문서로 정리.
   - 평가 결과 일부만 인용 금지 — 전체 표 포함.
4. **추천 해결안 보고서 작성**
   - 후속 실행 PA 에 전달할 인수 양식으로 작성.
   - 가정·전제 조건·후속 조치 권고 포함.
5. **결재**
   - Approver 결재.
   - 결재 후 결정 기록 [[PRO-CMMI-04-01]] 베이스라인 등록.
6. **호출 PA 로 핸드오프**
   - TS·SAM·OPF 등 호출 PA 에 결과 통보.
   - 통보 일시·수신자 기록.

### 5.3 완료 조건
- [ ] 선정 사유가 평가 결과와 일치 또는 차이 시 사유 명기
- [ ] 의사결정 근거서에 전체 평가 결과 포함
- [ ] 추천 해결안 보고서에 후속 조치 권고 포함
- [ ] Approver 결재 완료
- [ ] CM 베이스라인 등록 (의사결정 근거서)
- [ ] 호출 PA 통보 기록 보관

## 6. 인터페이스 부서
- **DAR**: 본 지침 주 수행
- **Decision Owner**: 선정 책임 (R)
- **Approver**: 최종 결재 (A)
- **호출 PA (TS/SAM/OPF/IPM 등)**: 결과 인수
- **CM**: 의사결정 근거서 베이스라인 등록 ([[PRO-CMMI-04-01]])

## 7. 주의사항 / 예외 처리

### 7.1 1순위 대안 차순위 채택
- 평가 1순위가 아닌 대안을 채택하는 경우:
  - 사유(전략·리스크·이해관계자) 명기 의무.
  - Approver 가 별도 결재 동의 필요.

### 7.2 결정 번복
- 결재 후 6개월 내 결정을 번복하는 경우:
  - 번복 사유서 작성.
  - DAR PRO §7 KPI(번복율 ≤10%) 모니터링 입력.

### 7.3 후속 실행 PA 에서 결정 불수용
- 호출 PA 가 결정을 수용하지 못하는 상황(기술적·법적 제약 발견):
  - 즉시 Decision Owner·Approver 통보.
  - 재평가 (SP1.5) 또는 기준 재정의 (SP1.2) 환류.
