# 인터페이스 완전성·관리 (WI-CMMI-03-03-02)

> 상위 절차: [[PRO-CMMI-03-03_제품_통합_절차]]

## 1. 업무 목적
인터페이스를 카테고리화·매핑하여 호환성·완전성을 확보하고, 인터페이스 변경을 통제·관리한다.

## 2. 수행 주체
- **주 수행자**: Integration Lead
- **검토자**: Architect, CM, 양 끝점 컴포넌트 Owner
- **승인자**: Project Manager

## 3. 범위
PRO-CMMI-03-03 §5 의 **SP2.1 (Review Interface Descriptions for Completeness) ~ SP2.2 (Manage Interfaces)**.

## 4. 입력 자료 / 산출물
- **Input**
  - ICD (TS WI-03 산출 — PI 인계)
  - 인터페이스 요구사항 정의서 (RD WI-02)
- **Output**
  - [[TMP-CMMI-03-03-02-01_인터페이스_카테고리_매핑표_관리기록]]
  - ICD 변경요청·승인 기록

## 5. 수행 절차

### 5.1 사전 준비
1. ICD + IR 정의서 베이스라인 확보.
2. 인터페이스 카테고리 정의(내부/외부, 데이터/제어/사용자/물리 등).

### 5.2 수행 단계
1. **인터페이스 분류·매핑 (SP2.1)**
   - 카테고리별 인터페이스 그룹화.
   - From↔To 컴포넌트 매트릭스 작성.
2. **완전성 검토**
   - IR ↔ ICD 매핑 (모든 IR 에 ICD 1개 이상).
   - ICD 7요소(스키마/프로토콜/SLA/에러/재시도/관측/버전) 점검.
3. **인터페이스 검사 (호환성)**
   - 양 끝점 합의·계약 시험(Contract Test) 결과 확인.
   - 불일치 발견 시 TS SP2.3 회귀.
4. **인터페이스 변경 관리 (SP2.2)**
   - 모든 ICD 변경은 CR → 영향평가 → 승인 절차.
   - 영향 받는 컴포넌트 Owner 통지·동의.
5. **베이스라인 등록**
   - ICD 베이스라인 ↔ PI 통합 베이스라인 매핑.
6. **PM 승인**

### 5.3 완료 조건 체크리스트
- [ ] 인터페이스 카테고리 분류 완료
- [ ] From↔To 매트릭스 작성
- [ ] IR ↔ ICD 100% 매핑
- [ ] ICD 7요소 점검 결과 기록
- [ ] Contract Test 통과 또는 회귀 처리
- [ ] 변경관리 기록 (CR ID·승인자)
- [ ] CM 베이스라인 매핑
- [ ] PM 승인 결재

## 6. 인터페이스 부서
- **Integration 팀**: 주 수행
- **Architect / TS Lead**: 불일치 시 회귀 ([[PRO-CMMI-03-02]] SP2.3)
- **CM**: ICD 변경 통제 ([[PRO-CMMI-04-01]])
- **양 끝점 컴포넌트 Owner**: 합의·동의

## 7. 주의사항 / 예외 처리

### 7.1 ICD 부분 미작성
- 일부 인터페이스 ICD 미작성:
  - TS WI-03 재실행 요청.
  - PI 단계에선 진행 보류.

### 7.2 Contract Test 실패 — 외부 시스템
- 외부 시스템 측 비호환:
  - 합의 문서(MOU) 따라 협의.
  - 인터페이스 적응 계층 또는 외부 측 변경 요청.

### 7.3 잦은 ICD 변경
- 한 인터페이스 변경 빈도 ≥ 3회/주:
  - 인터페이스 안정성 문제 → RSKM 등록.
  - 분기별 인터페이스 안정성 회고 실시.
