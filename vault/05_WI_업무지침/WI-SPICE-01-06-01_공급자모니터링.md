# 공급자 모니터링 업무지침 (WI-SPICE-01-06-01)

> 상위 절차: [[PRO-SPICE-01-06_구매및공급망프로세스]]
> ASPICE 매핑: ACQ.4 (Supplier Monitoring) — BP1~BP5

## 1. 업무 목적

외주 SW/HW/ML 공급사와의 공동활동·정보교환·진척·산출물 품질을 정기적으로 모니터링하여 합의된 약속을 추적하고 이탈 시 시정조치를 협의·실행한다.

## 2. 수행 주체
- **주 수행자**: Supplier Quality Engineer
- **검토자**: 도메인 Lead (SW/HW/ML), QA
- **승인자**: CTO

## 3. 범위
공급사 계약 체결 후부터 계약 종료 + 인수 완료 시점까지 적용한다.

## 4. 입력 자료 / 산출물
- **Input**: 공급사 계약서, SoW (Statement of Work), 합의된 산출물·일정·KPI
- **Output**: 공동활동 회의록, 진척 보고서, 시정조치 요청서, 공급사 평가 보고서

## 5. 수행 절차

### 5.1 사전 준비
1. 공급사 SPOC 확정 + 정기 회의 주기 합의 (주/월).
2. 정보교환 채널(이메일/포털/PLM) 정의 + 보안 협약(NDA) 확인.

### 5.2 수행 단계

1. **공동활동·인터페이스 합의** (ACQ.4.BP1)
   - 회의 주기·산출물 인계 일정·검토 절차.
   - 위험·개방 이슈 추적 메커니즘.

2. **합의 정보 교환** (ACQ.4.BP2)
   - 정의된 채널로 정기 보고서·산출물 교환.
   - 무결성·기밀성 준수.

3. **개발 산출물 검토** (ACQ.4.BP3)
   - 공급사 산출물 정기 검토 (기술·문제·리스크).
   - 미해결 항목 추적.

4. **공급사 진척 검토** (ACQ.4.BP4)
   - 일정·품질·비용 KPI 정기 검토.
   - MAN.5 리스크 완화 활동 연계.

5. **이탈 시정조치** (ACQ.4.BP5)
   - 합의 목표 미달 시 즉시 시정조치 요청.
   - 협상 필요 시 합의서 변경 후 문서화.

### 5.3 완료 조건 체크리스트
- [ ] 공동활동·인터페이스·정보교환 합의서 존재
- [ ] 정기 회의록 100% 작성·공유
- [ ] 산출물 검토 결과 추적 (개방 항목 0 유지)
- [ ] 공급사 KPI(일정·품질·비용) 월간 보고 완료
- [ ] 이탈 시 시정조치 요청서 + 후속 추적 완료
- [ ] 공급사 평가 보고서 분기 1회 발행
- [ ] [[MAT-001_문서관리대장]] 갱신

## 6. 인터페이스 부서
- **공급사 (External)**: 정보 교환·검토
- **도메인 Lead**: 산출물 기술 검토
- **QA**: 검토 결과 평가
- **Legal**: 계약 변경 협상
- **MAN.5 (리스크 관리)**: 리스크 완화 연계

## 7. 주의사항 / 예외 처리

### 7.1 공급사 산출물 품질 미달
- 인계물이 합의 품질 기준 미달 시:
  - 인수 거부 → 재작업 요청 (시한 명시).
  - 반복 발생 시 RCA 요청 + 평가 점수 반영.

### 7.2 정보 보안 위반
- 공급사 측에서 기밀 정보 유출 의심:
  - 즉시 정보 교환 중단 + 법무·보안 통보.
  - NDA 위반 조사 + 계약 종료 검토.

### 7.3 공급사 일정 지연
- 마일스톤 지연 발생 시:
  - 1) 영향 분석 (다른 도메인 일정 영향).
  - 2) 회복 계획 협상 (공급사 책임 + VWAY 지원).
  - 3) 미회복 시 대체 공급사 활성화.

### 7.4 공급사 부도/사업 종료
- 공급사 사업 중단 통보 수령 시:
  - 즉시 산출물·소스 코드·도구 라이선스 회수.
  - 대체 공급사 또는 내재화 계획 수립.
  - PM + CTO 에 즉시 보고.
