# 컴포넌트 통합 준비·조립 (WI-CMMI-03-03-03)

> 상위 절차: [[PRO-CMMI-03-03_제품_통합_절차]]

## 1. 업무 목적
통합 절차에 따라 각 컴포넌트의 통합 준비를 확인(인수)하고, 라운드별로 조립을 수행하여 통합 베이스라인을 형성한다.

## 2. 수행 주체
- **주 수행자**: Integration Lead, Engineer
- **검토자**: CM, 컴포넌트 Owner
- **승인자**: Project Manager

## 3. 범위
PRO-CMMI-03-03 §5 의 **SP3.1 (Confirm Readiness of Product Components for Integration) ~ SP3.2 (Assemble Product Components)**.

## 4. 입력 자료 / 산출물
- **Input**
  - 통합 절차·기준 (WI-CMMI-03-03-01)
  - 인터페이스 관리 기록 (WI-CMMI-03-03-02)
  - 컴포넌트 인수문서 (TS WI-04 산출)
- **Output**
  - [[TMP-CMMI-03-03-03-01_컴포넌트_인수_조립_기록부]]
  - 통합 베이스라인 (CM)

## 5. 수행 절차

### 5.1 사전 준비
1. 통합 절차서 + ICD 관리기록 확보.
2. 통합 환경 헬스체크 (R-1 통과 상태 확인).

### 5.2 수행 단계
1. **컴포넌트 인수 (SP3.1)**
   - 인수문서 점검: 식별·버전·SBOM·라이선스·시험 결과·매뉴얼·인터페이스 검증.
   - 인수 체크리스트 통과 시 라운드 진입.
2. **라운드별 조립 (SP3.2)**
   - 통합 절차서의 라운드 순서에 따라 조립.
   - 진입 기준 통과 → 조립 → 종료 기준 시험.
   - 각 라운드 완료 후 CM 베이스라인 등록.
3. **부적합 처리**
   - 종료 기준 미달 시 NCR 발행 → 시정 → 재조립.
   - 영향 큰 부적합은 상위 라운드 진입 차단.
4. **CM 통합 베이스라인**
   - 라운드별 베이스라인 + 최종 통합 베이스라인.

### 5.3 완료 조건 체크리스트
- [ ] 모든 컴포넌트 인수문서 점검 통과
- [ ] 라운드별 진입·종료 기준 모두 충족
- [ ] 라운드별 CM 베이스라인 등록
- [ ] 부적합(NCR) 모두 시정 완료 또는 PM 결재 보류
- [ ] 최종 통합 베이스라인 등록
- [ ] PM 승인 결재

## 6. 인터페이스 부서
- **Integration 팀**: 주 수행
- **TS / Engineer**: 컴포넌트 인수문서 공급 ([[PRO-CMMI-03-02]] WI-04)
- **CM**: 베이스라인 등록 ([[PRO-CMMI-04-01]])
- **PPQA**: 절차 준수 감사 ([[PRO-CMMI-04-03]])

## 7. 주의사항 / 예외 처리

### 7.1 인수 불합격
- 인수 체크리스트 불합격:
  - TS WI-04 측으로 차감 항목 통지 → 보완 후 재인수.
  - 라운드 일정 영향 시 PM 재계획.

### 7.2 라운드 종료 기준 미달
- 시험 미통과:
  - NCR 발행 → 시정 → 재시험.
  - 3회 재시험 미통과 시 설계 회귀 (TS) 검토.

### 7.3 부분 조립 인도
- 비즈니스 사유로 부분 조립을 인도해야 하는 경우:
  - PM 결재 + 인수자 동의 + 후속 조립 일정 명시.
  - 부분 인도 산출물은 별도 베이스라인 ID.
