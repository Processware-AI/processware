# 시정조치 분석·실행·종결 관리 (WI-CMMI-02-02-03)

> 상위 절차: [[PRO-CMMI-02-02_프로젝트_모니터링_통제_절차]]

## 1. 업무 목적
WI-01·WI-02 에서 식별된 편차·이슈를 분석하여 시정조치를 결정하고, 실행을 추적하여 종결까지 closed-loop 으로 관리한다.

## 2. 수행 주체
- **주 수행자**: Project Manager
- **검토자**: PMO Director, 시정조치 책임자
- **승인자**: Senior Management (cost-effort 영향이 임계치 초과 시)

## 3. 범위
PRO-CMMI-02-02 §5 의 **SP2.1(분석) ~ SP2.3(종결)**.

## 4. 입력 자료 / 산출물
- **Input**
  - WI-01 발견 편차 알림
  - WI-02 진행/마일스톤 검토 Fail/Conditional 항목
  - PPQA audit 부적합
- **Output**
  - [[TMP-CMMI-02-02-03-01_이슈_분석_결과]]
  - [[TMP-CMMI-02-02-03-02_시정조치_계획서]]
  - [[TMP-CMMI-02-02-03-03_시정조치_결과보고]]

## 5. 수행 절차

### 5.1 사전 준비
1. 이슈 분석 큐 확보 (WI-01·WI-02 에서 등록).
2. 근본원인 분석 (RCA) 기법 (5-Why, Fishbone) 도구 준비.

### 5.2 수행 단계
1. **이슈 분석 (SP2.1)**
   - 이슈별 근본원인 분석 (5-Why 권장)
   - 영향 범위·심각도 평가
   - 분석 결과 [[TMP-CMMI-02-02-03-01]] 작성
2. **시정조치 결정 (SP2.2)**
   - 분석 결과 기반 조치 옵션 도출
   - 비용·일정·범위 영향 평가
   - 임계치 초과 시 (예: 비용 +5% 이상) Senior Mgmt 승인
   - [[TMP-CMMI-02-02-03-02]] 작성
3. **시정조치 실행 추적**
   - 책임자·마감일 명시
   - 주간 PMC 모니터링에 추적 항목 등록 ([[WI-CMMI-02-02-01]])
4. **종결 검증 (SP2.3)**
   - 실행 결과 검증 (의도된 효과 달성 여부)
   - 재발 방지 조치 확인
   - [[TMP-CMMI-02-02-03-03]] 작성
   - 종결 또는 재 open 결정

### 5.3 완료 조건
- [ ] 모든 이슈 RCA 완료 — 근본원인 확정
- [ ] 시정조치 계획서 — 책임자·마감·예산 영향 명시
- [ ] 임계치 초과 조치 → Senior Mgmt 승인
- [ ] 실행 결과 검증 통과 — 의도 효과 달성
- [ ] 종결 후 [[MAT-001]] 등록 (조치 통계 PAL 입력)

## 6. 인터페이스 부서
- **시정조치 책임자**: 실행
- **MA**: 조치 통계 PAL 등록
- **PPQA**: 부적합 종결 검증 ([[PRO-CMMI-04-03]])
- **CM**: 베이스라인 변경 동반 시 ([[PRO-CMMI-04-01]])

## 7. 주의사항 / 예외 처리

### 7.1 RCA 미완료 (시간 부족)
- 긴급 이슈는 임시 조치 (workaround) + RCA 후속 수행.
- 후속 RCA 14일 내 완료 의무.

### 7.2 종결 후 재발
- 동일 근본원인 재발 시 RCA 재수행 + 시정 조치 강화.
- 분기 동일 이슈 ≥ 3회 발생 시 OPF 개선 입력 ([[PRO-CMMI-01-02]]).

### 7.3 시정조치가 베이스라인 변경 요구
- 베이스라인 변경 영향 (>= 30%) 시: PP 재진입 ([[PRO-CMMI-02-01]] §5 재실행).
- 그 미만은 시정조치로 처리 후 베이스라인 maintenance 갱신.
