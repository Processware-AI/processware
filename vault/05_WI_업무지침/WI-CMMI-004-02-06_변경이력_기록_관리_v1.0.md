# 변경이력 기록 관리 (WI-CMMI-004-02-06)

> 상위 절차: [[PRO-CMMI-402_형상관리_절차_v1.0]]

## 1. 업무 목적
모든 변경 이력을 검색 가능하게 보존하여 감사·추적·교훈에 활용한다.

## 2. 수행 주체
- 주 수행자: CM Manager
- 검토자: QA
- 승인자: PM

## 3. 범위
- 모든 변경

## 4. 입력 / 산출물
- Input: 변경요청서, 커밋 로그
- Output: [[TMP-CMMI-004-02-06_변경이력표_v1.0]]

## 5. 수행 절차

### 5.1 사전 준비
1. 보존 정책.
2. 검색 인덱싱.

### 5.2 수행 단계
1. **이력 수집**.
2. **표준화**.
3. **저장**.
4. **검증**.
5. **공개**.
6. **만료 처리**.

### 5.3 완료 조건
- [ ] 이력 무결성
- [ ] 검색 가능
- [ ] 권한 점검

## 6. 인터페이스 부서
- IT, 법무

## 7. 주의사항 / 예외 처리

- 사후 등록 강제.

### 7.2 손상
- 백업 복구.

### 7.3 외부 감사
- 신속 제공.
