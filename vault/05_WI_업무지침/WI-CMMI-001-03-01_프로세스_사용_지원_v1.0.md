# 프로세스 사용 지원 (WI-CMMI-001-03-01)

> 상위 절차: [[PRO-CMMI-103_프로세스_관리_및_개선_절차_v1.0]]

## 1. 업무 목적
프로젝트가 OSSP 를 정확하게 적용할 수 있도록 가이드·코칭·지원 데스크를 운영하여, 프로세스 적용 부적합과 우회 사용을 줄인다.

## 2. 수행 주체
- 주 수행자: SEPG 코치
- 검토자: PCB
- 승인자: SEPG Lead

## 3. 범위
- 진행 중 프로젝트 전수 + 신규 착수 프로젝트 30일 코칭

## 4. 입력 / 산출물
- Input: 지원요청서, FAQ
- Output: 코칭 결과, 지원 통계, FAQ 갱신

## 5. 수행 절차

### 5.1 사전 준비
1. 지원 채널(메일·메신저·티켓) 정의.
2. FAQ 라이브러리 확인.

### 5.2 수행 단계
1. **요청 접수** — 프로젝트가 요청서 제출.
2. **분류** — 단순 질의·해석·코칭으로 분류.
3. **응답** — SLA 내 답변(단순 1일·해석 3일·코칭 7일).
4. **지식화** — 반복 질의는 FAQ·작성예시로 등재.
5. **만족도 확인** — 응답 후 1주일 내 만족도 조사.
6. **분기 통계 보고** — PCB 에 지원 현황 보고.

### 5.3 완료 조건
- [ ] 요청 응답 SLA 준수
- [ ] FAQ 등재
- [ ] 만족도 조사 회수

## 6. 인터페이스 부서
- 각 PMO, OT, QA

## 7. 주의사항 / 예외 처리

### 7.1 SLA 초과
- SLA 초과 시 SEPG Lead 에 자동 에스컬레이션.

### 7.2 동일 질의 반복
- 분기 5회 이상 동일 질의는 표준 자산 개정 후보로 등재.

### 7.3 비표준 사용 발견
- 코칭 중 비표준 사용 발견 시 [[WI-CMMI-001-04-06_프로세스_준수_검토_v1.0]] 로 시정조치 발의.
