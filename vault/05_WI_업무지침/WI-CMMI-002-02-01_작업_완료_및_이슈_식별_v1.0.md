# 작업 완료 및 이슈 식별 (WI-CMMI-002-02-01)

> 상위 절차: [[PRO-CMMI-202_프로젝트_모니터링_및_통제_절차_v1.0]]

## 1. 업무 목적
주간 단위로 작업 완료 상태와 이슈를 식별·기록하여 통제 의사결정의 입력을 제공한다.

## 2. 수행 주체
- 주 수행자: PM
- 검토자: PMO
- 승인자: PM

## 3. 범위
- 진행 중 모든 프로젝트, 주간 1회

## 4. 입력 / 산출물
- Input: WBS 진척, 팀 보고
- Output: [[TMP-CMMI-002-02-01_주간_상태보고서_v1.0]]

## 5. 수행 절차

### 5.1 사전 준비
1. 팀 데일리 자료 수집.
2. 이슈 분류 기준 확인.

### 5.2 수행 단계
1. **완료 항목 확인** — Done 정의 충족 여부.
2. **진행 항목 % 갱신**.
3. **이슈 식별** — Blocker/Risk/Decision.
4. **에스컬레이션 판단** — 임계값 기준.
5. **보고서 작성**.
6. **PMO 공유**.

### 5.3 완료 조건
- [ ] 보고서 PMO 공유
- [ ] 이슈 등록부 갱신
- [ ] 차주 회의 의제

## 6. 인터페이스 부서
- PMO, 부서장

## 7. 주의사항 / 예외 처리

### 7.1 데이터 결측
- 팀 미보고 시 PM 이 직접 확인.

### 7.2 임계값 위반
- 자동 에스컬레이션, Sponsor 통보.

### 7.3 허위 진척
- 허위 보고 발견 시 즉시 시정.
