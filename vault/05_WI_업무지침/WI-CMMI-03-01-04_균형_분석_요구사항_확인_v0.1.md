# 균형 분석·요구사항 확인 (WI-CMMI-03-01-04)

> 상위 절차: [[PRO-CMMI-03-01_요구사항_개발_절차]]

## 1. 업무 목적
요구사항 간 충돌(비용·일정·기술·리스크 등 다면)을 식별하고 균형 분석을 통해 결정을 내리며, 최종 요구사항 베이스라인이 의도된 환경에서 사용자 니즈를 충족함을 확인(VAL) 한다.

## 2. 수행 주체
- **주 수행자**: Chief Engineer (균형), VAL Lead (확인)
- **검토자**: Architect, Project Manager
- **승인자**: Project Manager

## 3. 범위
PRO-CMMI-03-01 §5 의 **SP3.4 (Analyze Requirements to Achieve Balance) ~ SP3.5 (Validate Requirements)** 단계.

## 4. 입력 자료 / 산출물
- **Input**
  - WI-01 ~ WI-03 산출물 베이스라인
  - 이관된 충돌 로그(WI-02, WI-03)
  - 비용·일정·리스크 입력 (PP, RSKM 산출물)
- **Output**
  - [[TMP-CMMI-03-01-04-01_요구사항_균형_분석_보고서]]
  - [[TMP-CMMI-03-01-04-02_요구사항_확인_결과_보고서]]
  - 최종 요구사항 베이스라인 (REQM 인계)

## 5. 수행 절차

### 5.1 사전 준비
1. 이전 WI(02, 03)에서 이관된 충돌·이슈 로그 수집.
2. 비용·일정 모델(PP), 리스크 등록부(RSKM) 최신본 확보.
3. DAR 적용 후보 결정(영향 큰 결정만) — [[PRO-CMMI-04-04_의사결정_분석_결정_절차]] 연계.
4. VAL 확인 환경·방법(시연/시뮬레이션/사용자 시험) 확정.

### 5.2 수행 단계
1. **충돌 우선순위 평가 (SP3.4)**
   - 이관된 충돌을 영향도(High/Med/Low) × 빈도 매트릭스로 평가.
   - High-High 충돌은 DAR 적용 의무.
2. **DAR 적용 (해당 충돌)**
   - 대안 식별·평가 기준·평가·선정 — [[PRO-CMMI-04-04]] DAR WI 참조.
   - DAR 결정 기록을 균형 분석 보고서에 첨부.
3. **균형 결정**
   - 충돌별 최종 결정 (수용/거부/연기/타협).
   - 거부·연기는 후속 변경요청(CR)으로 발행.
4. **요구사항 베이스라인 갱신**
   - 균형 결정을 WI-01~03 산출물에 반영 → 최종 베이스라인.
5. **요구사항 확인 (SP3.5)**
   - VAL Lead 주관으로 시연·시뮬레이션·프로토타입 시험.
   - 사용자·이해관계자 참여 — 의도 충족 여부 확인.
6. **확인 결과 정리**
   - 충족/부분 충족/미충족 분류.
   - 미충족·부분 충족은 변경요청 발행 또는 후속 마일스톤 항목으로 등록.
7. **PM 승인 및 REQM 인계**
   - 최종 베이스라인을 REQM ([[PRO-CMMI-02-03]]) 으로 인계.

### 5.3 완료 조건 체크리스트
- [ ] 이관된 모든 충돌이 결정(수용/거부/연기/타협) 상태 부여
- [ ] High-High 충돌에 DAR 적용 100%
- [ ] 균형 결정이 WI-01~03 산출물에 반영(베이스라인 갱신)
- [ ] VAL 확인 실시 — 모든 핵심 시나리오 커버
- [ ] 미충족 항목 변경요청(CR) 발행 또는 후속 마일스톤 등록
- [ ] PM 승인 결재
- [ ] REQM 인계 송부 기록

## 6. 인터페이스 부서
- **요구사항 엔지니어링팀 + Chief Eng**: 균형 분석 주관
- **VAL Lead**: 확인 주관 ([[PRO-CMMI-03-05]] 와 동일 인력)
- **DAR Owner**: 영향 큰 결정 DAR 적용 ([[PRO-CMMI-04-04]])
- **REQM Owner**: 최종 베이스라인 수신 ([[PRO-CMMI-02-03]])
- **CM**: 베이스라인 등록

## 7. 주의사항 / 예외 처리

### 7.1 결정 불가 충돌
- 양 측 이해관계자가 끝까지 합의 불가:
  - PM 결정으로 격상.
  - PM 도 결정 불가시 경영진(C-level) 결정 회부.
  - 결정 지연 위험을 RSKM 등록.

### 7.2 VAL 확인 환경 부재
- 시제품·시뮬레이터가 없어 VAL 불가:
  - 대안: 사용자 인터뷰 + 프로토타입 paperware + 시나리오 워크스루.
  - 대체 사용 시 신뢰도 + PM 결재.

### 7.3 사용자 불참
- 사용자가 확인 세션에 참여 불가:
  - 대리인 + 비동기 회람.
  - 대리인 의견과 후속 사용자 피드백 차이가 발견되면 재VAL 의무.
