# 표준프로세스·라이프사이클 모델 정의 (WI-CMMI-01-01-01)

> 상위 절차: [[PRO-CMMI-01-01_조직_표준프로세스_수립_유지_절차]]

## 1. 업무 목적
조직의 모든 프로젝트가 따를 표준프로세스 집합(OSSP)을 정의하고, 프로젝트가 적용할 라이프사이클 모델(워터폴/반복/애자일/하이브리드 등)을 문서화하여 일관된 작업 기준을 제공한다.

## 2. 수행 주체
- **주 수행자**: EPG (Engineering Process Group)
- **검토자**: Process Owner, CM
- **승인자**: 경영진 (CEO/CTO)

## 3. 범위
PRO-CMMI-01-01 §5 의 **SP1.1(OSSP 수립) ~ SP1.2(라이프사이클 모델)** 단계에 적용. 테일러링은 [[WI-CMMI-01-01-02]] 참조.

## 4. 입력 자료 / 산출물
- **Input**
  - OPF 식별 프로세스 니즈 (PRO-CMMI-01-02 SP1.1 산출물)
  - 비즈니스 목표·외부 표준(CMMI/ISO 등)
  - 기존 OPA·교훈 등록부
- **Output**
  - [[TMP-CMMI-01-01-01-01_조직표준프로세스_OSSP_정의서]] (작성본)
  - [[TMP-CMMI-01-01-01-02_라이프사이클모델_기술서]] (작성본)
  - CM 베이스라인 등록

## 5. 수행 절차

### 5.1 사전 준비
1. OPF 산출 "프로세스 니즈·목표 기술서" 확보 (PRO-CMMI-01-02 §6 SP1.1).
2. 기존 OSSP·라이프사이클 자산 PAL 에서 조회.
3. EPG 워크숍 일정 확정 (2회 권장).

### 5.2 수행 단계
1. **프로세스 아키텍처 도출**
   - 프로세스 분류: 관리·엔지니어링·지원 3계열로 그룹화
   - 18 PA를 컴포넌트 프로세스로 매핑
2. **프로세스 요소(Process Element) 분해**
   - 각 표준프로세스를 입력·활동·산출물·역할 단위로 분해
   - 요소 간 관계(선후·의존) 명시
3. **OSSP 정의서 작성**
   - [[TMP-CMMI-01-01-01-01]] 양식 따라 작성
   - 적용 정책·표준·모델 부합성 명시
4. **라이프사이클 모델 식별·선정**
   - 조직 사용 모델 후보(Waterfall/Iterative/Agile/Hybrid) 리스트업
   - 각 모델별 단계·게이트·산출물 정의
5. **라이프사이클 모델 기술서 작성**
   - [[TMP-CMMI-01-01-01-02]] 양식 작성
   - OSSP 와의 매핑 관계 표시
6. **피어 리뷰**
   - EPG + Process Owner + 대표 프로젝트 PM 참여
   - Defect 항목 수정·재검토
7. **경영진 승인 및 CM 베이스라인 등록**
   - 승인 후 [[PRO-CMMI-04-01_형상_관리_절차]] 따라 베이스라인화
   - PAL 등록 ([[WI-CMMI-01-01-03]] 와 연동)

### 5.3 완료 조건
- [ ] OSSP 정의서 작성·승인 완료 (모든 프로세스 요소 분해 포함)
- [ ] 라이프사이클 모델 최소 1개 이상 기술 (조직 채택 모델 모두)
- [ ] 피어 리뷰 결함 0건 또는 모두 해소
- [ ] CEO/CTO 승인 결재 첨부
- [ ] CM 베이스라인 ID 부여 + MAT-001 등록
- [ ] 적용 정책·표준 부합성 체크리스트 통과

## 6. 인터페이스 부서
- **EPG**: 본 지침 주 수행
- **CM**: 베이스라인 등록·버전 통제 ([[PRO-CMMI-04-01]])
- **OPF**: 프로세스 니즈 공급 ([[PRO-CMMI-01-02]] SP1.1)
- **OT**: 본 OSSP 가 교육 니즈 입력 ([[PRO-CMMI-01-03]] SP1.1, BPM-feeds-OT)

## 7. 주의사항 / 예외 처리

### 7.1 신규 표준 추가 (예: ISO 27001 신규 적용)
- 기존 OSSP 와의 충돌 영향평가 후 통합 — 별도 신규 OSSP 분기 금지.
- 영향평가 결과 60% 이상 변경 시 OSSP 메이저 버전 증분 (v1.x → v2.0).

### 7.2 프로젝트별 예외 요청
- 단일 프로젝트가 OSSP 미준수 영역 발생 시: 우선 [[WI-CMMI-01-01-02]] 테일러링 가이드로 처리.
- 가이드 범위 초과 시 **waiver 신청** → EPG 검토 → CEO 승인.

### 7.3 라이프사이클 모델 미정의 영역
- 새로운 프로젝트 형태(예: 데이터 사이언스 PoC) 등장 시 임시 모델 사용 허용:
  - 6개월 내 정식 모델로 등재 의무
  - 임시 사용 기간 동안 PPQA 가 별도 모니터링
