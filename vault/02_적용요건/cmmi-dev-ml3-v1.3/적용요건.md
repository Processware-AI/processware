## 1. 표준 개요

| 항목 | 내용 |
|---|---|
| 표준명 | CMMI for Development, Version 1.3 (Capability Maturity Model Integration — Development) |
| 발행기관 | Software Engineering Institute, Carnegie Mellon University |
| 발행보고서 | CMU/SEI-2010-TR-033 / ESC-TR-2010-033 |
| 발행일 | 2010-11 |
| 표현형식 | Continuous (4 Capability Levels: CL0-CL3) / Staged (5 Maturity Levels: ML1-ML5) |
| 본 적용 범위 | **Staged ML3 cumulative** — ML2(7 PA) + ML3(11 PA) = **18 PA** |
| 후속 표준 | CMMI V2.0 (2018, ISACA) → CMMI V3.0 (2023, ISACA Performance Solutions) |
| 라이선스 | CMU/SEI 내부 사용·파생물 허용 (PDF p.2 저작권 고지) |

### 1.1 컴포넌트 분류 체계
- **Required**: SG (Specific Goal), GG (Generic Goal) — 적합성 평가 시 필수
- **Expected**: SP (Specific Practice), GP (Generic Practice) — 동등 대안 가능
- **Informative**: Subpractice, Example Work Product, Note, Example, Reference, Elaboration — 안내 목적

### 1.2 채택 가이드 요지 (Part One Ch.5 발췌)
- **시작점**: 상위경영진 후원 확보 → 프로세스 그룹 구성 (IDEAL 모델 참조).
- **선택 3가지**: 조직 일부 선정 / 모델 선정(CMMI-DEV) / 표현형식 선정(staged or continuous).
- **Agile 호환**: CM·PI·PMC·PP·PPQA·RD·REQM·RSKM·TS·VER 10개 PA에 "In Agile environments..." 형식 참고박스 존재. 본 적용요건은 Agile 환경에서도 해석 가능.
- **평가**: ARC(Appraisal Requirements for CMMI) 기준 SCAMPI A/B/C 평가법. ML 인증은 SCAMPI A.

### 1.3 핵심 용어 (Glossary 발췌)
- **OSSP** (Organization's Set of Standard Processes): 조직의 표준프로세스 집합 (OPD가 정의·유지).
- **PAL** (Process Asset Library): 조직 프로세스 자산 라이브러리 (OPD가 운영).
- **OPA** (Organizational Process Assets): 조직의 프로세스 자산 (OPD가 관리, OPF가 배포).
- **CCB** (Configuration Control Board): 형상관리위원회 (CM 관할).
- **Defined Process**: OSSP에서 테일러링된 프로젝트별 정의 프로세스 (GG3/IPM SP1.1).
- 전체 227개 용어: `inputs/01_표준원문/CMMI-DEV/definitions.yaml` 참조.

- ML4: OPP (Process Management), QPM (Project Management) — 정량적 관리, 본 빌드 제외.
- ML5: CAR (Support), OPM (Process Management) — 원인 분석/조직 성과 관리, 본 빌드 제외.
