# 02_적용요건

## 역할
`inputs/` 의 표준·법규·가이드·기존 자산을 분석하여 **우리 조직에 적용되는 요건**을 정의한 문서.

외부 표준을 그대로 옮기는 것이 아니라, 조직이 해석·통합·선별한 **우리만의 요건 베이스라인**.

## 명명 원칙
폴더명은 표준 코드가 아닌 **조직이 부르는 이름**으로 자유롭게 결정.

```
02_적용요건/
  OOO사_품질경영체계/     ← ISO 9001 기반이지만 조직 이름으로
    적용요건.md
  OOO사_정보보호체계/     ← ISO 27001 + 개인정보보호법 통합
    적용요건.md
  고객불만처리/           ← 특정 프로세스 단위도 가능
    적용요건.md
```

## 적용요건.md 구조
- frontmatter: doc_id, title, sources(참조 표준 + 조항), version
- 본문: REQ-NNN 형식의 요건 목록
- 각 요건마다 출처 매핑 표 포함 (어느 표준 어느 조항에서 왔는지)

## 생성 시점
`/process-plan "{적용요건명}"` 실행 시 standard-analyzer 가 inputs/ 분석 후 자동 생성.
기존 파일이 있으면 재사용.

## 이후 흐름
```
적용요건.md → /process-plan → POL/PRO/WI/TMP/EX 생성
                                    ↓
                              MAT-002 (조항 ↔ 산출물 매핑) 자동 채워짐
```

## 갭 분석 기준점
이 파일이 있으면 `/process-plan --gap-only` 실행 시
"REQ 중 커버되지 않은 것" 을 자동 탐지 가능.
