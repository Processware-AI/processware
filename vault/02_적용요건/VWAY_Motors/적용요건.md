## 0. 본 문서의 범위

본 적용요건은 ASPICE 4.0 의 **407건 전체 요건(Purpose 32 + Outcomes 181 + Base Practices 194)** 중에서 **각 프로세스 단위로 대표 요건(Purpose 1 + 핵심 BP 2~3)** 을 발췌해 매트릭스로 정리한 **운영 적용본**이다.

- 전체 407건의 1:1 추적성은 [[MAT-011_VWAY_Motors_추적성]] 에서 별도 관리한다.
- 본 문서의 Req-ID 체계: `ASPICE-{프로세스}-R-{NNN}` (3자리 0-padding, 프로세스별 연번).
- 모든 Req 는 `inputs/01_표준원문/VWAY_Motors/requirements.yaml` 의 원천 `req_id` 를 `source_citation.locator` 로 보존한다.

**Req 유형 표기**
- 유형: `프로세스` (BP/Outcome 중심) / `기록` (record/evidence) / `정책` (cross-process governance) / `역량` (PA 충족) / `인프라` (Tooling)
- 의무/권고: ASPICE BP 는 모두 `shall` 톤 → **의무**. Note 는 권고.
- 대상 자산(target_asset): PRO=절차, WI=업무지침, TMP=양식, REC=기록, EX=예시, POL=정책

## 4. 해석 노트

1. **대표 BP 선정 기준**: 각 프로세스의 BP 중 (a) 핵심 활동의 정의 BP1, (b) 추적성/일관성 BP, (c) 기록·소통 BP 를 우선했다. 전체 BP 추적성은 [[MAT-011_VWAY_Motors_추적성]] 에서 1:1 로 보존된다.
2. **Outcome 의 매트릭스 미포함 사유**: Outcome 은 결과 상태(WHAT)이고 BP 가 그 달성 활동(HOW)이므로, 절차 작성을 위한 적용요건에서는 BP 우선 노출이 효율적이다. Outcome 은 검증·평가 단계에서 PA 1.1 의 evidence 로 활용된다.
3. **POL 최소화 원칙**: ASPICE 는 capability_model 이라 POL 은 1건(역량 거버넌스)만 두고, 32 프로세스를 PRO 로 1:1 매핑한다. 상위 정책 통합이 필요하면 IATF 16949 / ISO 9001 의 POL 에 hook 만 둔다.
4. **MLE/SUP.11 의 우선순위**: 신규 그룹이지만 자율주행/ADAS 프로젝트에서는 즉시 활성화 필요. 본 적용요건에서 동일 우선순위로 매핑.
5. **HWE.3/HWE.4 분리**: ASPICE 4.0 에서 HW V-모델의 검증 좌(HWE.4: 요구사항 vs HW) / 우(HWE.3: 설계 vs HW)가 명확히 분리되었으므로 PRO 도 별도로 구성.
