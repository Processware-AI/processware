# 검증 및 확인 절차 (PRO-CMMI-03-04)

> 상위 정책: [[POL-CMMI-03_엔지니어링_정책_v1.0]]

## 1. 목적
검증(요구사항 충족)과 확인(의도 환경 작동)을 분리 계획·환경·절차로 운영하여 솔루션의 적합성·유효성을 입증한다.

## 2. 적용 범위
- 모든 솔루션·구성요소의 검증·확인
- 단위·통합·시스템·인수 시험 전 단계
- 시뮬레이션·실환경 확인 모두 적용

## 3. 역할과 책임 (RACI)
| 단계 | V&V Lead | 시험자 | 개발자 | QA | 이해관계자 |
|---|---|---|---|---|---|
| 기본 검증 | C | **R** | C | C | I |
| 기본 확인 | C | **R** | C | C | **C** |
| 대상·방법 선정 | **R** | C | C | C | C |
| 환경 | **R** | C | C | I | I |
| 절차·기준 | **R** | C | C | **C** | I |
| 검증 수행 | **R** | **R** | C | C | I |
| 확인 수행 | **R** | **R** | C | C | **C** |
| 결과 분석·전달 | **R** | C | C | C | C |
