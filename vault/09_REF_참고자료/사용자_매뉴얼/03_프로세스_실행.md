# 03. 프로세스 실행 — `/process-do`

> `/process-plan`이 만들어 놓은 WI(업무지침)를 AI가 실행하고, 대화를 통해 기록(REC)을 자동으로 작성한다.

---

## 기본 사용법

```
/process-do WI-PPR-101-01              # WI 번호 직접 지정
/process-do 프로젝트 계획 수립         # 자연어 (자동 매칭)
```

---

## 실행 흐름

```
사용자 명령
  → AI가 WI 파일 읽기
  → step별 대화 (AI가 질문 → 사용자가 답변)
  → 승인 단계 있으면 HITL 정지
  → 사용자 승인 후 재개
  → REC 기록 자동 생성
  → MAT-005 갱신
```

---

## 진입 모드 8가지

### 1. start 모드 — 신규 실행

```bash
/process-do WI-PPR-101-01                        # WI 번호 직접 지정
/process-do "프로젝트 계획 수립"                  # 자연어 자동 매칭
/process-do --executor "홍길동" WI-PPR-101-01     # 실행자 명시
/process-do --dry-run WI-PPR-101-01               # 실제 저장 없이 미리보기
```

**자연어 매칭 동작**:
- 신뢰도 90% 이상 → 자동 선택 후 진행 (`"자동 매칭: WI-PPR-101-01, 신뢰도 0.94"`)
- 90% 미만 → 후보 3개 제시 후 선택 요청
- 매칭 없음 → 더 구체적으로 입력 요청

### 2. resume 모드 — 중단된 실행 재개

```bash
/process-do --resume run-a3f9c2b1
```

이전 세션에서 중단된 경우 또는 HITL 승인 대기 상태에서 재개.

상태별 동작:
| 상태 | 동작 |
|---|---|
| `pending_approval` | 승인 대기 중임을 안내 + 승인 명령 안내 |
| `running` | 마지막 step부터 재개 |
| `completed` | 이미 완료 안내 + REC 경로 출력 |

### 3. approve 모드 — 승인

```bash
/process-do --approve run-a3f9c2b1
/process-do --approve run-a3f9c2b1 --approver "박팀장"
```

### 4. reject 모드 — 반려

```bash
/process-do --reject run-a3f9c2b1 --reason "표본 부족 — 재수집 필요"
```

`--reason`은 필수다. 반려 사유가 REC에 기록된다.

### 5. status 모드 — 상태 확인

```bash
/process-do --status run-a3f9c2b1
```

특정 trace의 현재 상태·승인자·경과 시간 출력.

### 6. check-approvals 모드 — 일괄 승인 회수

```bash
/process-do --check-approvals
```

외부에서 파일을 직접 편집해 승인/반려 표시한 경우, 미처리된 것들을 일괄 처리.

### 7. check-timeouts 모드 — 타임아웃 검출

```bash
/process-do --check-timeouts
/process-do --check-timeouts --dry-run    # 검출만, 발송 안 함
```

승인 대기 중 오래된 것들을 에스컬레이션. 권장: 30분~1시간 주기 실행.

### 8. rebuild-catalog 모드 — 프로세스 카탈로그 재구축

```bash
/process-do --rebuild-catalog             # 전체
/process-do --rebuild-catalog --scope CMMI  # 특정 표준만
```

새로운 WI가 추가된 후 자연어 매칭 카탈로그(MAT-007)를 갱신할 때 사용.

## HITL (승인 게이트) 상세

WI 단계 중 "승인" 표현이 있으면 자동으로 HITL 게이트가 발동된다.

### 흐름

```
[AI 실행 중] step-04: 표본 평가 완료
[HITL 게이트] → 실행 정지 (status: pending_approval)

출력:
  ⏸ HITL 정지 — trace_id: run-a3f9c2b1
  📋 approval_request.md 생성됨
  승인: /process-do --approve run-a3f9c2b1 --approver "이름"
  반려: /process-do --reject run-a3f9c2b1 --reason "사유"

[사용자 확인 후 승인]
/process-do --approve run-a3f9c2b1 --approver "박팀장"

[AI 실행 재개] → step-05 이후 진행 → REC 완성
```

### approval_request.md

HITL 정지 시 생성되는 파일의 위치: `.claude/runs/{trace_id}/approval_request.md`

이 파일을 직접 편집해서 `status: approved`로 바꿔도 되며, 이후 `/process-do --check-approvals`로 일괄 처리된다.

## 실행 결과물

### REC 파일

`vault/08_REC_기록/REC-PPR-101-01-01-2026-001_...md` 형식으로 저장.

### trace 디렉터리

```
.claude/runs/run-a3f9c2b1/
├── state.yaml          실행 상태
├── trace.jsonl         전체 대화·LLM 출력 로그 (심사 증적)
└── rec_payload.yaml    REC 생성용 데이터
```

---

## 옵션 전체 목록

| 옵션 | 효과 |
|---|---|
| `--dry-run` | 대화는 진행하되 REC·MAT-005 저장 생략 (미리보기) |
| `--executor <이름>` | 실행자(작성자) 명시 |
| `--auto-approve` | HITL 자동 승인 (**PoC·테스트 전용, 실운영 금지**) |
| `--approver <이름>` | 승인/반려 응답자 명시 |
| `--reason "..."` | 반려 사유 (reject 모드 필수) |
| `--auto-threshold <0~1>` | 자연어 자동 채택 신뢰도 임계 (기본 0.9) |

---

## 주의사항

1. `/process-do`는 POL/PRO/WI/TMP/EX 파일을 **절대 수정하지 않는다**. 읽기 전용.
2. 결과물은 `vault/08_REC_기록/`에만 저장된다.
3. REC 파일은 덮어쓰지 않는다 — 같은 날 재실행하면 `2026-002`, `2026-003`으로 새 번호 부여.
