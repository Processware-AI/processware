# 04-1. 외부 표준 GAP 분석 — `/process-audit`

> `/process-check`가 *이행 증적*을 심사한다면, `/process-audit`은 *프로세스 설계 자체*가 외부 표준을 얼마나 커버하는지 분석한다.

---

## 기본 사용법

```
/process-audit start --against ISO9001
/process-audit start --against ISO27001 --scope "OOO사_품질경영체계"
/process-audit start --against CMMI-ML3 --llm-fallback
/process-audit --confirm run-gXXXXXXXX
/process-audit --export run-gXXXXXXXX --format xlsx
```

---

## 선행 조건

`inputs/01_표준원문/{표준코드}/requirements.yaml` 파일이 있어야 분석을 시작할 수 있다.

해당 파일이 없으면 먼저 `/process-ingest`로 표준 원문을 전처리한다.

```bash
/process-ingest sources/ISO9001.pdf --standard ISO9001 --version 2015
/process-ingest sources/ISO27001.pdf --standard ISO27001 --version 2022
```

`--llm-fallback` 플래그를 사용하면 `inputs/` 파일 없이도 실행할 수 있다. 단, LLM이 자체 학습 데이터를 기준으로 조항을 추론하므로 신뢰도가 낮게 표시된다. 인증 심사 제출용에는 `--llm-fallback` 결과를 사용하지 않는 것을 권장한다.

---

## 전체 흐름

```
Phase 0: Preflight — inputs/ 표준 요건 파일 탐색
Phase 1: gap-analyzer → coverage_matrix.yaml (조항별 COVERED/PARTIAL/GAP/N/A 판정)
   ↓ HITL 정지 (커버리지 초안 확정)
Phase 2 (confirm 후):
  ├ gap-reporter → REC-GAP-{표준코드}-{YYYY}-{NNN}.md 발행
  ├               → MAT-002 §매핑표 갱신
  └ act-trigger  → critical/major GAP → act queue 자동 push
Phase 3 (선택):
  └ gap-exporter → --export --format xlsx|html|pdf → exports/ 내보내기
```

---

## 진입 모드

### 1. start 모드 — 신규 GAP 분석

```bash
/process-audit start --against ISO9001
/process-audit start --against ISO9001 --scope "OOO사_품질경영체계"
/process-audit start --against ISO27001 --strictness strict
/process-audit start --against CMMI-ML3 --llm-fallback
```

`--scope`를 생략하면 vault 전체 POL/PRO/WI를 대상으로 분석한다. 특정 경영체계 범위만 분석할 때 명시한다.

`--strictness` 3단계:

| strictness | 보고 대상 |
|---|---|
| strict | GAP + PARTIAL + THIN (PRO 있으나 WI 없음) |
| normal (기본) | GAP + PARTIAL |
| lenient | GAP 만 |

인증 심사 준비 단계에서는 `strict`를 권장한다. 평상시 현황 파악에는 `normal`이 적합하다.

### 2. confirm 모드 — HITL 확정 후 보고서 발행

AI가 생성한 커버리지 초안을 사람이 검토하고 확정한다. 이 단계에서 N/A 처리, GAP 등급 조정이 가능하다.

```bash
/process-audit --confirm run-gXXXXXXXX
/process-audit --confirm run-gXXXXXXXX --not-applicable "8.3"="외주 개발 미실시"
/process-audit --confirm run-gXXXXXXXX --adjust-gap "6.1"=minor
/process-audit --confirm run-gXXXXXXXX --no-act-queue   # act 큐 자동 push 억제
```

- `--not-applicable`: 조직 특성상 해당 조항이 적용되지 않음을 근거와 함께 명시한다. 인증심사에서 N/A 근거로 활용된다.
- `--adjust-gap`: AI 판정 등급(critical/major/minor)을 수동으로 조정한다. 조정 이유는 보고서 audit trail에 자동 기록된다.
- `--no-act-queue`: 보고서만 발행하고 act 큐 등록은 하지 않는다. 파일럿·내부 현황 파악 목적에 사용한다.

### 3. export 모드 — 외부 인증기관 제출용 파일 생성

```bash
/process-audit --export run-gXXXXXXXX --format xlsx
/process-audit --export run-gXXXXXXXX --format html
/process-audit --export run-gXXXXXXXX --format pdf
/process-audit --export run-gXXXXXXXX --format xlsx --template iso27001
/process-audit --export run-gXXXXXXXX --format xlsx --org "OOO 주식회사"
```

**XLSX 포맷**

Python 스크립트(`{doc_id}_compliance_matrix.py`)를 생성한다. 스크립트를 실행하면 .xlsx 파일이 만들어진다.

```bash
pip install openpyxl
python REC-GAP-ISO9001-2026-001_compliance_matrix.py
```

3시트 구성:
- **시트 1 — 커버리지 요약**: 전체 조항 수, COVERED/PARTIAL/GAP/N/A 집계, 커버리지 비율
- **시트 2 — 부합성 매트릭스**: 전체 조항 행별 판정 결과, 매핑된 내부 문서 ID
- **시트 3 — GAP 상세**: critical → major → minor 순 정렬, 개선 방향 포함

`--template iso27001`처럼 표준별 공식 양식 템플릿을 지정할 수 있다. 지원 템플릿: `iso9001`, `iso27001`, `iso14001`.

**HTML / PDF 포맷**

`{doc_id}_report.html`을 생성한다 (Noto Sans KR, A4 landscape print CSS 적용).

PDF 변환 방법:
```bash
# 방법 1: 브라우저 인쇄 (Cmd+P → PDF로 저장)
open REC-GAP-ISO9001-2026-001_report.html

# 방법 2: wkhtmltopdf
wkhtmltopdf --orientation Landscape REC-GAP-ISO9001-2026-001_report.html output.pdf

# 방법 3: pandoc
pandoc REC-GAP-ISO9001-2026-001_report.html -o output.pdf --pdf-engine=weasyprint
```

### 4. status / list 모드

```bash
/process-audit --status run-gXXXXXXXX
/process-audit --list
/process-audit --list --against ISO9001
```

`--list`는 전체 GAP 분석 실행 이력(run-g로 시작하는 trace ID, 표준, 상태, 커버리지 비율)을 보여준다.

## 산출물

| 산출물 | 경로 | 설명 |
|---|---|---|
| GAP 분석 보고서 | `vault/08_REC_기록/AUDIT/REC-GAP-{표준코드}-{YYYY}-{NNN}.md` | COVERED/PARTIAL/GAP 상세, audit trail 포함 |
| 커버리지 매트릭스 | `.claude/runs/{trace_id}/coverage_matrix.yaml` | 조항별 판정 원본 데이터 |
| MAT-002 갱신 | `vault/90_MAT_통합매핑/MAT-002_규제요구사항_대조표.md` | 규제 요구사항 대조표 |
| act queue | `.claude/queues/process-act/queue-q*.yaml` | critical/major GAP 자동 등록 |
| 내보내기 파일 | `vault/08_REC_기록/AUDIT/exports/` | XLSX 스크립트 또는 HTML |

GAP 보고서 번호 패턴: `REC-GAP-{표준코드}-{YYYY}-{NNN}`
예시: `REC-GAP-ISO9001-2026-001`, `REC-GAP-ISO27001-2026-001`

## GAP → 차원 4 자동 인계

confirm 후 critical/major GAP은 자동으로 `kind: gap_remediation` act queue로 등록된다.

```
confirm 완료 → critical 2건, major 4건
  → act_queue: queue-qa1b2c3d4, queue-qe5f6a7b8
  → /process-act start queue-qa1b2c3d4
```

- **중복 방지**: 동일한 (gap_rec_id, clause) 조합은 재발행되지 않는다. 기존에 처리 중인 큐가 있으면 해당 queue ID를 참조로 표시한다.
- **억제**: `--no-act-queue` 플래그를 사용하면 보고서만 발행하고 큐 등록을 건너뛴다.
- **수동 처리**: 자동 등록된 큐는 `/process-act start queue-qXXXXXXXX`로 처리를 시작한다.

## `/process-check`와의 차이

| 구분 | `/process-check` | `/process-audit` |
|---|---|---|
| 질문 | "우리가 절차대로 이행했는가?" | "우리 프로세스가 표준을 커버하는가?" |
| 입력 | PRO/WI 요건 vs REC 증적 | 외부 표준 조항 vs POL/PRO/WI 설계 |
| 기준 | 내부 (자체 PRO/WI) | 외부 (ISO/IEC/KS/법규) |
| 산출 | REC-AUDIT, NCR | REC-GAP |
| 내보내기 | 없음 | XLSX / HTML / PDF |

두 커맨드는 상호 보완적이다. 일반적으로 `/process-audit`으로 프로세스 설계의 표준 커버리지를 먼저 점검하고, GAP 보완 후 `/process-check`로 실행 이행을 심사하는 순서로 진행한다.

## 옵션 목록

### start 모드

| 옵션 | 효과 |
|---|---|
| `--against <표준코드>` | 대조할 외부 표준 지정 (필수). 예: `ISO9001`, `ISO27001`, `CMMI-ML3` |
| `--scope <명칭>` | 분석 범위 한정. 생략 시 vault 전체 POL/PRO/WI 대상 |
| `--strictness strict\|normal\|lenient` | 판정 엄격도 조정 (기본: normal) |
| `--llm-fallback` | inputs/ 없이 LLM 학습 데이터 기반으로 분석 (낮은 신뢰도) |
| `--dry-run` | 산출물 저장 없이 커버리지 추정 미리보기 |

### confirm 모드

| 옵션 | 효과 |
|---|---|
| `--not-applicable "<조항>"="<사유>"` | 특정 조항을 N/A로 처리하고 사유를 기록 |
| `--adjust-gap "<조항>"=<등급>` | GAP 등급 수동 조정 (critical/major/minor) |
| `--no-act-queue` | act 큐 자동 push 억제. 보고서만 발행 |

### export 모드

| 옵션 | 효과 |
|---|---|
| `--format xlsx\|html\|pdf` | 내보내기 포맷 선택 (필수) |
| `--template <템플릿명>` | 표준별 공식 양식 적용. 지원: `iso9001`, `iso27001`, `iso14001` |
| `--org <조직명>` | 내보내기 파일 헤더에 표시할 조직명 |

---

## 주의사항

1. `/process-audit`은 POL/PRO/WI/TMP/EX/REC/MAT 파일을 **절대 수정하지 않는다**. MAT-002 갱신과 신규 REC-GAP 발행만 수행한다.
2. `--llm-fallback` 결과는 인증기관 제출 자료로 사용하지 않는다. 반드시 `inputs/` 기반 분석 결과를 사용할 것.
3. XLSX 내보내기는 Python 스크립트를 생성하는 방식이다. 스크립트를 직접 실행해야 `.xlsx` 파일이 만들어진다.
4. 동일 표준에 대해 여러 번 실행하면 각각 독립된 run-g trace로 관리된다. 최신 run 결과가 MAT-002를 덮어쓴다.
