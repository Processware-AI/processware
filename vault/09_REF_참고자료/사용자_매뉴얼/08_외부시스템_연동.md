# 08. 외부 시스템 연동 — vault → Redmine

## 개요

vault에서 생성된 표준 프로세스 자산(POL/PRO/WI/TMP/EX/REC/MAT)을 **Redmine Process Library**에 단방향으로 Push하는 기능입니다.

**Headless CMS 모델**: vault = 콘텐츠 저장소(SSoT), Redmine = API 소비자. 역방향 동기화 없음.

```
vault (SSoT)
  ├── 03_POL_정책/
  ├── 04_PRO_절차/
  ├── 05_WI_업무지침/
  ├── 06_TMP_템플릿/
  └── 08_REC_기록/
        │
        │  /vault-push (단방향 Push)
        ▼
Redmine Process Library
  lib-iso27001/
    lib-iso27001-p001-001   ← PRO별 Library Module (READ ONLY)
    lib-iso27001-p001-002
        │
        │  /workspace-create (복사·조합)
        ▼
Workspace (팀 자율 편집)
  ws-product-alpha/
    ws-product-alpha-iso27001-p001-001
    ws-product-alpha-iso27001-p001-002
```

---

## 1. 사전 준비

### 1-1. 의존성 설치

```bash
pip3 install -r publish/redmine/requirements.txt
```

필요 패키지: `python-frontmatter`, `requests`, `PyYAML`

### 1-2. 설정 파일 수정

`publish/redmine/config.yaml` 에 실제 값 입력:

```yaml
redmine:
  url: "https://your-redmine.example.com"
  api_key: ""          # 환경변수 REDMINE_API_KEY 권장
```

### 1-3. 환경변수 설정

```bash
export REDMINE_API_KEY=your_api_key
export REDMINE_URL=https://your-redmine.example.com   # GitHub Actions용
```

### 1-4. Redmine 상위 프로젝트 생성 (1회)

Redmine 관리 화면에서 표준 코드별 상위 프로젝트를 수동 생성합니다.

| 표준 | 생성할 프로젝트 identifier |
|---|---|
| ISO 27001 | `lib-iso27001` |
| CMMI | `lib-cmmi` |
| ISO 9001 | `lib-iso9001` |

### 1-5. Library Module 서브프로젝트 자동 생성

```bash
/vault-push --setup --dry-run   # 생성 예정 미리보기
/vault-push --setup             # 실제 생성
```

PRO 파일의 frontmatter를 스캔하여 `lib-{scope_code}-{pro_slug}` 서브프로젝트를 자동 생성합니다. 결과는 `config.yaml by_pro` 섹션에 자동 기록됩니다.

### 1-6. git hook 설치 (선택, 로컬 자동화)

```bash
bash publish/redmine/install-hook.sh
```

커밋 시 `vault/*.md` 변경이 감지되면 백그라운드에서 자동 push합니다.

## 2. /vault-push — vault → Redmine 동기화

### 기본 사용법

| 명령 | 동작 |
|---|---|
| `/vault-push` | vault 전체 → Library 동기화 |
| `/vault-push --changed` | git diff 변경분만 (CI/CD 권장) |
| `/vault-push --setup` | Library 프로젝트 계층 자동 생성 |
| `/vault-push --setup --dry-run` | 생성 예정 미리보기 |
| `/vault-push WI-ISO27001-001-001-001` | 특정 문서 1건 |
| `/vault-push --type WI` | WI 전체 |
| `/vault-push --scope ISO27001` | 특정 표준 전체 |
| `/vault-push --dry-run` | 실제 전송 없이 미리보기 |
| `/vault-push --status` | 마지막 동기화 상태 및 통계 |

### 동기화 규칙

| 조건 | 동작 |
|---|---|
| `status: draft` | Skip (Library에 노출 안 함) |
| POL/PRO/WI/TMP/EX/MAT | Redmine Wiki 페이지로 Push |
| REC | Redmine Issue (Tracker: 프로세스기록) |
| REC-NCR | Redmine Issue (Tracker: NCR) |
| Obsidian `[[wikilink]]` | Redmine `[[WikiLink]]` 자동 변환 |
| 자동생성 헤더 | 모든 페이지에 삽입 (직접 편집 금지 안내) |

### REC/NCR 이슈 상태 매핑

| vault status | Redmine 상태 |
|---|---|
| open / in_progress | 진행중 |
| closed / done | 완료 |
| withdrawn / cancelled | 종결 |

### NCR 심각도 → Redmine 우선순위

| NCR severity | Redmine 우선순위 |
|---|---|
| critical | 긴급 |
| major | 높음 |
| minor | 보통 |

## 3. /workspace-create — Workspace 생성·관리

### 개념

Process Library의 표준 프로세스를 현업 업무 단위(Workspace)로 복사·조합합니다.

- **Library Module** (`lib-{scope}-{pro_slug}`): vault push 관리, 직접 편집 금지
- **Workspace Module** (`ws-{slug}-{scope}-{pro_slug}`): 생성 후 팀 소유, 자유 편집 가능
- 버전 업데이트가 필요하면 `/workspace-create sync`로 Library 최신 내용을 반영

### Workspace 생성

```bash
/workspace-create create \
  --name "제품Alpha 개발" \
  --slug product-alpha \
  --modules PRO-ISO27001-001-001,PRO-ISO27001-001-002
```

| 옵션 | 설명 |
|---|---|
| `--name` | 워크스페이스 표시 이름 |
| `--slug` | URL-safe 식별자 (`ws-{slug}` 로 생성) |
| `--modules` | 쉼표 구분 PRO doc_id 목록 |
| `--description` | 설명 (선택) |
| `--dry-run` | 실제 생성 없이 미리보기 |

생성 결과:

```
ws-product-alpha                        ← 상위 프로젝트
  ws-product-alpha-iso27001-p001-001   ← Library 복사본 (Module)
  ws-product-alpha-iso27001-p001-002   ← Library 복사본 (Module)
```

### Library 최신화 (선택적 버전 업데이트)

```bash
/workspace-create sync \
  --workspace ws-product-alpha \
  --module lib-iso27001-p001-001
```

Library Module의 최신 wiki 내용을 Workspace Module에 반영합니다. 기존 Workspace 편집 내용은 덮어쓰여질 수 있으니 주의.

### 목록 조회 및 상태 확인

```bash
/workspace-create list                               # 전체 Workspace 목록
/workspace-create status --workspace ws-product-alpha  # 특정 Workspace 상세
```

## 4. GitHub Actions 자동화

`.github/workflows/vault-push.yml` — `feat/*-output` 또는 `main` 브랜치에 `vault/**/*.md` 변경이 push될 때 자동 실행됩니다.

### 설정 필요 항목 (GitHub Repository Secrets)

| Secret | 설명 |
|---|---|
| `REDMINE_API_KEY` | Redmine API Key |
| `REDMINE_URL` | Redmine 서버 URL |

### 동작 흐름

```
push to main (vault/*.md 변경)
    → checkout (fetch-depth: 2)
    → pip install requirements.txt
    → Redmine 접속 확인
    → config.yaml URL 패치 (secret에서)
    → python3 sync.py --changed
    → sync log artifact 업로드 (30일 보존)
```

## 5. 동기화 상태 추적

모든 push 이력은 `.claude/states/redmine_sync.db` (SQLite)에 기록됩니다.

```bash
/vault-push --status   # 현재 동기화 상태 요약
```

출력 예시:
```
동기화 상태 요약:
  synced : 42
  failed :  2
  skipped:  8
```

## 6. 전체 워크플로우 예시

```bash
# 1. 최초 설정 (1회)
export REDMINE_API_KEY=your_key
pip3 install -r publish/redmine/requirements.txt
/vault-push --setup

# 2. ISO27001 표준 수립 후 Redmine에 push
/process-plan ISO27001                     # vault에 표준 자산 생성
/vault-push --scope ISO27001               # Redmine Library에 push

# 3. 프로젝트 팀용 Workspace 생성
/workspace-create create \
  --name "신제품 개발팀" \
  --slug newprod \
  --modules PRO-ISO27001-001-001

# 4. 이후 변경은 자동 (git hook 또는 GitHub Actions)
git commit -m "feat: WI 개정"              # → post-commit hook → 자동 push

# 5. Library 업데이트 후 Workspace 최신화
/vault-push --scope ISO27001
/workspace-create sync \
  --workspace ws-newprod \
  --module lib-iso27001-p001-001
```

## 7. 트러블슈팅

| 증상 | 원인 | 해결 |
|---|---|---|
| `REDMINE_API_KEY 미설정` 오류 | 환경변수 없음 | `export REDMINE_API_KEY=...` |
| 프로젝트 없음 오류 | 상위 Library 프로젝트 미생성 | Redmine 관리 화면에서 수동 생성 후 `--setup` |
| `by_pro 매핑 없음` | `--setup` 미실행 | `/vault-push --setup` 실행 |
| dry-run에서 wiki 복사 오류 | `client=None` 접근 시도 | 이미 수정됨 (v1.0) |
| Python 3.9 타입 오류 | `X \| Y` 구문 | `from __future__ import annotations` + `Optional` 사용 |
