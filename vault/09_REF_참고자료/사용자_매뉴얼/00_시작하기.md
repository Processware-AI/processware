# 00. 시작하기 — 15분 퀵스타트

> 이 페이지를 읽으면 처음 사용자도 15분 안에 첫 번째 프로세스 모듈을 빌드할 수 있다.

---

## 사전 준비

| 항목 | 최소 버전 | 확인 방법 |
|---|---|---|
| Claude Code CLI | 최신 | `claude --version` |
| git | 2.30 이상 | `git --version` |
| Obsidian | 최신 권장 | (선택) vault 시각화용 |

```bash
# 저장소 클론
git clone <저장소_URL> ~/MyProjects/STD_Process_Builder
cd ~/MyProjects/STD_Process_Builder

# Claude Code 실행
claude
```

---

## 핵심 개념 한 줄 요약

| 명령 | 한 줄 설명 | 언제 쓰나 |
|---|---|---|
| `/process-ingest` | 표준 문서 전처리 (Ingest) | ISO PDF 등을 요건 패키지로 변환할 때 |
| `/process-plan` | 프로세스 문서 빌드 (Plan) | 새 업무 체계를 만들 때 |
| `/process-do` | 프로세스 실행·기록 (Do) | 절차를 실제로 수행할 때 |
| `/process-check` | 프로세스 심사·점검 (Check) | 잘 지켜지고 있는지 확인할 때 |
| `/process-act` | 프로세스 개정·개선 (Act) | 문제를 찾아 고칠 때 |

---

## Step 1 — 입력자료 준비

### 자연어로 바로 시작하는 경우 (추천)

표준 PDF 없이 자연어로 필요한 업무 체계를 설명하면 된다. AI가 지식으로 추정해서 빌드한다.

→ Step 2로 바로 이동

### 표준 PDF를 기반으로 하는 경우

PDF를 sources/ 폴더에 배치하고 전처리를 먼저 실행한다.

```bash
# 1. sources/ 에 파일 배치
cp ~/Downloads/ISO9001_2015.pdf sources/

# 2. 전처리 실행
/process-ingest sources/ISO9001_2015.pdf --standard ISO9001 --version 2015
# → Phase 7 완료 후 정지. review_request.md 확인 요청.

# 3. HITL 검토 후 확정
/process-ingest --confirm ISO9001
```

→ 확정 후 Step 2로 이동

---

## Step 2 — 첫 번째 빌드 실행

Claude Code 프롬프트에서:

```
/process-plan "OOO사 품질경영체계"    # 조직이 원하는 이름으로 자유 명명
```

또는 자연어로:
```
/process-plan "프로젝트 계획·추정·리스크 관리"
```

**진행 과정 (약 10분)**:
1. AI가 슬러그 제안 → 확인 또는 수정
2. inputs/에 요건 패키지가 있으면 자동 활용, 없으면 "LLM 추정으로 진행" 선택
3. 적용요건.md 생성 → Analyze → Design → Write → Trace → QA 순서로 자동 진행
4. 완료 시 생성된 파일 목록 표시

---

## Step 3 — 결과 확인

빌드가 끝나면 새 브랜치(`feat/prj-planning-est-risk-output`)에 다음 파일들이 생성된다:

```
vault/
├── 02_적용요건/         적용요건.md (inputs 분석 결과)
├── 03_POL_정책/         POL-*.md  (정책 1~2건)
├── 04_PRO_절차/         PRO-*.md  (절차 1~3건)
├── 05_WI_업무지침/      WI-*.md   (업무지침 다수)
├── 06_TMP_템플릿/       TMP-*.md  (각 WI 대응 양식)
├── 07_EX_작성예시/      EX-*.md   (각 TMP 작성 예)
└── 90_MAT_통합매핑/     MAT-*.md  (추적성 매트릭스)
```

**Obsidian으로 보기**:
```bash
# worktree로 브랜치 폴더 별도 마운트
git worktree add ../prj-planning-view feat/prj-planning-est-risk-output
# Obsidian → Open Folder as Vault → ../prj-planning-view/vault 선택
```

---

## Step 4 — 절차 한 번 실행해보기

WI 번호를 확인하고 실행:

```
/process-do WI-PPR-01-01          # 실제 생성된 WI 번호로 변경
```

AI가 각 단계별로 질문하면 답변 → 자동으로 기록(REC) 생성.

---

## Step 5 — 심사 실행

```
/process-check start PRO-PPR-01 --auditor "나"
```

> 주의: 실제 운영에서는 직접 실행한 사람과 심사하는 사람이 달라야 한다.  
> 처음 테스트 시에는 `--override-independence` 플래그를 추가.
