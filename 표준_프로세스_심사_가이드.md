## 4. 사용법 (Phase 1)

### 4-1. 7개 진입 모드 (Phase 1: start/resume/confirm/reject-finding/status 동작 / list-ncr·close-ncr 는 Phase 2)

| 모드 | 명령 | 용도 |
|---|---|---|
| **start** | `/audit start PRO-CMMI-04-01 --auditor "이감사"` | 신규 심사 — PRO 단위 |
|  | `/audit start WI-CMMI-04-01-03 --auditor "이감사"` | 신규 심사 — 단일 WI (좁은 심사) |
|  | `/audit start CMMI-DEV-ML3 --auditor "이감사"` | 신규 심사 — 표준 전체 |
| **resume** | `/audit --resume run-a1c2d3e4` | 정지된 심사 재개 |
| **confirm** | `/audit --confirm run-a1c2d3e4` | 매트릭스 확정 → 보고서 발행 |
| **reject-finding** | `/audit --reject-finding F-002 --reason "..." --trace run-a1c2d3e4` | 부적합 인정 거부 (오탐 제거) |
| **status** | `/audit --status run-a1c2d3e4` | 현재 진행 단계 확인 |
| list-ncr (Phase 2) | `/audit --list-ncr` | NCR 목록 |
| close-ncr (Phase 2) | `/audit --close-ncr <id> --capa <REC>` | 시정조치 종결 |

### 4-2. 공통 옵션
| 플래그 | 효과 | 적용 모드 |
|---|---|---|
| `--dry-run` | 분석은 하되 REC-AUDIT 미저장·MAT-005 미갱신, 미리보기만 | start, resume, confirm |
| `--auditor "이름"` | 심사원 명시 (start 에서 필수) | start, confirm, reject-finding |
| `--override-independence` | 독립성 위반 무시 (PoC 한정 — Phase 4 에서 제거) | start |
| `--period from..to` | 심사 대상 기간 (REC.executed_at 기준). 기본 지난 90일 | start |
| `--strictness strict\|normal\|lenient` | 부적합 판정 임계 | start |
| `--reason "..."` | 인정 거부 사유 (reject-finding 필수) | reject-finding |
| `--adjust-finding F-NNN=등급` | 등급 조정 (critical/major/minor) | confirm |

### 4-3. 정상 흐름 예시 (PoC: PRO-CMMI-04-01)

```bash
# 1. 신규 심사 시작
/audit start PRO-CMMI-04-01 --auditor "이감사" --period 2026-01-01..2026-04-30

## 5. ISO §9.2 독립성 가드 (Phase 1 inline / Phase 4 정식 RBAC)

### 5-1. 동작
- `/audit start` 진입 시 MAT-005 §실행기록 의 본 심사 범위 trace 들 모두 스캔.
- `--auditor` 이름과 각 trace.`executed_by` 비교.
- 1건이라도 일치하면 abort + 위반 trace 목록 출력.

### 5-2. 위반 시 출력 예시
```

  - run-b7d4e3c5 (WI-CMMI-04-01-03, 2026-05-01)
  - run-c5f8a9d2 (WI-CMMI-04-01-04, 2026-05-01)
  - run-d8a3f6b7 (WI-CMMI-04-01-04, 2026-05-01)
본 심사를 진행하려면 다른 심사원을 지정하거나, 이행 trace 를 범위에서 제외하십시오.
