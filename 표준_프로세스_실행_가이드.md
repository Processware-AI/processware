## 1. 본 하네스가 하는 일

`/build-standard` 가 만든 표준 자산(POL/PRO/WI/TMP/EX/MAT/REF)을 **계약서**로 삼아, AI Agent 가 사람과 대화하며 절차를 이행하고 **REC(기록본)** 을 자동 작성한다.

```
[기존]  /build-standard ISO9001       → POL/PRO/WI/TMP/EX/MAT/REF (자산 — 차원 1 PLAN)
[신규]  /do WI-CMMI-04-01-03           → REC + MAT-005 갱신       (이행 — 차원 2 DO)
```

## 2. 구성 요소 (Phase 1)

```
.claude/
├── commands/
│   └── do.md                    ← /do 슬래시 커맨드 (진입점)
└── agents/
    ├── process-executor.md      ← WI 해석 · 대화 루프 · TMP 매핑
    └── rec-writer.md            ← REC 파일 생성 · MAT-005 갱신

vault/
├── 05_WI_업무지침/.cache/        ← WI → steps.yaml 자동 캐시 (실행 가속)
├── 08_REC_기록/                  ← 실행 산출물 (REC) 저장 위치
└── 90_MAT_통합매핑/
    └── MAT-005_심사증적_인덱스.md ← §"실행 기록" 섹션이 자동으로 누적

.claude/runs/{trace_id}/
├── state.yaml                   ← 실행 상태·HITL 대기·재개 정보
├── trace.jsonl                  ← 입력·LLM 출력·승인 전수 로그 (심사 증적)
└── rec_payload.yaml             ← process-executor → rec-writer 인계용 중간 산출물
```

## 3. 실행 흐름 (한눈에)

```
사용자: /do WI-CMMI-04-01-03
          │
          ▼
   [/do 커맨드]
   - 인자 파싱·WI 식별
   - trace_id 생성
   - state.yaml 초기화
          │
          ▼
   [process-executor]
   - WI / TMP / EX / PRO 로드
   - WI §5 → step 추출 (.cache/*.steps.yaml)
   - 사용자에게 step 별 질문 → 답변 수집
   - derivation (계산·판정) 자동 적용
   - HITL 게이트 발동 시 정지·승인
   - rec_payload.yaml 작성
          │
          ▼
   [rec-writer]
   - REC 일련번호 자동 결정
   - TMP 양식에 payload 매핑
   - vault/08_REC_기록/REC-*.md 저장
   - MAT-005 §"실행 기록" 1행 append
   - state.yaml 마감 + trace.jsonl 마감
          │
          ▼
   ✅ 완료 보고
```

## 4. Phase 1 사용법

### 4-1. 직접 실행 (WI 번호 알고 있을 때)

```
/do WI-CMMI-04-01-03
```

또는 전체 파일명:
```
/do WI-CMMI-04-01-03_작업산출물_평가_v1.0
```

### 4-2. 옵션

| 플래그 | 효과 |
|---|---|
| `--dry-run` | REC 파일 쓰지 않고 미리보기만 출력 |
| `--executor <이름>` | 실행자 명시 (미지정 시 시스템 사용자명) |
| `--auto-approve` | HITL 게이트 자동 승인 모킹 (PoC 검증용) |
| `--resume <trace_id>` | 이전 실행 재개 (Phase 2 에서 정식 지원) |

### 4-3. 자연어 진입 (Phase 3 까지는 후보 제시 후 사용자 선택)

```
/do 작업산출물 평가
```
→ Phase 1 에서는 후보 WI 목록을 보여주고 사용자에게 선택 요청. Phase 3 에서 `process-router` 에이전트가 자동 매칭으로 확장.

## 8. 안전 규칙 (4가지 강제)

1. **자산 영역 무결성** — 본 하네스는 `vault/03_POL/`, `04_PRO/`, `05_WI/`, `06_TMP/`, `07_EX/`, `09_REF/`, `90_MAT/MAT-001~004,006~` 의 어떤 파일도 수정하지 않는다. 위반 즉시 중단.
2. **REC 직접 수정 금지** — 일단 생성된 REC 는 심사 증적이므로 사람이 손대지 않는다. 정정 시 신규 REC 발행 (Phase 2 `--reissue`).
3. **trace.jsonl 누락 금지** — 모든 질문/답변/판정/승인 이벤트가 단일 JSON 라인으로 기록되어야 한다. 누락은 차원 3 심사에서 부적합 사유.
4. **번호 충돌 자동 회피** — 일련번호 동시 발생 시 자동으로 다음 번호 채택. 충돌 사실은 trace 에 `seq_collision_resolved` 메모.
