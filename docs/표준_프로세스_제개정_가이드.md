| 플래그 | 효과 | 적용 모드 |

| `--auto-approve` | PCB 즉시 승인 (PoC·테스트 한정 — 실운영 금지) | start, resume |

| `--no-mat001-update` | MAT-001 §개정 이력 갱신 보류 (테스트) | start, approve |

## 5. PCB 거버넌스 원칙

본 하네스의 핵심 게이트는 **PCB (Process Control Board)** 승인. ISO/IEC 27001 §9.3 (관리검토) 와 동형.

- **PCB 의 역할**: 표준 자산 개정의 최종 승인. AI 가 작성한 RCA + 개정 계획을 신중히 검토.
- **승인 매트릭스 (RBAC)**:
  - PCB 역할 = qmr 또는 admin (본 PoC 의 RBAC policy).
  - process_owner 는 PCB 멤버일 수 있음 (정책에 따라 — 운영 시 결정).
- **다단계 승인 (Phase 2.5 예정)**:
  - PCB 위원 3명 quorum.
  - critical → 7일 이내 / major → 14일 이내 / minor → 30일 이내 응답 SLA.
- **외부 채널 알림 (Phase 4.5)**: 이메일·Slack — PCB 위원회 일정 자동 안내.

## 9. 4차원 PDCA 폐쇄 루프 완성

본 가이드의 Phase 1 도입으로 4차원 PDCA 의 **모든 차원이 자동화**됨:

```
[차원 1 Plan]    /build-standard {표준}                          → POL/PRO/WI/TMP/EX/MAT/REF
                  ↓
[차원 2 Do]      /do {WI}                                        → REC + MAT-005 §실행기록
                  ↓
[차원 3 Check]   /audit start {PRO}                               → REC-AUDIT + finding
                 /audit --confirm                                  → NCR (REC-NCR-*) + MAT-009
                 /audit --kpi start                                → MAT-008 + 회귀 알림
                 (자동) → act-trigger → .claude/queues/act/queue-q*.yaml
                  ↓
[차원 4 Act]     /act start <queue_id>                            → As-Is 입력 파일 (vault/02_표준/{표준}/_inputs/04_AsIs/)
                  ↓                                                 + MAT-001 §개정 이력
                                                                    + 큐 done
[차원 1 재실행] /build-standard --from write --target {asset}    ← 사용자 실행 (Phase 1)
                                                                    개정판 v1.1 산출
                  ↓
[차원 2/3 재시작] (다음 사이클)
```

다음 큰 마일스톤:
- **Phase 2** — 다중 큐 일괄 처리 (의존성 그래프 + 통합 As-Is)
